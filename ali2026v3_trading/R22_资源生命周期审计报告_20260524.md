## 二、P0 级问题

### P0-1: shadow_strategy_engine.py  ThreadPoolExecutor 永不关闭

- **文件**: shadow_strategy_engine.py
- **创建点**: 第 243 行
  self._log_writer_executor = ThreadPoolExecutor(max_workers=1, thread_name_prefix='shadow_log')
- **使用点**: 第 746 行，self._log_writer_executor.submit(...)
- **释放点**: 不存在。类无 shutdown()/stop()/close() 方法。
- **泄漏速率**: 1 worker 线程 / 实例生命周期。若模块重载或多次实例化，线程累积。
- **影响**: 永驻 worker 线程占用 ~8MB 栈内存 + OS 线程表条目。

### P0-2: position_service.py  类级别 ThreadPoolExecutor 永不关闭

- **文件**: position_service.py
- **创建点**: 第 1028-1032 行（延迟创建）
  if PositionService._close_retry_executor is None: PositionService._close_retry_executor = ThreadPoolExecutor(max_workers=self.CLOSE_RETRY_MAX_THREADS, thread_name_prefix='pos_retry')
- **释放点**: 不存在。类属性创建后永不被 shutdown。
- **泄漏速率**: 最多 CLOSE_RETRY_MAX_THREADS 个 worker 线程。

### P0-3: strategy_core_service.py  线程和定时器无生命周期管理

- **文件**: strategy_core_service.py
- **资源清单**:

| 行号 | 资源类型 | 变量名 | 释放点 |
|:---:|---------|--------|:---:|
| 149 | threading.Thread | self._storage_warm_thread | 无 |
| 151 | threading.Thread | self._platform_subscribe_thread | 无 |
| 726 | threading.Timer | self._storage_warm_timer | 无 |
| 794 | threading.Timer | 局部变量 timer (创建即启动) | 无 |

- **释放点**: 类无 stop()/shutdown()/close()/cleanup() 方法（grep 返回 0 结果）。
- **泄漏速率**: 至少 2 Thread + 2 Timer / 实例。

### P0-4: ProductionQuantSystem.shutdown() 不完整

- **文件**: ProductionQuantSystem.py
- **现有 shutdown** (L310-317): 仅关闭 persistence + hot_config + hmm._em_thread
- **缺失清理**: event_bus, shadow_strategy_engine, position_service, strategy_core_service, risk_service, ds_db_connection, subscription_manager

## 三、P1 级问题

### P1-1: risk_service.py  递归 Timer 无 cancel 机制

- **文件**: isk_service.py
- **创建点**: 第 189-210 行
  - __init__: self._dashboard_timer = threading.Timer(60.0, callback); .daemon=True; .start()
  - 回调 _dashboard_timer_callback() 中递归重建 Timer
- **释放点**: 无 cancel。daemon=True 保底，但模块重载时旧 Timer 不释放。
- **影响**: Timer 计数随模块重载递增。

### P1-2: ds_db_connection.py  close_all() 存在但从未被调用

- **文件**: ds_db_connection.py
- **资源**: close_all() (L138-154), _stop_monitor Event (L140)
- **释放点**: close_all() 全代码库 grep 无任何调用者。
  - 唯一设置 _stop_monitor.set() 的地方 -> 后台 DuckDB-PerfMonitor 线程永不被停止
  - _all_connections 逐线程累积
- **修复建议**: 在 shutdown 链路中调用 close_all()。

### P1-3: event_bus.py  shutdown 依赖 atexit，主 shutdown 路径未级联

- **文件**: event_bus.py
- **释放点**: L611-626 shutdown() 正确实现; L873 atexit 注册
- **问题**: ProductionQuantSystem.shutdown() 未主动调用 event_bus.shutdown()。
  - atexit 中 wait=False 可能丢失未处理回调。
