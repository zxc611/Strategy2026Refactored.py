## 六、第三轮审计关键发现

### 6.1 与前两轮审计的核心差异

本轮审计从手册出发正向追踪数据流，发现了前两轮**从代码反向核对手册**时遗漏的4个致命问题：

1. **decision_score_filter默认关闭（R3-D-01）**：前两轮仅检查了compute_decision_score()是否存在和被调用，未发现SignalService中`_decision_score_filter_enabled = False`导致整个11维度评分系统形同虚设。这是"功能存在但未真正起作用"的典型。

2. **position_scale未传递到回测仓位计算（R3-D-02）**：前两轮确认评分→signal的传递通畅，但未追踪signal→_compute_lots()的最终消费，position_scale写入signal字典后回测引擎不读取。

3. **_check_safety_meta_layer异常时返回PASS（R3-D-04）**：前两轮检查了断路器逻辑是否完整，但未发现try/except块中SafetyMetaLayer异常时返回PASS，断路器检查失效时交易继续。

4. **TVF三方默认值冲突（R3-P-04）**：前两轮确认TVF六维度完整实现，但未发现ModeConfig默认tvf_enabled=False导致TVF完全不生效。

### 6.2 "回测冠军实战崩盘"核心根因链

基于三轮审计的交叉验证，核心根因链如下：

```
回测冠军实战崩盘
  ├── 根因1: TVF默认关闭 → 六维仓位调整不生效 → 回测/实盘仓位计算一致但均无风控缩放
  ├── 根因2: 11维度评分默认关闭 → 评分不影响交易决策 → 回测/实盘均无评分过滤
  ├── 根因3: Alpha衰减信号无消费方 → 影子策略形同虚设 → 策略失效时无法自动降级
  ├── 根因4: 到期日滑点倍增恒返回1.0 → 回测低估到期附近成本 → 实盘到期日交易亏损
  ├── 根因5: box_extreme/spring开仓零滑点 → 回测过于乐观 → 实盘滑点吃掉利润
  ├── 根因6: governance与主流程脱节 → E7-E13检测形同虚设 → 策略异常时无自动干预
  └── 根因7: judgment_report不影响Optuna → 过拟合策略可能入选 → 回测冠军实盘崩盘
```
