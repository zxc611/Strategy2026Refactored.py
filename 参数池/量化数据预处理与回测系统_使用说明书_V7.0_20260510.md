# 量化数据预处理与回测系统 使用说明书

**版本：** V7.0  
**日期：** 2026-05-10  
**适用范围：** ali2026v3_trading 数据预处理 + 参数网格回测  
**前置依赖：** Python 3.10+, duckdb, pandas, numpy, pyyaml, tqdm, pyarrow

---

## 一、系统总览

本系统由三个模块组成，形成从原始Tick到P0验证绿灯的完整流水线：

```
原始Tick(Parquet)
      │
      ▼  preprocess_ticks.py
preprocessed.duckdb（分钟Bar + 衍生指标）
      │
      ▼  task_scheduler.py
quant_results.duckdb（回测绩效 + 最优参数）
      │
      ▼  参数池统一执行方案
生产策略参数锁定（P0绿灯）
```

| 模块 | 文件 | 功能 | 输入 | 输出 |
|------|------|------|------|------|
| 数据预处理 | `preprocess_ticks.py` | Tick→分钟Bar + 五态/Greeks/订单流 | Parquet Tick文件 | `preprocessed.duckdb` |
| 任务调度 | `task_scheduler.py` | 两轮分层参数网格扫描 + 多进程回测 | `preprocessed.duckdb` | `quant_results.duckdb` |
| 参数池 | `params_service.py` + `parameter_attribute_matrix.yaml` | 属性矩阵校验 + 审计日志 + 路由优先级 | YAML | 运行时校验 + 审计日志 |

## 二、数据预处理：preprocess_ticks.py

### 2.1 功能概要

将原始Tick数据（Parquet格式）聚合为分钟级Bar，同时计算衍生指标，写入DuckDB。

**衍生指标一览：**

| 类别 | 字段 | 来源 |
|------|------|------|
| OHLCV | open, high, low, close, volume, tick_count | Tick聚合 |
| 资金流 | turnover, vwap | Tick聚合（自动检测列是否存在） |
| 持仓量 | open_interest | Tick聚合（取最后值） |
| 五态 | correct_rise/fall_pct, wrong_rise/fall_pct, other_pct, strength | `compute_option_state_vectorized()` |
| 订单流 | imbalance, consistency | `compute_order_flow_vectorized()` |
| Greeks | iv, delta, gamma, vega, theta | `compute_greeks_vectorized()` |

### 2.2 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `TICK_DATA_DIR` | `/data/ticks` | 原始Tick Parquet文件目录 |
| `OUTPUT_DB` | `preprocessed.duckdb` | 输出DuckDB文件 |
| `MIN_DATE` | `2020-01-01` | 数据起始日期 |
| `MAX_DATE` | `2026-05-09` | 数据截止日期 |
| `SYMBOLS` | 49品种（已去重） | 待处理品种列表 |
| `MAX_WORKERS` | `min(cpu//2, len(SYMBOLS))` | 并行进程数 |
| `ROWS_PER_CHUNK` | 500,000 | 分块行数阈值 |

### 2.3 运行方法

```bash
# 直接运行
python ali2026v3_trading/preprocess_ticks.py

# 指定数据目录（修改脚本顶部 TICK_DATA_DIR）
# 或通过环境变量
export TICK_DATA_DIR=/data/ticks_prod
python ali2026v3_trading/preprocess_ticks.py
```

### 2.4 架构关键设计

**1. 多进程安全写入**

```
子进程1 → 临时Parquet_1
子进程2 → 临时Parquet_2      → 主进程串行合并 → preprocessed.duckdb
...
子进程N → 临时Parquet_N
```

每个子进程写独立临时Parquet文件（消除DuckDB并发写冲突），主进程串行合并（单写者安全），`finally`块确保临时文件删除。

**2. 分钟边界安全分块**

`_minute_safe_chunk_indices()` 保证同一分钟的Tick永远不会被拆到两个Chunk中：

```
Tick流: [09:01的200tick] [09:02的200tick] [09:03的200tick]
ROWS_PER_CHUNK=300

❌ 固定行数分块: [0:300] [300:600]  → 09:02被切断 → 重复分钟Bar
✅ 分钟边界分块: [0:400] [400:600]  → 09:02完整 → 无重复
```

**3. 数据质量过滤**

`_filter_bars()` 严格执行：
- volume > 0（无成交分钟剔除）
- OHLC > 0（零价Tick剔除）
- high ≥ low, high ≥ open/close, low ≤ open/close（OHLC一致性）
- 日期范围裁剪

**4. 大品种按年拆分子任务**

`_build_task_list()` 自动检测文件大小，超过2GB的品种按年份拆分为独立子任务，避免单品种拖慢并行进度。

### 2.5 输出验证

```bash
python -c "
import duckdb
con = duckdb.connect('preprocessed.duckdb', read_only=True)
print('行数/日期范围:', con.execute(

con.close()
"
```

## 三、任务调度与回测：task_scheduler.py

### 3.1 功能概要

两轮分层参数网格扫描，多进程并行回测，结果存入DuckDB，输出Top5最优参数及样本外衰减。

### 3.2 两轮分层优化策略

**问题：** 16个参数全排列 = 2.12亿组合，不可行。

**方案：** 分层优化——先粗扫核心参数，再精扫辅助参数。

| 轮次 | 扫描参数 | 组合数 | 任务数 | 策略 |
|------|----------|--------|--------|------|
| **Round1粗扫** | 6个核心交易参数 | 972 | 1,944 | 全排列，找Top10 |
| **Round2精扫** | 10个辅助参数 | 1,024/Top | 20,480 | 固定Round1核心，扫描辅助 |
| **合计** | 16参数全覆盖 | — | **22,424** | ~30分钟@14核 |

**Round1核心参数（6个）：**

| 参数 | 搜索范围 | 说明 |
|------|----------|------|
| `close_take_profit_ratio` | [1.1, 1.5, 2.5] | 止盈倍数 |
| `close_stop_loss_ratio` | [0.3, 0.4, 0.5, 0.6] | 止损比例 |
| `max_risk_ratio` | [0.2, 0.3, 0.5] | 最大风险比例 |
| `lots_min` | [1, 3, 5] | 最小开仓手数 |
| `signal_cooldown_sec` | [0, 60, 120] | 信号冷却(秒) |
| `non_other_ratio_threshold` | [0.3, 0.4, 0.5] | 非other状态触发阈值 |

**Round2辅助参数（10个）：**

| 参数 | 搜索范围 | 说明 |
|------|----------|------|
| `max_signals_per_window` | [3, 8] | 窗口最大信号数 |
| `state_confirm_bars` | [2, 5] | 状态确认bar数 |
| `spring_stop_profit_ratio` | [3.0, 8.0] | 弹簧止盈倍数 |
| `spring_max_loss_pct` | [0.85, 0.95] | 弹簧最大亏损 |
| `spring_max_position_pct` | [0.010, 0.025] | 弹簧最大仓位 |
| `capital_route_master_base` | [0.50, 0.70] | 主策略资金基数 |
| `shadow_alpha_threshold` | [0.0, 0.2] | 影子Alpha阈值 |
| `rate_limit_global_per_min` | [30, 120] | 全局下单速率 |
| `hard_time_stop_minutes` | [60, 120] | 硬时间止损(分) |
| `daily_loss_hard_stop_pct` | [0.03, 0.08] | 日回撤硬停止 |

### 3.3 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `PREPROCESSED_DB` | `preprocessed.duckdb` | 预处理数据源 |
| `RESULTS_DB` | `quant_results.duckdb` | 回测结果输出 |
| `MAX_WORKERS` | `min(cpu//2, 16)` | 并行进程数 |
| `TRAIN_START` | `2023-01-01` | 训练集起始 |
| `TEST_START` | `2025-01-01` | 测试集起始（训练集结束） |
| `TEST_END` | `2026-12-31` | 测试集结束 |
| `TARGET_SYMBOLS` | `None`（全部品种） | 品种过滤列表 |
| `INITIAL_EQUITY` | 1,000,000 | 初始权益 |
| `COMMISSION_PER_LOT` | 1.5 | 每手佣金 |
| `SLIPPAGE_BPS` | 1.0 | 滑点(bps) |
| `ROUND1_TOP_K` | 10 | Round1保留Top-K进入Round2 |

### 3.4 运行方法

```bash
# 前提：preprocessed.duckdb 已生成
python ali2026v3_trading/task_scheduler.py
```

### 3.5 回测引擎逻辑

每个参数组合的回测执行完整V7策略闭环：

```
逐Bar循环:
  1. 日切重置（权益/断路器）
  2. 状态路由（五态→三态 + 确认机制）
  3. 持仓检查（止盈 → 止损 → 硬时间止损 → 收盘平仓）
  4. 安全检查（日回撤硬停止 + 断路器暂停）
  5. 信号生成（strength > 0.3 + 冷却 + 持仓上限）
  6. 开仓执行（风险比率约束手数 + 佣金/滑点扣减）
  7. 权益曲线追加
```

**止盈止损路由优先级（与生产代码一致）：**

```
P3: reason_overrides（开仓理由级，如CORRECT_RESONANCE=1.5/0.50）
P2: state_overrides（状态级，如correct_trending=2.5/0.4）
P1: default（默认值，1.5/0.50）
```

### 3.6 结果查询

```bash
python -c "
import duckdb, json
con = duckdb.connect('quant_results.duckdb', read_only=True)

# Top5 训练夏普
print('=== Top5 训练夏普 ===')
for row in con.execute('''
    SELECT sharpe, total_return, max_drawdown, params_json
    FROM backtest_results
    WHERE is_train = true AND sharpe IS NOT NULL
    ORDER BY sharpe DESC LIMIT 5
''').fetchall():
    p = json.loads(row[3])
    print(f'  夏普={row[0]:.3f}  收益={row[1]:.4f}  回撤={row[2]:.3f}  tp={p.get(\"close_take_profit_ratio\")}  sl={p.get(\"close_stop_loss_ratio\")}')

# 样本外衰减
print('\\n=== 样本外衰减 Top5 ===')
for row in con.execute('''
    SELECT t.sharpe, e.sharpe, t.params_json
    FROM backtest_results t
    JOIN backtest_results e ON t.params_json = e.params_json
    WHERE t.is_train=true AND e.is_train=false
      AND t.sharpe IS NOT NULL AND e.sharpe IS NOT NULL
    ORDER BY t.sharpe DESC LIMIT 5
''').fetchall():
    decay = (row[1]-row[0])/row[0] if abs(row[0])>1e-8 else 0
    print(f'  训练={row[0]:.3f}  测试={row[1]:.3f}  衰减={decay:.1%}')

con.close()
"
```

## 四、参数池基础设施

### 4.1 parameter_attribute_matrix.yaml

392行参数属性矩阵，定义每个参数的元属性：

| 元属性 | 说明 | 示例 |
|--------|------|------|
| `type` | 数据类型 | float, int, bool |
| `default` | 默认值 | 1.5 |
| `range` | 取值范围 | [0.5, 10.0] |
| `layer` | 所属层级 | L1_safety, L1_trading |
| `editable` | 可编辑性 | runtime, runtime_readonly, restart_required |
| `state_routed` | 是否状态路由 | true/false |
| `state_overrides` | 状态级覆盖 | correct_trending=2.5 |
| `reason_overrides` | 理由级覆盖 | CORRECT_RESONANCE=1.5 |
| `side_effects` | 联动参数 | debug_mode → rate_limit=300 |

**路由优先级（显式声明）：**

```yaml
route_priority:
  reason_overrides: 3    # 最精细，直接决定资金盈亏
  state_overrides: 2     # 中等粒度
  default: 1             # 兜底
```

### 4.2 ParamsService 集成

`ParamsService` 启动时自动加载 `parameter_attribute_matrix.yaml` 并执行全参数校验：

| 校验项 | 方法 | 说明 |
|--------|------|------|
| 类型校验 | `_check_type()` | int/float/bool/str |
| 范围校验 | `_validate_param_against_matrix()` | set()写入前拦截越界值 |
| 依赖约束 | `_check_dependency_constraint()` | calm_period≥pause_sec, spring_tp*(1-loss)>1.0 |
| 互斥规则 | `mutual_exclusion.rules` | debug_mode与stress_test_mode不可同时开启 |
| L1_safety只读 | `editable=runtime_readonly` | 运行时修改触发警告 |

### 4.3 审计日志

`ParamAuditObserver` 自动注册到 `ParamsService`，任何参数修改均记录结构化审计日志：

```
2026-05-10 14:30:00 | AUDIT | param_changed | key=close_take_profit_ratio | old=1.5 | new=2.0 | stack=[...]
2026-05-10 14:30:00 | AUDIT | SAFETY_PARAM_MODIFIED | key=close_take_profit_ratio | old=1.5 | new=2.0
```

安全关键参数（8个）修改时双重日志（INFO + WARNING）：

```
close_take_profit_ratio, close_stop_loss_ratio,
daily_loss_hard_stop_pct, circuit_breaker_trigger_sigma,
max_risk_ratio, hard_time_stop_minutes,
spring_stop_profit_ratio, spring_max_loss_pct
```

## 六、故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| `preprocess_ticks.py` 报OOM | 单个Parquet文件过大 | 减小 `ROWS_PER_CHUNK`（如250000） |
| DuckDB `CatalogException` | 多进程并发写 | 确认使用临时Parquet架构（已修复） |
| 回测sharpe全为0 | `preprocessed.duckdb` 为空 | 先运行预处理，确认行数>0 |
| 回测参数敏感性为零 | 仿真数据信号质量不足 | 使用真实分钟Bar数据 |
| `Attribute matrix validation FAILED` | 参数值违反YAML约束 | 检查日志中的VIOLATION详情，修正参数 |
| 审计日志中出现 `SAFETY_PARAM_MODIFIED` | 安全关键参数被修改 | 确认是否为预期操作，非预期则回滚 |
| Round2耗时过长 | `ROUND1_TOP_K` 过大 | 减小为5（默认10） |
