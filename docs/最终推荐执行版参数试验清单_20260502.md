## 一、执行前硬性约束

每一组试验在运行前必须先通过以下约束校验：

| 约束ID | 表达式 | 说明 |
| --- | --- | --- |
| C1 | `close_take_profit_ratio > 1.0` | 避免止盈逻辑失真 |
| C2 | `0 < close_stop_loss_ratio < 1.0` | 避免止损逻辑反转 |
| C3 | `0 < max_risk_ratio <= 1.0` | 风险比率上限必须有效 |
| C4 | `option_width_min_threshold >= 1.0` | 宽度阈值不可过低 |
| C5 | `signal_cooldown_sec <= rate_limit_window_sec` | 冷却不得大于限频窗口 |
| C6 | `max_signals_per_window >= 1` | 限频上限必须有效 |
| C7 | `history_minutes >= 60` | 历史数据长度不得过短 |
| C8 | `history_load_batch_size >= 1` | 历史批次大小必须有效 |
| C9 | `history_load_max_workers >= 1` | 历史并发度必须有效 |
| C10 | `max_hold_minutes >= 1` | 时间止损必须有效 |
| C11 | `subscription_batch_size >= 1` | 订阅批大小必须有效 |
| C12 | `subscription_interval > 0` | 订阅节奏必须有效 |

3. 在极端配置下仍能稳定退出和回收资源。

## 六、阶段 D：Walk-forward 验证

目标：过滤只在单一时间窗有效的伪优解。

执行方式：

1. 从阶段 B 选出前 3 名参数组。
2. 每组做 3 个滚动窗口验证，形成 9 次验证运行。
3. 若某组只在训练窗优秀、在 2 个及以上验证窗明显失效，则淘汰。

建议窗口：

| 窗口 | 训练 | 验证 |
| --- | --- | --- |
| WF1 | 窗口 1 | 窗口 2 |
| WF2 | 窗口 2 | 窗口 3 |
| WF3 | 窗口 3 | 窗口 4 |

阶段 D 保留标准：

1. 3 个窗口中至少 2 个窗口收益为正或收益回撤比达标。
2. 回撤波动不出现失控放大。
3. 参数排名在不同窗口不发生完全反转。

---

## 七、阶段 E：Phase 2 升级入口

只有满足以下条件，才进入 Phase 2：

1. 阶段 A 到 D 已完成。
2. 已经识别出 5 个以内关键因子。
3. 已有稳定的参数覆盖写入方式和结果汇总脚本。

推荐入口：

1. 若目标是快速寻优，用贝叶斯优化 15 到 30 组。
2. 若目标是统计显著性与局部曲率，用 RSM CCD 15 组左右。
3. 若目标是补全空间认知，用 LHS 或 Sobol 做额外均匀探索。

## 八、条件能力试验，不纳入主清单统计

以下试验仅在测试脚本或 harness 已经就位时启用，不纳入当前主执行清单评分：

1. 等权排序对比试验，如 `ranking_mode=equal_weight_round_robin`。

4. EventBus、Storage 队列水位、异步落盘等纯工程压力试验。
