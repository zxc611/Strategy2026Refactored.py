## 三、箱体极值策略设计原理

### 3.1 逻辑可行性

**核心论点：** 共振策略在箱体内不是不能用，而是用在"转折点"上。

| 阶段 | 共振含义 | 操作 |
|---|---|---|
| 趋势行情 | 方向延续的确定性 | 追趋势 |
| 箱底极值 | 下跌共振 → "跌透了" → 方向衰竭 | 反向做多 |
| 箱顶极值 | 上涨共振 → "涨透了" → 方向衰竭 | 反向做空 |

这与`incorrect_reversal`逻辑同源：当市场结构显示方向即将衰竭时，反着做。

### 3.2 买方在箱体中的致命问题与对策

| 问题 | 对策 | 实现参数 |
|---|---|---|
| 亏Theta：时间成本 | 极短硬时间止损(30min) | `max_hold_minutes=30` |
| 亏Vega回落：IV高位买入后回落 | IV必须处于近期高位才入场 | `iv_percentile_min=50.0` |
| 容错空间小 | 极窄止盈(0.4x)、极紧止损(0.3x) | `take_profit_ratio=0.4, stop_loss_ratio=0.3` |
| 需要极高胜率 | 仅在置信度≥0.6+IV过滤+衰竭确认时入场 | `min_extreme_confidence=0.6` |

### 3.3 箱体极值状态定义

| 子状态 | 触发条件 | 操作方向 |
|---|---|---|
| `box_bottom_extreme` | 期货在箱体下沿 + 全链期权下跌共振 + IV高位 + 流衰竭 | long |
| `box_top_extreme` | 期货在箱体上沿 + 全链期权上涨共振 + IV高位 + 流衰竭 | short |

### 3.4 箱体检测方法（三法融合）

| 方法 | 实现 | 调用现有接口 |
|---|---|---|
| 价格振幅法 | 近期高低价差/收盘价 ≤ `box_width_max_pct` + ADX<25 | `update_bar()` + `_compute_adx_simplified()` |
| 行权价聚类法 | 支撑/阻力位由低价/高价聚类产生 | `_find_support_resistance()` (新增) |
| 弹跳计数法 | 价格触及边沿次数 ≥ `min_bounce_count` | `detect_box()` 内统计 |
