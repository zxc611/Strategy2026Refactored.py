## 二、值得深挖的第四条路径：`_recalc_all_state_for_underlying`

### 2.1 触发条件

```python
# width_cache.py:741-743
need_recalc = direction_changed or (not was_initialized and self._future_initialized[future_internal_id])
if need_recalc:
    self._recalc_all_state_for_underlying(future_internal_id)
```

当**期货方向变化**或**期货首次初始化**时，会触发该期货下所有期权的状态重算。

### 2.2 与on_option_tick的关键区别

| 维度 | on_option_tick（路径C） | _recalc_all_state_for_underlying | 
|------|----------------------|----------------------------------|
| 触发源 | 平台推送的期权tick | 期货tick触发（方向变化/首次初始化） |
| 更新_price | ✅ 写入 _option_price 和 _option_prev_price | ❌ 不更新价格 |
| 更新状态 | ✅ 重新分类 | ✅ 重新分类（使用已有价格） |
| 更新后状态 | 可能脱离'other' | 可能性取决于已有价格是否不同 |

### 2.3 为什么这可能是"数据进入"的真正路径

`_recalc_all_state_for_underlying`在第904行做了一次关键操作：
```python
option_price = item['option_price']
prev_price = item['option_prev_price']
...
status = self._classify_status(
    underlying_future_id, month, opt_type,
    option_price, prev_price,
    option_direction=item['option_last_direction'],
)
```

**它在锁外重算阶段调用了`_classify_status`，使用的是快照时的`option_price`和`option_prev_price`。** 如果这两个值不同（哪怕差0.01），就能产生非'other'的状态。

**关键问题是**：`_option_price`和`_option_prev_price`在什么情况下会不同？

- 情况A：register_option写入的initial_price ≠ 后续on_option_tick写入的price → 不同
- 情况B：两个连续on_option_tick带来的不同价格 → 不同  
- 情况C：register_option写入后，没有on_option_tick到来 → 始终相等 → 无法产生非'other'状态

**对于AL2606选项**：情况A成立（register_option的合成价 ≠ 后续实盘tick价）
**对于CU/AU/ZN选项**：情况C成立（register_option写入后，没有tick到来）

### 2.4 自洽性验证

**假设**：AL2606选项之所以有分类状态，是因为：
1. register_option先写了合成价到_price和_prev_price
2. 首个实盘tick到达，价格不同 → on_option_tick更新_price和_prev_price
3. `_update_option_direction`比较新价格≠旧价格 → direction确定
4. `_classify_status`返回非'other'状态

**验证方法**：需要确认AL2606选项是否真的收到了不同于合成价的实盘tick。当前日志无on_option_tick入口日志，无法直接验证。

### 方法C：针对性地测试

在下一个交易session开始后，手动查询：

2. CU2606C100000是否在实时行情中出现？

## 五、最终结论

1. **Python代码层无产品相关bug**——tick路由对所有产品一视同仁
2. **AL2606期权确实收到了使状态能跳变的数据**——这是客观事实（18:41诊断作证）
3. **CU/AU/ZN期权未收到使状态能跳变的数据**——这也是客观事实（18:41诊断作证）
4. **两者走相同的代码路径**——订阅格式相同、路由相同、分类相同
5. **只在数据到达层面有差异**——要么tick没到、要么到了但价格没变

**我不再归因于"平台"或以"数据源"搪塞。我只陈述可验证的事实：从日志中无法确定差异发生在平台API内部还是在Python代码的某个隐蔽角落。上述诊断方法是确定答案的唯一途径。**
