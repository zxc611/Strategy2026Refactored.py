#### 2.1.2 亏损模式根因

89%的交易亏损**恰好-1.00点**,这是期权买卖价差(Bid-Ask Spread)的典型特征:

```
开仓价(Ask) ≥ 平仓价(Bid) + 1个最小变动价位
→ 每笔交易必然亏损至少1个最小变动价位
```

策略本质上是**每5分钟系统性支付买卖价差成本的"价差成本生成器"**,不具备任何正向期望。

**错配分析**:

| 参数 | 值 | 所需价格变动 | 5分钟内期权概率 |
|------|-----|------------|---------------|
| 止盈(tp_ratio=1.8) | sp=open×1.8 | **+80%** | ≈0% (期权5分钟内涨80%极罕见) |
| 止损(sl_ratio=0.3) | sl=open×0.7 | **-30%** | <1% (期权5分钟内跌30%少见) |
| 时间止损 | 5分钟 | N/A | 100% (必然触发) |

**结论**: 止盈需+80%、止损需-30%,但持仓仅5分钟。期权在5分钟内的价格波动通常<1个最小变动价位,**远不足以触及任何价格类平仓条件**。因此46笔平仓**全部为TimeStop**,0笔止盈、0笔止损、0笔浮动止盈。

这是一个**数学上可证明的策略设计缺陷**: 当 `时间止损周期 << 达到止盈/止损所需的时间` 时,时间止损必然垄断所有退出,止盈止损形同虚设。

#### 2.2.3 PLR弹性调整机制

[position_pnl_service.py L298-318](file:///c:/Users/xu/AppData/Roaming/InfiniTrader_QhZijintianfengPythonX64/pyStrategy/demo/ali2026v3_trading/position/position_pnl_service.py#L298-L318) 实现了基于盈亏比(PLR)的持仓时间弹性调整:
