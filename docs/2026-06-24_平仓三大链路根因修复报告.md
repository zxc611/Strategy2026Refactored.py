## 三、修复方案

### 3.1 核心设计

**PositionRecord 新增字段**：

| 字段 | 类型 | 默认值 | 含义 |
|---|---|---|---|
| `closing_order_id` | str | '' | 平仓订单唯一绑定 |
| `open_signal_snapshot` | str | '' | 开仓状态快照 |
| `close_method` | str | '' | 平仓方式 |

**`closing_order_id` 状态机**：

| 状态 | 值 | 含义 |
|---|---|---|
| 未平仓 | `''` | 无平仓订单 |
| 已锁定(下单前) | `PENDING_{position_id}` | 防竞态窗口 |
| 下单成功 | 实际order_id | 唯一绑定 |
| 追单成功 | 新order_id | 追单更新 |
| 重试中 | `RETRY_{position_id}` | 重试路径锁定 |
| 平仓完成/重置 | `''` | 清除绑定 |

**position_id 新格式**：`{instrument_id}_{策略组}_{毫秒时间戳}_{6位随机ID}`

**开仓快照**：`sig={signal_id}|reason={open_reason}|strat={strategy_group}|order={order_id}`

### 10.2 修复详情

#### 10.2.1 开仓唯一性（5个）

**O-05 排除平仓中持仓的开仓计数**
- 文件: `position_command_service.py` `_add_position`
- 问题: `_existing_same_group`计数未排除已在平仓中的持仓(closing_order_id非空)，导致已触发平仓但尚未完全成交的持仓阻止新的开仓信号
- 修复: 计数条件添加`and not getattr(_r, 'closing_order_id', '')`

**O-06 self_trade_detector.add_order移入锁内**
- 文件: `position_command_service.py` `_add_position`
- 问题: `self_trade_detector.add_order`在`instrument_lock`外执行，并发开仓时两个线程可能同时通过自成交检测但都未及时add_order
- 修复: 将`add_order`延迟到`instrument_lock`内执行，保证检测-注册原子性

**O-07 execute_by_ranking防重复开仓**
- 文件: `order_executor.py` `execute_by_ranking`
- 问题: 批量下单时同一合约同方向可能被多次触发开仓信号导致重复挂单
- 修复: 添加`_seen_open_instruments`去重集合 + 检查order_service中未成交OPEN订单

**O-08 position_id微秒精度**
- 文件: `position_command_service.py` `_add_position`
- 问题: position_id使用毫秒+6位随机ID，高频场景下可能冲突
- 修复: 升级为微秒+8位随机ID

**O-09 策略组为空时归类**
- 文件: `position_command_service.py` `_add_position`
- 问题: 策略组为空时与所有空策略组持仓合并计数
- 修复: 空策略组归类为`_default_`

#### 10.2.2 平仓唯一性（8个）

**C-07 PENDING/RETRY状态区分**
- 文件: `position_command_service.py` `_trigger_close_position`
- 问题: `closing_order_id`非空时直接return，未区分PENDING/RETRY/实际order_id三种状态
- 修复: 三种状态分别记录不同级别日志

**C-08 operation_id包含signal_id**
- 文件: `position_command_service.py` `_trigger_close_position`
- 问题: 网络重试`operation_id`不含signal_id，同持仓不同信号的平仓操作可能共享重试状态
- 修复: `operation_id`追加signal_id后缀

**C-09 重试worker读取record属性加锁**
- 文件: `position_command_service.py` `_schedule_close_retry`
- 问题: 重试worker读取record属性(volume/exchange/signal_id等)无锁保护
- 修复: 在`_get_instrument_lock`内捕获属性快照

**C-10/C-11 部分平仓/删除持仓重置close_method**
- 文件: `position_command_service.py` `_reduce_position`
- 问题: 部分平仓和删除持仓时未重置`close_method`，残留值误导后续平仓路径
- 修复: 两处都添加`rec.close_method = ''`

**C-12 紧急平仓设置close_method**
- 文件: `order_chase_service.py` `emergency_close_all_positions`
- 问题: 设置PENDING_EMERGENCY状态时未设置close_method，失败时也未重置
- 修复: PENDING状态时设置`close_method='emergency_close'`，失败时重置

**C-13 熔断减仓失败重置close_method**
- 文件: `safety_meta_circuit.py` `_force_position_reduction_on_circuit_breaker`
- 问题: 熔断减仓失败时重置_closing和closing_order_id但未重置close_method
- 修复: 添加`rec.close_method = ''`

**C-14 追单成功更新close_method**
- 文件: `order_chase_service.py` `_chase_reorder`
- 问题: 追单成功更新closing_order_id但未更新close_method
- 修复: 追单成功时设置`close_method = _prev_method or 'chase_reorder'`

#### 10.2.3 更新及时性（4个）

**U-08 slope写回前验证快照**
- 文件: `position_command_service.py` `on_tick`
- 问题: `_computed_slopes`锁外计算后锁内写入，期间profit_history可能被追加新数据
- 修复: 写入前验证profit_history长度未变

**U-09 _processed_trade_ids提前写入**
- 文件: `strategy_business_layer.py` `update_position_from_trade`
- 问题: `_processed_trade_ids`写入在风控检查后，并发场景下两个线程可能同时通过dedup检查
- 修复: 写入前移到风控检查之前，风控阻断/循环依赖拒绝时回滚

**U-10 重试耗尽升级处理**
- 文件: `position_command_service.py` `_schedule_close_retry`
- 问题: `_BoundedRetry`重试耗尽时仅warn级别日志
- 修复: 升级为error级别，保留close_method用于排查

**U-11 回滚重置close_method**
- 文件: `position_command_service.py` `_rollback_position`
- 问题: 回滚时重置_closing和closing_order_id但未重置close_method
- 修复: 添加`rec.close_method = ''`

#### 10.2.4 读取唯一性（3个）

**R-10 检查移出global_lock**
- 文件: `position_check_service.py` `check_all_positions`
- 问题: `_check_time_stop`/`_check_two_stage_stop`在global_lock内调用，可能触发`_trigger_close_position`(需instrument_lock)，形成死锁
- 修复: 在global_lock内收集record快照，在global_lock外执行检查

**R-11 使用instrument_lock替代global_lock**
- 文件: `position_check_service.py` `_validate_pnl_equity_consistency`
- 问题: 遍历positions使用global_lock，长时间持有阻塞其他操作
- 修复: 改为逐合约`_get_instrument_lock`

**R-12 恢复时重置close_method**
- 文件: `position_persistence.py` `_restore_position_state`
- 问题: 重启恢复时重置_closing和closing_order_id但未重置close_method
- 修复: 添加`rec.close_method = ''`
