# P2级问题修复说明文档

## 1. 缓存序列化格式混用说明 (SER-P2-26)

### 各种缓存格式适用场景

#### JSON格式
- **适用场景**: 配置文件、元数据、跨语言交互
- **优点**: 可读性强、通用性好、支持嵌套结构
- **缺点**: 文件较大、序列化速度中等
- **推荐用途**: 
  - 策略配置文件 (.json)
  - 参数池配置
  - 回测结果元数据
  - API响应数据

#### Parquet格式
- **适用场景**: 大规模时序数据、列式分析
- **优点**: 压缩率高、列式查询快、支持schema evolution
- **缺点**: 不可读、需要专门工具查看
- **推荐用途**:
  - K线历史数据
  - Tick级别数据存储
  - 回测明细数据
  - DuckDB查询结果缓存

#### CSV格式
- **适用场景**: 小规模表格数据、数据导出
- **优点**: 通用性最强、Excel可直接打开
- **缺点**: 无类型信息、不支持嵌套、文件较大
- **推荐用途**:
  - 数据导出给分析师
  - 简单参数表
  - 临时调试数据

#### JSONL格式
- **适用场景**: 流式日志、增量追加数据
- **优点**: 支持逐行追加、单行解析、便于日志轮转
- **缺点**: 整体不可读、需要逐行处理
- **推荐用途**:
  - 事件日志 (WAL)
  - 交易流水日志
  - 策略信号日志
  - 实时数据管道

#### Pickle格式
- **适用场景**: Python对象缓存、临时中间结果
- **优点**: 序列化速度最快、支持任意Python对象
- **缺点**: 安全风险(RCE)、不可跨语言、版本兼容差
- **推荐用途**:
  - 临时缓存对象
  - 回测中间结果
  - **禁止**用于网络传输或持久化存储

### 选择决策树
```
需要存储数据?
├─ 是否需要人类可读?
│  ├─ 是 → 数据量小? → CSV
│  │        └─ 数据量大/有嵌套? → JSON
│  └─ 否 → 是否需要列式分析? → Parquet
│           └─ 是否需要追加写入? → JSONL
│                └─ 临时Python对象? → Pickle (仅可信环境)
```

## 2. 延迟导入技术债务说明 (P2项8)

### 延迟导入的原因

#### 循环依赖解耦
```python
# ❌ 直接导入会导致循环依赖
from ali2026v3_trading.position_service import PositionService

# ✅ 延迟导入避免循环依赖
def get_position_service():
    from ali2026v3_trading.position_service import PositionService
    return PositionService()
```

#### 启动性能优化
```python
# ❌ 启动时加载所有依赖
import heavy_ml_library  # 阻塞启动

# ✅ 使用时才加载
def predict():
    import heavy_ml_library  # 仅在需要时加载
    return heavy_ml_library.predict()
```

#### 可选依赖处理
```python
# ✅ 可选依赖优雅降级
try:
    import duckdb
    HAS_DUCKDB = True
except ImportError:
    HAS_DUCKDB = False
    logging.warning("DuckDB未安装，部分功能降级")
```

### 延迟导入注意事项

1. **避免在热路径中使用**
   ```python
   # ❌ 高频调用路径中延迟导入会影响性能
   def on_tick(tick):
       from strategy import Strategy  # 每次tick都导入！
       return Strategy().process(tick)
   
   # ✅ 在模块级别或初始化时导入
   from strategy import Strategy  # 一次导入
   
   def on_tick(tick):
       return Strategy().process(tick)
   ```

2. **提供清晰的导入文档**
   ```python
   # 延迟导入: duckdb - 用于列式数据存储，可选依赖
   # 原因: 不是所有环境都安装DuckDB
   # 影响: 未安装时自动降级为Pandas处理
   try:
       import duckdb
       HAS_DUCKDB = True
   except ImportError:
       HAS_DUCKDB = False
   ```

3. **使用类型提示的 `TYPE_CHECKING` 技巧**
   ```python
   from typing import TYPE_CHECKING
   
   if TYPE_CHECKING:
       from ali2026v3_trading.position_service import PositionService
   
   def process(service: "PositionService") -> None:
       from ali2026v3_trading.position_service import PositionService
       # 实际实现
   ```

4. **避免过度使用**
   - 仅在必要时使用（循环依赖、可选依赖、启动优化）
   - 过度使用会降低代码可读性和IDE自动补全效果

## 3. 导入别名规范说明 (P2项13)

### 推荐别名规范

#### 数值计算库
```python
import numpy as np           # ✅ 标准别名
import pandas as pd          # ✅ 标准别名
import scipy.stats as stats  # ✅ 标准别名
```

#### 可视化库
```python
import matplotlib.pyplot as plt  # ✅ 标准别名
import seaborn as sns            # ✅ 标准别名
```

#### 类型提示
```python
from typing import (
    Dict, List, Optional, Any,  # ✅ 常用类型
    Callable, Tuple, Union,     # ✅ 常用类型
)
```

#### 项目内部模块
```python
# ✅ 保持完整路径，不使用别名
from ali2026v3_trading.shared_utils import CHINA_TZ
from ali2026v3_trading.exceptions import StrategyError

# ❌ 避免使用模糊别名
from ali2026v3_trading import s  # 太短，不清晰
from ali2026v3_trading.strategy import st  # 不清晰
```

### 别名使用原则

1. **行业标准别名优先**
   - numpy → np
   - pandas → pd
   - matplotlib.pyplot → plt

2. **避免单字母别名**
   ```python
   # ❌ 不推荐
   import numpy as n
   import pandas as p
   
   # ✅ 使用标准别名
   import numpy as np
   import pandas as pd
   ```

3. **内部模块不使用别名**
   - 项目内部模块保持完整名称，便于IDE跳转和代码搜索

4. **别名一致性**
   - 同一项目中同一库使用相同别名
   - 在 `__init__.py` 中统一re-export常用符号
