# 资金规模感知量化研究 — 策略模块缺失核查报告

**版本**: v1.5-audit  
**日期**: 2026-05-16  
**系统**: ali2026v3_trading  
**核查范围**: 策略原模块系统（strategy_ecosystem, box_spring_strategy, box_detector, state_param_manager, signal_service, order_service, risk_service, position_service, strategy_tick_handler, hft_enhancements）

## 一、核心结论

**原报告（`资金规模感知量化研究_三系统落地与对齐方案_v1.5.md`）存在根本性缺陷。**

从"夏普导向"到"盈亏比导向"的转换，**绝不仅仅是评判引擎换一个公式**。这是一场贯穿整个策略执行链路的系统性重构。原报告只完成了"评判层"的公式替换，但**策略执行层（入场、仓位、止盈止损、风控、资金分配）全部仍停留在夏普思维**。

用一个比喻：原报告相当于给赛车换了一个速度表（从显示"平均速度"换成显示"最高时速"），但发动机、变速箱、悬挂、轮胎全部没有调整——这辆车仍然按原来的方式跑，只是仪表盘显示的数字变了。

```python
    wins = [t for t in trades if t.net_pnl > 0]
    losses = [t for t in trades if t.net_pnl < 0]
    avg_win = sum(t.net_pnl for t in wins) / len(wins) if wins else 0
    avg_loss = abs(sum(t.net_pnl for t in losses) / len(losses)) if losses else 0

    total_win = sum(t.net_pnl for t in trades if t.net_pnl > 0)
    total_loss = abs(sum(t.net_pnl for t in trades if t.net_pnl < 0))
    return total_win / total_loss if total_loss > 0 else 0
```

```python

def calculate_potential_plr(self, box: BoxRange, current_price: float, direction: str) -> float:
    """计算该箱体在该方向上的潜在盈亏比"""
    box_height = box.top - box.bottom

        stop = box.bottom

        stop = box.top
```

### 3.6 order_service.py — 订单服务

**当前状态**: 手数计算基于固定风险百分比，无凯利公式/盈亏比优化。

#### 严重问题

**P1-1: 仓位计算仍为固定百分比**

```python
# 当前逻辑（隐含）：
# lot_size = equity * risk_per_trade / (stop_loss_distance * contract_value)
# risk_per_trade = 0.02 (固定2%)

# 应增加：
def calculate_kelly_position_size(
    self, equity: float, win_rate: float, avg_win: float, avg_loss: float
) -> int:
    """凯利公式仓位计算"""
    kelly_fraction = (win_rate * avg_win - (1 - win_rate) * avg_loss) / avg_win
    conservative_fraction = kelly_fraction * 0.3  # 保守系数
    return int(equity * conservative_fraction / avg_loss)

def adjust_lot_by_actual_plr(
    self, target_plr: float, actual_plr: float, current_lot: int
) -> int:
    """根据实际盈亏比动态调整手数"""
    if actual_plr < target_plr * 0.5:
        return int(current_lot * 0.5)  # 减半
    elif actual_plr > target_plr * 1.5:
        return int(current_lot * 1.2)  # 增加20%
    return current_lot
```

```python
                consecutive += 1
            else:
                break
        if consecutive >= self._consecutive_loss_limit:

        return RiskCheckResult.PASS
```

## 九、最终结论

**原报告是一个"评判层补丁"，而非"系统性重构"。**

从夏普到盈亏比的转换，如果只在评判层做公式替换，而策略执行层全部不变，结果是：
- 回测时弹簧策略得分很高（评判层认可）
- 但实盘时弹簧策略仍按固定5倍止盈/95%亏损容忍执行（执行层未变）
- 资金分配仍按固定比例，未向高盈亏比策略倾斜
- 风控仍是统一回撤阈值，未区分"正常PLR回撤"和"异常回撤"
- 信号仍是"强度>阈值即通过"，未过滤低盈亏比信号

**这就像给F1赛车换了一个显示最高时速的仪表盘，但发动机转速限制、变速箱齿比、悬挂硬度、轮胎配方全部没有调整——车还是按原来的方式跑。**

真正的"从夏普到盈亏比"转换，需要：
1. **入场前**：预估盈亏比，不达标拒绝入场
2. **仓位时**：凯利公式，赢时重仓输时轻仓
3. **持仓中**：动态止盈止损，根据实际盈亏比调整
4. **风控时**：连亏保护 + 非对称风控
5. **资金分配**：向高盈亏比策略倾斜
6. **信号过滤**：低盈亏比信号直接丢弃
7. **状态切换**：盈亏比下降时强制切换
8. **评判时**：用盈亏比公式评分

原报告只做了第8点。第1-7点全部缺失。
