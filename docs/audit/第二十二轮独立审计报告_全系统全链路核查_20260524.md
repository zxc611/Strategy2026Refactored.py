## 六、新发现的关键风险模式

### 6.1 "静默通过"模式（错误传播维度独有）

系统存在一种危险的异常处理模式：**风控/安全关键路径的异常被 `except Exception: pass` 或等效模式完全静默**。R22发现的6个P0中，4个属于此模式：

```
风险检查 → 异常 → except: pass → 返回"通过" → 交易继续
```

这构成了R16发现的"三系统独立实现"之后的**第四层根因**：即使风控代码存在且被调用，异常路径也会让风控完全失效。

### 6.2 "时间假阳"模式（时间处理维度独有）

全系统 `datetime.now()` 无时区标注。在开发环境（Windows CST时区）下一切正常，但部署到UTC时区的Linux服务器后：
- 所有日切逻辑偏移8小时
- 期权到期日计算少0.33天
- 仓位ID包含错误时间戳

这是典型的"开发环境假阳性"——本地回测通过但生产部署时系统性崩溃。

### 6.3 "创建即遗忘"模式（资源生命周期维度独有）

线程池/Timer在创建后从未被shutdown/cancel。R21的内存维度关注的是对象引用，而R22的资源维度揭示了更严重的问题：**活跃线程和定时器在系统重启/策略切换时继续运行，与新实例产生竞争**。
