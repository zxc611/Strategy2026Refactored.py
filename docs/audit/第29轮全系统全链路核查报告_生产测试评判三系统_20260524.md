## 四、跨系统架构级发现

### 发现1：两套独立运行系统

```
ProductionQuantSystem.py ──(time.sleep轮询)──▶ quant_core + quant_infra + quant_platform + quant_services
                                              └─ HMM/趋势/波动率/IV曲面/协整/生存分析
                                              └─ 结果仅存储到 SQLite，永不喂入策略决策链路

Strategy2026 ──(onTick回调)──▶ StrategyCoreService ──▶ strategy_tick_handler ──▶ HFT/position/risk/...
                                                       └─ 使用自己的 state_param_manager 的状态分类
```

**根本问题**：ProductionQuantSystem 的高级分析（HMM状态、波动率区间、IV曲面PCA、协整扫描）是完全独立的分析系统，其结果从不流入 Strategy2026 的生产策略决策链路。

### 发现2：EventBus 架构完备但零激活

`event_bus.py` 实现了完整的发布-订阅事件系统（线程安全/背压/速率限制/持久化/线程池），但：
- 零处生产代码调用 `EventBus.subscribe()`
- 跨模块通信仍为直接函数调用
- 缺少事件驱动的松耦合优势

### 发现3：治理系统三层分裂

| 层级 | 文件 | 状态 |
|:---|:---|:---|
| 定义层 | `governance_engine.py` → E7-E13/WF6-WF10 checker 类 | ✅ 已定义 |
| 实例层 | `strategy_ecosystem.py:361-367` → 创建 checker 实例 | ⚠️ 创建但从不调用 check_all() |

### 隐患修复明细

#### 隐患1: P0-5修复引入安全绕过（P1级）

**问题**: R30修复将新交易日的`_daily_hard_stop_triggered`和`_daily_new_open_blocked`都重置为False。这消除了原状态分裂问题，但引入了更严重的安全隐患：硬停止后进入新交易日，系统自动恢复交易，完全绕过了`confirm_daily_resume()`的审批流程（令牌验证+approver_id+市场安全验证）。

**修复**: 新交易日不自动重置硬停止标志。当`_daily_hard_stop_triggered=True`时，`_daily_new_open_blocked`也保持True，需通过`confirm_daily_resume()`人工确认才能解除。仅当未触发硬停止时，`_daily_new_open_blocked`才在新交易日重置为False。

**修复代码**: risk_service.py:4586-4600
```python
if self._daily_hard_stop_triggered:
    # R32-P0-05-v2修复: 新交易日不自动重置硬停止标志
    self._daily_new_open_blocked = True
    logging.warning(
        "[SafetyMetaLayer] 新交易日(%s)，日回撤硬停止仍然生效。"
        "必须调用confirm_daily_resume()经审批后才能恢复交易。",
        today,
    )
else:
    self._daily_new_open_blocked = False
```

#### 隐患3: onTick PQS节流缺失（P2级）

**问题**: R30-ARCH-02修复在每个tick回调中都调用`_pqs.update_tick()`+`_pqs.publish_analysis_to_strategy()`。期货高频场景下tick频率可达每秒数十个，PQS内部执行trend_scorer/vol_filter/hmm/crm等计算，每次调用都有CPU开销，且每次发布最多4个EventBus事件。

**修复**: 添加tick节流机制，默认每5秒或每10个tick才调用一次PQS更新。

**修复代码**: strategy_core_service.py:2217-2242
```python
_now_pqs = time.time()
_pqs_tick_count = getattr(self, '_pqs_tick_count', 0) + 1
self._pqs_tick_count = _pqs_tick_count
_pqs_last_time = getattr(self, '_pqs_last_update_time', 0.0)
_pqs_throttle_sec = 5.0
_pqs_throttle_ticks = 10
if (_now_pqs - _pqs_last_time) >= _pqs_throttle_sec or _pqs_tick_count >= _pqs_throttle_ticks:
    # ... PQS update logic ...
    self._pqs_last_update_time = _now_pqs
    self._pqs_tick_count = 0
```
