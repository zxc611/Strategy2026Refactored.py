## 一、批判式吸收摘要

### 1.1 对第十四章至第十九章的批判性吸收结论

原报告第十五章至第十九章对"顶级基金视角"差异化排序方案进行了多轮批判与自我批判。经逐条审阅，提炼出以下**可保留的核心思想**与**必须规避的系统性缺陷**：

| 来源章节 | 可保留思想（精华） | 必须规避的缺陷（毒素） |
|---------|------------------|---------------------|
| 第十四章 | 品种差异化处理的方向性直觉；风险预算驱动的仓位分配动机 | 主观权重分配（0.35/0.20等拍脑袋数值）；乘积评分导致分数过度压缩；风险预算固定每类3个标的过于粗糙；Sharpe>1.5的激进验收标准 |
| 第十五章 | 渐进改进路径（短期参数化→中期衰减→长期数据驱动）；明确否定在趋势跟随框架中混入均值回归权重 | — |

| 第十七章 | 品种特征数据库；向量化回测引擎；Walk-Forward滚动窗口验证；贝叶斯优化（Optuna）；Calmar比率目标函数；参数漂移约束 | 复杂度过高可能导致分析瘫痪；需防范回测与实盘gap |
| 第十八章 | 共振倍数（resonance）作为监控指标；市场平均正确率作为仓位调节因子 | MRR非真正"零参数"；相对标准在全市场偏弱时存在致命缺陷（33%正确率品种被选中）；丢弃Wilson/Coverage/Noise_ratio三个关键风控维度 |

### 1.2 本方案遵循的六大架构原则

1. **数据驱动，拒绝拍脑袋**：任何参数必须经过10年历史数据（2015-2025）回测验证，不接受主观赋值。
2. **统计严谨，兼容现有框架**：保留现有 `correct_up_pct` / `noise_ratio` / `coverage` / `wilson` / `Tier` 的数学结构，不做破坏性变更。
3. **趋势跟随，不混均值回归**：`wrong_fall` / `wrong_rise` 在排序中仅作为噪声惩罚项，不赋正权重。若需均值回归策略，另建独立服务。
4. **全序保留，不偏序破坏**：排序输出维持 `(tier升序, score降序, wilson降序)` 的全序关系，不引入接口契约级别的破坏性变更。
5. **非对称衰减，替代主观权重**：通过让错误状态更快衰减，自然降低其对评分的污染，而非在评分阶段给错误状态赋正权重。
6. **决策触发器，避免分析瘫痪**：建立基于回测结果的决策树，到达触发条件即执行，不允许无限递归的讨论。

**澄清3：MRR正确定位**

## 三、Layer 1：单品种月份层排序

### 3.1 输入定义

对单一期货品种（如 `cu`、`IF`），采集其下挂所有期权的五态统计数据。交易月份覆盖：**当月 + 下月 + 三个季月**，最多5个月份。

| 输入字段 | 符号 | 说明 |
|---------|------|------|
| 月份权重 | `w_m` | `m ∈ {0,1,2,3,4}` 对应当月到下月再到三季月 |
| 五态原始计数 | `CR_m, WR_m, CF_m, WF_m, Other_m` | 各月份下的五态计数 |
| 最后更新时间戳 | `t_last` | 该状态最近一次出现的Unix时间戳 |
| Greeks | `Delta, Gamma, Vega` | 用于微观结构修正 |
| 期权成交量 | `V_opt` | 用于流动性过滤 |

**月份权重配置**（与现有框架兼容）：
```python
MONTH_WEIGHTS_5 = (0.35, 0.25, 0.20, 0.12, 0.08)  # 当月→下月→季月1→季月2→季月3
```

### 3.2 非对称状态时效衰减

原报告第十五章提出状态衰减，第十六章进一步提出**非对称衰减**：正确状态趋势可能持续，衰减慢；错误状态（假突破）快速失效，衰减快。

**衰减公式**：
```
count_eff = count × exp(-λ_status × (t_now - t_last))
```

**半衰期配置**（按行为簇，非单个品种）：

| 状态 | 行为簇：金融期货 | 行为簇：黑色系/有色 | 行为簇：农产品 | 行为簇：能源化工 |
|------|---------------|------------------|-------------|---------------|
| CR (correct_rise) | 600s | 300s | 180s | 240s |
| CF (correct_fall) | 600s | 300s | 180s | 240s |
| WR (wrong_rise) | 120s | 60s | 60s | 60s |
| WF (wrong_fall) | 120s | 60s | 60s | 60s |

**金融工程含义**：
- 金融期货（IF/IC/IM/IH）tick噪声小、趋势持续性强，正确状态半衰期10分钟。
- 农产品季节性波动大、假突破多，错误状态半衰期仅60秒，快速出清污染。
- 若衰减效果不佳，可将半衰期设为极大值（86400s），等价于关闭衰减，具备回滚能力。

### 3.3 Greeks微观结构修正

原报告第十六章指出，第十四章完全缺失期权Greeks，导致深度实值/虚值期权的统计噪声被等同处理。Layer 1引入三项Greeks修正：

**3.3.1 Delta范围过滤（硬门槛）**

仅保留 `Delta ∈ [0.15, 0.85]` 的期权参与五态统计。深度实值（Delta>0.85）和深度虚值（Delta<0.15）的期权，其价格变动主要由时间价值或流动性噪声驱动，而非方向判断。

```python
def _delta_filter(options: List[Dict]) -> List[Dict]:
    return [o for o in options if 0.15 <= abs(o['delta']) <= 0.85]
```

**3.3.2 Gamma时效权重（软修正）**

近月平值期权（Gamma > 0.05）的状态翻转速度快，其CR/CF信号更具时效性。对满足 `Gamma > 0.05` 且 `|Delta - 0.5| < 0.15` 的期权，其CR/CF计数在计算 `correct_up_pct` 时给予 **1.2倍权重**。

```python
gamma_boost = 1.2 if (gamma > 0.05 and abs(delta - 0.5) < 0.15) else 1.0
cr_weighted = cr_count * gamma_boost
```

**3.3.3 Vega波动率归因惩罚**

若某品种当日Vega > 0.10（高IV敏感度），且隐含波动率IV在5分钟内变化超过5%，则该时段内的五态分类可能被IV变动主导而非方向判断主导。此类状态计数在衰减前乘以 **0.8倍惩罚系数**。

```python
vega_penalty = 0.8 if (vega > 0.10 and iv_change_5min > 0.05) else 1.0
count_eff = count * exp(-λ * Δt) * vega_penalty
```

**3.3.4 Theta时间价值损耗惩罚**

近月虚值期权Theta衰减极快，每日时间价值损耗可达期权价格的5-10%。一个被分类为`correct_rise`的期权，可能因为Theta损耗导致实际收益为负。五态分类只区分方向正确与否，不区分时间价值的影响。

对近月（`m=0,1`）且 `Gamma > 0.03` 的期权，在计算 `correct_up_pct` 时施加Theta惩罚：

```python
theta_penalty = 1.0 - min(0.15, abs(theta) / option_price * 10)  # 最高15%惩罚
# 例：theta/price = 0.01（日损耗1%）→ penalty = 0.90；theta/price = 0.005 → penalty = 0.95
cr_theta_adjusted = cr_count * theta_penalty
cf_theta_adjusted = cf_count * theta_penalty
```

**金融工程含义**：Theta惩罚仅作用于近月期权（时间价值占比高），远月期权（`m≥2`）的Theta影响较小，不施加惩罚。这防止了"方向正确但时间价值损耗吞噬利润"的伪信号。

### 3.4 单品种核心指标计算

在衰减与Greeks修正后，计算Layer 1核心指标：

```
correct_up_pct_1 = Σ_m [ w_m × (CR_m_eff + CF_m_eff) ] / Σ_m [ w_m × Total_m_eff ]

noise_ratio_1    = Σ_m [ w_m × (WR_m_eff + WF_m_eff) ] / Σ_m [ w_m × Total_m_eff ]

coverage_1       = (有非零计数的月份数) / 5

wilson_1         = Wilson score interval lower bound for correct_up_pct_1
```

**Wilson校正公式**（保留现有框架）：
```
wilson = (p̂ + z²/(2n) - z × √[p̂(1-p̂)/n + z²/(4n²)]) / (1 + z²/n)
```
其中 `p̂ = correct_up_pct_1`, `n = Σ Total_m_eff`, `z = 1.96`（95%置信度）。

### 3.5 行为簇与差异化Tier分级

原报告第十六章提出"按品种分组（group）配置阈值"，本方案进一步将品种按**历史五态分布相似度**自动聚类为行为簇（behavior cluster），每个行为簇独立配置Tier阈值。

**3.5.1 行为簇定义**（基于KL散度聚类，非主观分类）

| 行为簇 | 品种成员 | 簇特征 |
|--------|---------|--------|
| 金融期货簇 | IF, IC, IM, IH | 低波、高流动性、趋势持续性强 |
| 黑色系簇 | i, j, jm, rb, hc | 高相关性、政策敏感、波动率聚类 |
| 有色金属簇 | cu, al, zn, ni, sn | 外盘联动强、夜盘影响大、Gamma稳定 |
| 农产品簇 | m, c, a, y, p, rm, oi | 季节性显著、假突破多、IVskew大 |
| 能源化工簇 | sc, ta, eg, l, pp, v, bu | 成本驱动、产业链传导、波动率均值回归 |

**聚类方法**：**每季度初**（而非每月）计算各品种近60日（而非20日）五态分布（CR/WR/CF/WF/Other占比），用Jensen-Shannon距离度量分布相似度，采用**Gaussian Mixture Model（GMM，软聚类）**替代K-means硬聚类。聚类结果写入 `behavior_cluster_map.json`。

**软聚类设计**：
- GMM允许品种以概率形式同时属于多个簇（如某品种60%属于有色金属簇、40%属于能源化工簇）
- 归属概率最高的簇作为主簇，决定Tier阈值和衰减半衰期
- 归属概率低于0.60的品种，使用**全局默认阈值**（最保守配置），避免硬切换导致的参数跳变
- 聚类数量K不由主观设定，而用**BIC（Bayesian Information Criterion）自动选择**（通常K=4~6），消除"K=5拍脑袋"问题

**3.5.2 各簇Tier阈值配置**

默认阈值（全簇统一）：
```python
TIER1_WILSON_THRESHOLD = 0.60
TIER2_COVERAGE_THRESHOLD = 0.50
TIER2_CORRECT_UP_THRESHOLD = 0.50
TIER3_CORRECT_UP_THRESHOLD = 0.40
```

**金融期货簇差异化**（因tick噪声小、样本量大，Wilson门槛可适度放宽）：
```python
FINANCIAL_CLUSTER = {
    'TIER1_WILSON_THRESHOLD': 0.55,      # ↓ 0.05
    'TIER2_COVERAGE_THRESHOLD': 0.50,    # 不变
    'TIER2_CORRECT_UP_THRESHOLD': 0.52,  # ↑ 0.02
    'TIER3_CORRECT_UP_THRESHOLD': 0.42,  # ↑ 0.02
}
```

**农产品簇差异化**（因假突破多、噪声大，提高Coverage门槛以过滤数据稀疏月份）：
```python
AGRICULTURAL_CLUSTER = {
    'TIER1_WILSON_THRESHOLD': 0.65,      # ↑ 0.05
    'TIER2_COVERAGE_THRESHOLD': 0.60,    # ↑ 0.10
    'TIER2_CORRECT_UP_THRESHOLD': 0.48,  # ↓ 0.02
    'TIER3_CORRECT_UP_THRESHOLD': 0.38,  # ↓ 0.02
}
```

> **重要**：上述差异化阈值仅为初始值，后续必须通过第十七章的贝叶斯优化流程在10年历史数据上校准。初始值的作用是允许系统在无历史校准数据时启动，但应在1个月内被数据驱动值替代。

**3.5.3 Tier分级逻辑**

```python
def determine_tier_layer1(correct_up_pct, coverage, wilson, cluster_thresholds):
    t = cluster_thresholds
    
    if wilson >= t['TIER1_WILSON_THRESHOLD'] and coverage >= 0.8:
        return 1  # 高置信
    elif coverage >= t['TIER2_COVERAGE_THRESHOLD'] and correct_up_pct >= t['TIER2_CORRECT_UP_THRESHOLD']:
        return 2  # 中置信
    elif correct_up_pct >= t['TIER3_CORRECT_UP_THRESHOLD']:
        return 3  # 低置信
    else:
        return 4  # 噪声，过滤
```

### 3.6 Layer 1 输出

| 输出字段 | 说明 |
|---------|------|
| `score_1` | `correct_up_pct_1` 的Wilson修正值，即 `wilson_1` |
| `tier` | 1/2/3/4，Tier 4直接过滤，不进入Layer 2 |
| `coverage_1` | 有效月份占比 |
| `noise_ratio_1` | 噪声占比，用于Layer 2的簇离散度计算 |
| `month_structure_score` | 近月集中度 = (CR_0 + CF_0) / Σ(CR_m + CF_m)，衡量信号是否集中在主力月份 |

## 四、Layer 2：联动品种综合层排序

### 4.1 核心思想

产业链存在基本面联动（如原油→化工、铁矿石→螺纹钢）。若某品种在Layer 1得分高，但其同簇品种全部走弱，则该品种的强势可能是孤立的噪音；若同簇品种同步强势，则可能是产业链共振，信号更可靠。

### 4.2 簇内共振强度计算

对行为簇 `C`（如黑色系簇），设簇内有 `N_C` 个品种：

```
cluster_resonance(C) = Σ_{i∈C} [ score_1(i) × weight_i ] / N_C
```

其中 `weight_i` 为品种 `i` 在该簇内的**流通市值权重**（非主观等权），反映品种在簇内的经济重要性。

**簇离散度**（用于识别簇内信号分裂）：
```
cluster_dispersion(C) = std( [score_1(i) for i in C] )
```

若 `cluster_dispersion(C) > 0.15`（阈值经回测校准），认为簇内信号不一致，对该簇所有品种的Layer 2得分施加 **0.90倍惩罚**。

### 4.3 簇内相关性惩罚

若簇内两品种60日收益率相关系数 `ρ(i,j) > 0.80`，同时选入时给予 **0.85倍惩罚**，防止伪分散化。

```python
def correlation_penalty(selected_in_cluster, corr_matrix):
    penalty = 1.0
    for i, j in combinations(selected_in_cluster, 2):
        if corr_matrix[i][j] > 0.80:
            penalty *= 0.85
    return penalty
```

### 4.4 簇五态转移矩阵与稳态评估

计算簇内各品种近20日的五态转移概率矩阵（马尔可夫链），提取稳态分布：

```
steady_state(CR) = 长期趋势中处于CR状态的概率
```

若 `steady_state(CR) + steady_state(CF) > 0.60`，认为该簇处于**强趋势态**；若 `< 0.40`，认为处于**震荡态**。震荡态下，簇内所有品种的Layer 2得分乘以 **0.80倍**（趋势策略在震荡市应降权）。

### 4.5 Layer 2 综合得分

为避免纯乘法导致信号过度压缩，Layer 2采用**加权混合模型**（加法为主，乘法惩罚为辅）：

```
score_2(i) = 0.70 × score_1(i)                    # 基础Layer 1得分（保留主体）
           + 0.20 × f_cluster_resonance(C)         # 簇共振增强（加法，避免压缩）
           + 0.10 × f_regime(C)                    # 市态调节（加法，趋势/震荡）
           + penalty_dispersion                    # 离散度惩罚（阈值触发，非连续）
           + penalty_correlation                   # 相关性惩罚（阈值触发，非连续）
```

**惩罚因子的触发条件**（非连续乘法，改为阈值触发）：
- `penalty_dispersion = -0.05`（仅当 `cluster_dispersion(C) > 0.15` 时触发一次）
- `penalty_correlation = -0.05`（仅当存在 `ρ(i,j) > 0.80` 且同时选入时触发一次）

**下限保护**：`score_2(i) ≥ 0.60 × score_1(i)`，确保Layer 2不会将优质品种压到噪声水平。

**设计原理**：
- 主体评分由**加法**构成（0.70+0.20+0.10=1.00），避免多层乘法连乘导致的信号死亡
- 惩罚项仅在极端条件下触发（-0.05），与原报告第十八章批判的"乘积效用函数"（`score = weight × decay × liquidity × coverage`）划清界限

### 4.6 Layer 2 输出

| 输出字段 | 说明 |
|---------|------|
| `score_2` | 经簇共振、离散度、相关性、市态修正后的综合得分 |
| `cluster_rank` | 该品种在所属簇内的排名 |
| `cluster_state` | 簇状态标签：`strong_trend` / `normal` / `choppy` / `reversal_risk` |
| `correlation_penalty_flag` | 是否与簇内其他选中品种高相关 |

## 五、Layer 3：全品种综合排序层

### 5.1 核心思想

Layer 3回答最终问题：在全市场所有品种中，哪些应该交易、各配多少仓位。本层引入**市场基准状态**和**绝对标准硬门槛**，同时保留MRR的"共振"思想作为**辅助增强因子**而非核心标准。

### 5.2 市场基准状态计算

```
market_accuracy = Σ(全市场CR_eff + CF_eff) / Σ(全市场Total_eff)

market_bias = Σ(全市场CR_eff + WR_eff) - Σ(全市场CF_eff + WF_eff)
```

- `market_accuracy ∈ [0, 1]`：全市场整体趋势判断正确率
- `market_bias ∈ [-1, 1]`：全市场多空偏向（正值看多，负值看空）

### 5.3 全局共振评分

为避免纯乘法信号压缩，Layer 3采用**加权混合模型**：

```
score_3(i) = 0.65 × score_2(i)                    # Layer 2基础得分
           + 0.20 × f_resonance(i)                 # 市场共振增强（加法）
           + 0.15 × f_liquidity(i)                 # 流动性调节（加法）
           + penalty_budget                         # 风险预算惩罚（阈值触发）
```

**下限保护**：`score_3(i) ≥ 0.55 × score_2(i)`，防止三层传递后信号死亡。

**5.3.1 共振因子 `f_resonance(i)`**

```
f_resonance(i) = accuracy(i) / market_accuracy    if market_accuracy >= market_accuracy_floor
                 0.0                               if market_accuracy < market_accuracy_floor
```

**关键改进**：`market_accuracy_floor` 不再使用固定值 **0.35**（被批判为"拍脑袋"），而是基于**10年历史数据自适应计算**：

```python
# 每月初根据历史分位数动态更新
market_accuracy_floor = percentile(history_market_accuracy_10y, 20)  # 历史20%分位
# 例：若10年数据中 market_accuracy 的20%分位为0.32，则门槛为0.32
# 若20%分位为0.38，则门槛为0.38
```

**金融工程含义**：门槛随市场统计特性自适应。在交易所制度变化（如保证金调整、涨跌停幅度变化）导致市场整体正确率水平位移时，保护机制自动调整，而非死守固定阈值。

**绝对标准保护**：即使某品种相对市场"较好"，在全市场整体正确率低于历史20%分位时，`f_resonance` 强制为0，该品种不进入交易列表。这直接规避了原报告第十九章批判的MRR致命缺陷——"全市场温和偏弱时选中33%正确率品种"。

**5.3.2 流动性因子 `f_liquidity(i)`**

仅用成交量衡量流动性过于简化，忽略买卖价差和订单深度。引入**综合流动性评分**：

```python
f_liquidity(i) = 0.40 × volume_norm + 0.30 × spread_norm + 0.30 × depth_norm

# volume_norm  = min(1.0, V_opt_30min / 500)
# spread_norm  = max(0.0, 1.0 - bid_ask_spread_pct / 0.02)  # 价差2%时归一化为0
# depth_norm   = min(1.0, order_book_depth_5level / 100)   # 5档深度100手时满额
```

> 注：使用30分钟滚动成交量而非日累计，避免开盘/收盘时段的流动性骤变（原报告第十五章已指出日级阈值的问题）。

**5.3.3 动态风险预算因子 `f_budget(i)`**

风险预算基于各行为簇近期20日ATR（Average True Range）动态分配：

```
cluster_risk_budget(C) = (1 / ATR_20d(C)) / Σ_C'(1 / ATR_20d(C'))
```

高波动簇分配**更少**的风险预算（因单笔交易风险大），低波动簇分配**更多**风险预算（可分散化）。

单品种风险预算上限：
- 单品种 ≤ 总权益的 **15%**
- 单行为簇 ≤ 总权益的 **40%**

### 5.4 全市场仓位联动

基于 `market_accuracy` 调节整体仓位上限（保留第十八章的思想，但引入**历史分位数自适应分档**，消除固定阈值的武断性）：

```python
# 每月初根据10年历史数据计算分位数
p80 = percentile(history_market_accuracy_10y, 80)  # 历史80%分位
p50 = percentile(history_market_accuracy_10y, 50)  # 历史50%分位
p20 = percentile(history_market_accuracy_10y, 20)  # 历史20%分位（即market_accuracy_floor）
```

| market_accuracy 范围 | 整体仓位上限 | Tier交易门槛 | 说明 |
|---------------------|-------------|-------------|------|
| ≥ p80 | 100% | Tier 1/2/3 | 市场处于历史强势区间，满仓参与 |
| p50 - p80 | 75% | Tier 1/2/3 | 市场处于历史中上区间，适度降仓 |
| p20 - p50 | 50% | Tier 1/2 | 市场处于历史中下区间，仅中高置信标的 |
| < p20 | 0% | 禁止交易 | 市场处于历史弱势区间，空仓观望 |

**示例**：若10年历史中 `market_accuracy` 的20%/50%/80%分位分别为 `0.32/0.41/0.52`，则分档自动调整为：
- ≥ 0.52 → 100%
- 0.41 - 0.52 → 75%
- 0.32 - 0.41 → 50%
- < 0.32 → 0%

这与固定阈值（0.35/0.45/0.55）可能不同，但**更贴合实际市场统计分布**。

### 5.5 最终排序与输出

**排序键（全序关系，保留现有接口契约）**：
```python
sorted_targets = sorted(
    candidates,
    key=lambda x: (x['tier'], -x['score_3'], -x['wilson'])
)
```

**Layer 3 输出**：

| 输出字段 | 说明 |
|---------|------|
| `final_rank` | 全局排名 |
| `score_3` | 全局共振评分 |
| `tier` | 置信等级 |
| `suggested_lots` | 建议手数（基于动态风险预算与档位系数） |
| `position_limit_pct` | 该品种最大仓位占比 |
| `market_state` | 全市场状态标签：`strong_bull` / `bull` / `neutral` / `bear` / `strong_bear` |
| `risk_flags` | 风控标记列表：`high_correlation` / `choppy_cluster` / `low_liquidity` / `iv_spike` |

## 六、风控与回滚机制

### 6.1 参数漂移约束

相邻校准期间，Tier阈值的相对变化不超过 **20%**（原报告第十七章）：

```python
def apply_drift_constraint(new_val, old_val, max_drift=0.20):
    max_change = old_val * max_drift
    return max(old_val - max_change, min(old_val + max_change, new_val))
```

### 6.2 A/B测试框架

对流动性好的品种（IF、cu、al），`product_tier_thresholds.json` 支持 `experiment_group` 字段：

```json
{
  "IF": {
    "TIER1_WILSON": 0.55,
    "experiment_group": "treatment",
    "control_thresholds": {
      "TIER1_WILSON": 0.60
    }
  }
}
```

实盘同时运行对照组（默认阈值）和实验组（优化阈值），2周后对比Calmar。实验组仅分配 **20%仓位**，对照组分配 **80%仓位**，确保风险可控。

### 6.3 状态翻转预警（非自动减仓）

原报告第十五章指出，"CR→WR时自动减仓50%"会引入新的耦合点且可能过度反应。本方案改为**预警机制**：

- CR→WR翻转时，该品种 `risk_flags` 增加 `state_reversal_warning`
- 不自动减仓，但提升该品种监控频率（从分钟级提升到tick级）
- 若30分钟内连续出现2次翻转，则将该品种Tier临时降1级（如Tier 1→Tier 2）

### 6.4 流动性熔断

单品种期权过去30分钟成交量 < 100手时，该品种进入 `liquidity_watch` 状态：
- 不立即移出排序，但 `f_liquidity` 乘以0.5
- 若连续3个时间窗口（每个窗口5分钟）成交量均 < 100手，则该品种当日禁止交易

### 6.5 参数回滚触发器

| 触发条件 | 动作 |
|---------|------|
| 新参数首周Calmar为负 | 自动回滚到上期参数，回滚时间<10秒 |
| Walk-Forward测试期出现Calmar为负的窗口 | 标记该参数组合为`overfit`，不部署 |
| 单日回撤超过权益的5% | 触发全市场减仓50%，当日不再开新仓 |

## 七、验证与校准机制

### 7.1 基础假设验证（前置条件）

在部署任何排序方案前，必须先验证两个基础假设：

**测试A：五态统计的预测力验证**
```
按 correct_up_pct 分组：>0.6 / 0.5-0.6 / 0.4-0.5 / <0.4
计算每组的次日平均收益率
```
若 `>0.6` 组的收益不显著高于 `<0.4` 组（t检验p>0.05），则整个五态排序框架的经济学基础崩塌，策略需要重新设计。

**测试B：市场共振概念验证**
```
计算 market_accuracy 与次日全市场平均收益的Pearson相关系数
```
若相关性不显著（|r| < 0.10, p > 0.05），"市场整体共振"的概念需要重新定义。

### 7.2 三层独立回测验证

每层均可独立回测，验证该层的增量价值：

| 回测层级 | 验证目标 | 基准对比 |
|---------|---------|---------|
| Layer 1 | 月份权重+非对称衰减+Greeks修正是否提升单品种排序效果 | 现有框架（无衰减、无Greeks） |
| Layer 2 | 簇共振与离散度惩罚是否提升跨品种一致性 | Layer 1输出直接排序 |
| Layer 3 | 全局共振+动态预算+市场状态联动是否提升组合收益 | Layer 2输出直接排序 + MRR基线 |

### 7.3 决策触发器（终结分析瘫痪）

基于回测结果，按以下决策树执行，不允许无限讨论：

```
触发器1: MRR 10年Baseline Calmar < 0.3
  → 市场本身太嘈杂，任何排序方案都难以盈利
  → 行动：策略暂停，检查数据质量与五态分类逻辑

触发器2: MRR Baseline Calmar ≥ 0.3，但复杂方案仅比MRR高 < 0.2
  → 复杂方案增量价值不足，不值得维护成本
  → 行动：部署MRR，复杂方案降级为研究项目，3个月后重校准

触发器3: 复杂方案比MRR高 ≥ 0.2，且通过Diebold-Mariano检验（p < 0.05）
  → 复杂方案具有统计显著优势
  → 行动：部署三层排序体系，MRR作为监控基准

触发器4: 复杂方案Walk-Forward测试期出现Calmar为负的窗口
  → 复杂方案过拟合
  → 行动：回退到触发器2，收紧参数搜索空间后重新校准
```

> **Diebold-Mariano检验**：用于判断两个预测序列（复杂方案 vs MRR）的预测能力差异是否具有统计显著性，优于武断的Calmar差距阈值。

### 7.4 Walk-Forward滚动验证

```
时间轴: |----训练期(6个月)----|----验证期(2个月)----|----训练期----|...
                           ^ 贝叶斯优化搜索最优Tier阈值
                                             ^ 测试Calmar稳定性
```

**验收标准**：
- 各窗口测试期Calmar标准差 < 0.3
- 无参数导致测试期Calmar为负的极端异常
- 参数漂移 `param_drift_from_prev` < 20%

### 10.2 方案缺陷与风险

#### 10.2.1 三层乘法传递的信号衰减风险——**严重**

三层架构的核心问题是**多层乘法惩罚的累积效应**：

```
score_3 = score_1 × f_cluster_resonance × f_dispersion × f_correlation × f_regime × f_resonance × f_liquidity × f_budget
```

即使每个因子平均为0.85（已经很保守），8个因子连乘后 `0.85^8 ≈ 0.27`。这意味着一个原本评分0.6的优质品种，经过三层传递后可能降至0.16。

**原报告第十八章批判的"信号层层衰减"问题，在本方案中并未解决，只是从"8层处理"变成了"8个乘法因子"。** 当策略连续无交易时（第十三章记录的"连续6周0交易"），问题可能不是"信号不存在"，而是"信号在乘法传递中被杀死了"。

**建议**：
- 考虑**加法模型**或**加权混合模型**（如 `score = 0.6×score_1 + 0.2×f_resonance + 0.2×f_liquidity`）替代纯乘法
- 或设置**乘法因子的下限**（如所有惩罚因子不低于0.6），防止过度压缩
- 在回测中对比"纯乘法"vs"加法混合"vs"现有框架"的信号覆盖率差异

**评级：B-**。设计精巧但可能过度过滤。

#### 10.2.2 行为簇聚类稳定性——未解决的工程摩擦

方案3.5.1提出"每月初用K-means（K=5）聚类"，但存在以下问题：

1. **K=5的设定本身是主观的**——为什么是5个簇而非4个或6个？未提供肘部法则（elbow method）或轮廓系数（silhouette score）的验证结果
2. **月度重聚类导致品种归属跳变**——某品种可能上月在"黑色系簇"，本月跳到"能源化工簇"，导致Tier阈值和衰减半衰期同时变化，回测无法准确模拟
3. **聚类依赖近20日五态分布**——20天样本量不足以支撑稳定的分布估计（尤其在低成交量品种上）

**建议**：
- 改为**季度重聚类**，减少归属跳变频率
- 使用**软聚类**（soft clustering，如Gaussian Mixture Model），品种可以同时属于多个簇（带权重），避免硬切换
- 在聚类结果中增加"归属置信度"字段，置信度低的品种使用全局默认阈值

**评级：C+**。方向正确但实现细节粗糙。

#### 10.2.3 共振因子的绝对门槛——仍是拍脑袋

方案5.3.1定义：
```
f_resonance(i) = accuracy(i) / market_accuracy    if market_accuracy >= 0.35
                 0.0                               if market_accuracy < 0.35
```

`0.35`这个绝对门槛值与原报告第十九章批判的"0.3 Calmar差距阈值"本质相同——**缺乏统计检验支撑的武断阈值**。为什么是0.35而不是0.30或0.40？

虽然方案正确规避了MRR"全市场温和偏弱时选中33%品种"的缺陷，但`0.35`本身可能过于激进或过于保守，取决于历史数据中`market_accuracy`的实际分布。

**建议**：
- 用10年历史数据计算`market_accuracy`的分位数分布（如10%、25%、50%、75%、90%分位）
- 将门槛设为历史分布的**特定分位数**（如25%分位），而非固定值0.35
- 这样门槛会随市场制度变化自动调整（如交易所保证金调整后，market_accuracy的整体水平可能变化）

**评级：B**。风控方向正确，但阈值设定缺乏数据支撑。

#### 10.2.4 参数规模被低估——"仅9个参数"是幻觉

方案在1.2节声称"拒绝拍脑袋"，在9.2节风险声明中承认"引入行为簇差异化后参数扩展至20个"。但实际可调参数远超此数：

| 参数类别 | 数量 | 说明 |
|---------|------|------|
| Tier阈值（全局） | 4 | TIER1_WILSON, TIER2_COVERAGE, TIER2_CORRECT_UP, TIER3_CORRECT_UP |
| 月份权重 | 5 | MONTH_WEIGHTS_5 |
| 衰减半衰期 | 4簇×4状态=16 | 每行为簇的CR/CF/WR/WF半衰期 |
| 行为簇差异化Tier | 2簇×4阈值=8 | 金融期货簇和农产品簇的差异化阈值 |
| Greeks修正 | 4 | DELTA_RANGE(2), GAMMA_BOOST_FACTOR, VEGA_PENALTY_FACTOR |
| 簇离散度惩罚 | 2 | CLUSTER_DISPERSION_THRESHOLD, CLUSTER_DISPERSION_PENALTY |

在100万+有效状态下校准50个参数，参数-样本比约1:20000，统计学上可行，但**Optuna的搜索空间是50维而非9维**，2000 trials远远不够，可能需要10000+ trials。

**评级：B-**。诚实面对参数规模是防止过拟合的第一步。

#### 10.2.5 流动性因子设计过于简化

方案5.3.2定义：
```
f_liquidity(i) = min(1.0, V_opt_30min / 500)
```

仅用成交量衡量流动性，忽略了：
- **买卖价差**（bid-ask spread）：价差大的品种即使有成交量，滑点成本也可能吞噬利润
- **订单深度**（order book depth）：500手成交量可能由1手×500笔或500手×1笔构成，市场冲击成本完全不同
- **成交频率**：均匀分布的500手 vs 集中在开盘/收盘的500手，流动性质量差异巨大

**建议**：引入综合流动性评分 `liquidity_score = 0.4×volume_norm + 0.3×(1/spread) + 0.3×depth_norm`

**评级：C**。在期权市场（流动性差异极大），简化处理可能导致低流动性品种被高估。

#### 10.2.6 基础假设仍未验证——所有设计的前提

方案第七章设计了"测试A"和"测试B"作为基础假设验证，但**整个文档的所有设计都建立在这两个测试通过的前提上**。如果：

- 测试A失败：`correct_up_pct > 0.6`组的次日收益不显著高于`<0.4`组
- 测试B失败：`market_accuracy`与次日全市场平均收益相关性不显著

**那么整个三层架构（包括非对称衰减、Greeks修正、行为簇差异化、共振因子、动态预算）全部失去经济学基础，无论设计多么精巧。**

本方案花费了大量篇幅讨论"如何排序"，但对"为什么要这样排序"的经济学基础验证着墨不足。

**建议**：在启动任何Layer 1/2/3的实施之前，**优先执行测试A和测试B**。这两个测试只需3天（第十七章Phase A的时间），但其结果决定了整个方案是否值得投资。

**评级：B-**。基础假设验证的优先级应高于架构设计。

#### 10.2.7 实施周期过于乐观

方案8.2的7阶段实施路线图总计7周，但以下阶段的预估时间严重不足：

| 阶段 | 预估 | 实际（经验） | 差距原因 |
|------|------|-------------|---------|
| Phase 4（数据库与回测） | 2周 | 4-6周 | 10年历史数据的ETL清洗（合约换月、行权价调整、退市品种处理）耗时远超预期 |
| Phase 5（校准与验证） | 2周 | 4-8周 | 50维参数空间的Optuna搜索，2000 trials不够，需调至10000+ |
| Phase 2（Layer 2） | 1周 | 2-3周 | 马尔可夫链稳态评估+相关性矩阵计算的工程复杂度被低估 |

**实际实施周期可能需要16-24周（4-6个月）而非7周。**

**评级：C+**。路线图方向正确但时间估算不切实际。
