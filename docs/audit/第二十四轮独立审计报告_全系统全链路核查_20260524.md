## 六、新发现的关键风险模式

### 6.1 "源头污染"模式（输入验证维度独有）

```
tick到达 → 无任何NaN/Inf/负值检查 → NaN完整传播到趋势计算 → 趋势评分NaN → CRM共振模块基于NaN信号 → 开仓/平仓决策错误
```

这是新发现的**第六层根因**：即使所有状态机和控制流都正确，源头输入未验证会让整个链路输出错误结果。前轮发现了多个轮次的"回测冠军崩盘"根因，本轮揭示了源头验证的系统性缺失。

### 6.2 "默认值大战"模式（默认值维度独有）

系统存在四层默认值来源，互不统一且互相覆盖：
1. `DEFAULT_PARAM_TABLE` 代码级默认
2. YAML配置文件默认
3. 各策略类硬编码默认
4. 回退函数get()默认

其中`max_risk_ratio`在YAML和Code回退相差2.67倍，`close_stop_loss_ratio`相差67%，直接导致仓位和止损计算系统性偏差。这是典型的"配置漂移"导致回测参数与实盘代码实际参数不一致。

### 6.3 "黑箱交易"模式（审计可追溯性维度独有）

signal_id链路断裂意味着：**你无法回答一个基本问题——这笔交易到底是哪个信号触发的？** 系统完全不保存这个关联，只能看到订单，看不到信号来源。这使得任何事后分析、策略优化、bug定位都不可能，也违反了基本的监管要求。
