## 一、SortEntry 排序键计算（width_cache.py 第38-110行）

### 1.1 SortEntry 类定义

```python
@dataclass(order=True)
class SortEntry:
    sort_key: Tuple = field(compare=True)      # 排序键（唯一比较字段）
    internal_id: int = field(compare=False)    # internal_id
    instrument_id: str = field(compare=False)  # instrument_id
    option_type: str = field(compare=False)    # CALL/PUT
    strike_price: float = field(compare=False) # 行权价
    volume: float = field(compare=False)       # 成交量
    price: float = field(compare=False)        # 最新价
    sync_flag: bool = field(compare=False)     # 是否同步
    month: str = field(compare=False)          # 月份
    product: str = field(compare=False)        # 产品
```

**关键**: `@dataclass(order=True)` 使 SortEntry 可比较，比较时只使用 `sort_key` 字段（`compare=True`），其他字段不参与比较。

### 1.2 排序键构建（第91-97行）

```python
sort_key = (
    not sync_flag,                              # 第1优先级
    -(volume if volume > 0 else 0.01),          # 第2优先级
    strike_distance,                            # 第3优先级
    -last_tick_ts,                              # 第4优先级
    internal_id,                                # 第5优先级
)
```

### 1.3 排序优先级详解

| 优先级 | 字段 | 代码表达式 | 排序方向 | 说明 |
|--------|------|-----------|---------|------|
| 1 | sync_flag | `not sync_flag` | **False < True** | sync_flag=True → not True=False → 排前面 |
| 2 | volume | `-(volume)` | 越大越靠前 | 负值实现降序 |
| 3 | strike_distance | `strike_distance` | 越小越靠前 | 虚值距离 |
| 4 | last_tick_ts | `-last_tick_ts` | 越新越靠前 | 负值实现降序 |
| 5 | internal_id | `internal_id` | 越小越靠前 | 兜底稳定性 |

### 1.4 虚值距离计算（第84-88行）

```python
if option_type == 'CALL':
    strike_distance = strike_price - m_price if m_price > 0 else float('inf')
else:  # PUT
    strike_distance = m_price - strike_price if m_price > 0 else float('inf')
```

- **CALL**: 行权价 - 期货价（>0为虚值，=0为平值，<0为实值）
- **PUT**: 期货价 - 行权价（>0为虚值，=0为平值，<0为实值）
- 期货价<=0时，距离为无穷大（排最后）

### 1.5 排序示例

假设有以下期权数据（m_price=100）：

| 期权 | sync_flag | volume | strike | option_type | strike_distance | last_tick_ts |
|------|-----------|--------|--------|-------------|-----------------|--------------|
| A | True | 1000 | 110 | CALL | 10 | 1000 |
| B | True | 2000 | 105 | CALL | 5 | 900 |
| C | False | 3000 | 120 | CALL | 20 | 1100 |
| D | True | 1500 | 108 | CALL | 8 | 1050 |

**排序键计算**:
- A: `(False, -1000, 10, -1000, id_A)`
- B: `(False, -2000, 5, -900, id_B)`
- C: `(True, -3000, 20, -1100, id_C)`
- D: `(False, -1500, 8, -1050, id_D)`

**排序结果**（Python元组比较，逐元素）：
1. B: `(False, -2000, ...)` ← volume最大
2. D: `(False, -1500, 8, ...)` ← volume次之，strike_distance=8
3. A: `(False, -1000, 10, ...)` ← volume再次之
4. C: `(True, ...)` ← sync_flag=False，排最后

**步骤6: 插入/更新排序桶（第1362-1385行）**
```python
bucket = self._sort_buckets[future_internal_id][month][opt_type]

# 检查是否已存在
existing_idx = None
for i, e in enumerate(bucket):
    if e.internal_id == internal_id:
        existing_idx = i
        break

if existing_idx is not None:
    # 移除旧条目，重新堆化
    bucket[existing_idx] = bucket[-1]
    bucket.pop()
    heapq.heapify(bucket)

# 插入新条目
if len(bucket) < self._sort_bucket_max_size:
    heapq.heappush(bucket, entry)
else:
    # 桶满：找最差元素替换
    worst_idx = max(range(len(bucket)), key=lambda i: bucket[i])
    if entry < bucket[worst_idx]:
        bucket[worst_idx] = entry
        heapq.heapify(bucket)
```

### 2.3 桶满处理逻辑（关键修复）

**旧Bug**: 替换堆顶（最优元素），导致最优被丢弃。

**当前正确逻辑**:
1. `max(range(len(bucket)), key=lambda i: bucket[i])` → 找sort_key最大的元素（最差）
2. `if entry < bucket[worst_idx]` → 新条目比最差元素更优
3. 替换最差元素，重新堆化

**桶容量**: `DEFAULT_SORT_BUCKET_MAX_SIZE = 50`

## 三、从排序桶读取（width_cache.py 第1417-1461行）

```python
def select_from_sort_bucket(self, future_internal_id, month, opt_type, top_n=1):
    with self._get_lock(future_internal_id):
        bucket = self._sort_buckets.get(future_internal_id, {}).get(month, {}).get(opt_type, [])
        if not bucket:
            return []
        
        # 堆已按排序键排好序，直接取前top_n个
        sorted_entries = sorted(bucket)[:top_n]
        
        results = []
        for entry in sorted_entries:
            results.append({
                'instrument_id': entry.instrument_id,
                'internal_id': entry.internal_id,
                'option_type': entry.option_type,
                'strike_price': entry.strike_price,
                'volume': entry.volume,
                'price': entry.price,
                'month': entry.month,
                'product': entry.product,
                'sync_flag': entry.sync_flag,
            })
        
        return results
```

**注意**: `sorted(bucket)` 对堆进行全量排序，时间复杂度 O(n log n)，n <= 50。

### 5.4 状态切换决策逻辑（第175-240行）

```python
# 1. 统计各状态数量
for raw_status, count in counts.items():
    mapped = _STATE_MAP.get(raw_status, 'other')
    state_tally[mapped] += count
    total += count

# 2. 判断是否有明确趋势
non_other_total = state_tally.get('correct_trending', 0) + state_tally.get('incorrect_reversal', 0)
if non_other_total > 0 and non_other_total / total >= self._non_other_ratio_threshold:
    candidates = {k: v for k, v in state_tally.items() if k != 'other'}
    best_state = max(candidates, key=lambda s: (candidates[s], _STATE_PRIORITY.get(s, 0)))
else:
    best_state = 'other'

# 3. 确认机制
if best_state != self._current_state:
    # 检查最小持有期
    hold_elapsed = time.time() - self._state_enter_time
    if hold_elapsed < self._min_state_hold_seconds:
        return self._current_state  # 持有期内拒绝切换
    
    # 连续确认
    if self._pending_state == best_state:
        self._pending_confirm_count += 1
    else:

        self._pending_confirm_count = 1
```
