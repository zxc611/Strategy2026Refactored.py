# 三角验证是否应当内化为仓位调整因子 — 顶级基金高度评判报告

**评判日期**: 2026-05-17  
**评判对象**: `evaluation/cascade_judge.py` 三角验证体系 + `mode_engine.py` 仓位计算体系  
**评判视角**: 桥水/Renaissance/Two Sigma 等顶级量化基金仓位管理范式  
**评判结论**: 当前三角验证是**评判门控**而非**仓位调整因子**，存在架构级缺失。建议将三角验证内化为动态仓位调整因子。

---

## 一、当前三角验证架构解析

### 1.1 三角验证的定义与实现

当前代码中，**三角验证**是 `CascadeJudge` 的第二关（安全壳），由三个指标构成：

| 指标 | 阈值 | 行为 | 权重 |
|------|------|------|------|
| 索提诺比率 (Sortino) | ≥1.5 | BLOCK | 0.40 |
| 卡玛比率 (Calmar) | ≥0.8 | BLOCK | 0.30 |
| 夏普比率 (Sharpe) | ≥1.2 | WARN | 0.30 |

**综合评分公式**:
```
final = 0.60 × sigmoid(盈亏比) + 0.40 × tri_score
tri_score = 0.40 × sigmoid(索提诺) + 0.30 × sigmoid(卡玛) + 0.30 × sigmoid(夏普)
```

### 1.2 三角验证在系统中的位置

```
第零关: 数据质量检查 (num_trades ≥ 30)
    ↓
第一关: 盈亏比验证 (profit_loss_ratio ≥ 1.8) — 发动机
    ↓
第二关: 三角验证 (Sortino/Calmar/Sharpe) — 安全壳 ⭐
    ↓
第三关: 小资金约束 (连亏/横盘/保证金) — 驾驶座
    ↓
输出: PASS/BLOCK + final_score
```

### 1.3 关键发现：三角验证是"开关"而非"旋钮"

当前三角验证的**核心问题**：
- 它是一个**二进制门控**（通过/阻断），输出是离散的 PASS/BLOCK/WARN
- 它**不直接参与仓位计算**，只在策略准入阶段发挥作用
- 即使策略通过了三角验证（Sortino=1.6, Calmar=0.9, Sharpe=1.3），仓位计算时**完全不考虑**这些风险指标

## 二、顶级基金仓位管理范式

### 2.1 桥水 (Bridgewater) — 风险平价 + 因子暴露调整

桥水的核心仓位哲学：
- **风险平价**: 每个策略/资产的仓位与其风险贡献成反比
- **因子暴露调整**: 根据策略在宏观因子（增长、通胀、利率）上的暴露动态调整仓位
- **相关性惩罚**: 高相关性策略的仓位被压缩

**关键差异**: 桥水将**风险指标实时内化为仓位权重**，而非仅作为准入门槛。

### 2.2 Renaissance Technologies — 信号强度 × 置信度加权

文艺复兴的核心仓位哲学：
- **信号强度**: 预测模型的输出值直接决定仓位大小
- **置信度加权**: 模型在历史数据上的稳定性（类似Sortino的"下行稳定性"概念）调整仓位倍数
- **动态再平衡**: 每日根据最新信号重新计算全部仓位

**关键差异**: 文艺复兴将**模型置信度（类似三角验证指标）作为仓位乘数**。

### 2.3 Two Sigma — 多因子风险模型 + 目标跟踪误差

Two Sigma的核心仓位哲学：
- **Barra多因子模型**: 将组合风险分解为市场、行业、风格等因子暴露
- **目标跟踪误差**: 根据策略的风险调整后收益（Sharpe-like指标）设定跟踪误差预算
- **风险预算分配**: 高Sharpe策略获得更多风险预算（即更大仓位）

**关键差异**: Two Sigma将**风险调整后收益指标直接映射为风险预算**（即仓位上限）。

### 2.4 AQR — 质量因子 + 防御性仓位调整

AQR的核心仓位哲学：
- **质量评分**: 将盈利能力、成长性、安全性等质量指标综合为评分
- **防御性倾斜**: 在市场高波动期（低Sortino环境）自动降低仓位
- **跨资产风险预算**: 根据各资产类别的风险收益特征动态分配仓位

**关键差异**: AQR将**质量/风险指标作为仓位调整函数的自变量**。

## 三、当前实现与顶级标准的差距分析

### 3.1 差距一：三角验证是"静态门控"而非"动态调整"

| 维度 | 当前实现 | 顶级基金标准 | 差距等级 |
|------|---------|-------------|---------|
| **作用时机** | 策略准入时一次性评判 | 每笔交易/每日动态调整 | 🔴 高 |
| **输出形式** | 离散 PASS/BLOCK | 连续仓位调整因子 [0, 1] | 🔴 高 |
| **反馈闭环** | 无 | 实时风险指标 → 仓位调整 → 新风险指标 | 🔴 高 |

**具体表现**:
- 策略A: Sortino=1.6（刚及格）→ 通过 → 仓位=40%（小资金single_position_cap）
- 策略B: Sortino=3.0（优秀）→ 通过 → 仓位=40%（同样的single_position_cap）

**问题**: 两个风险收益特征截然不同的策略获得了**相同的仓位上限**。顶级基金会给策略B更高的仓位。

### 3.2 差距二：仓位计算与风险指标完全解耦

当前 `mode_engine.py` 的 `compute_position_size` 逻辑：

```python
def compute_position_size(self, equity, entry_price, stop_price, win_rate=0.0, win_loss_ratio=0.0):
    if self._config.kelly_fraction > 0 and win_rate > 0 and win_loss_ratio > 0:
        return kelly_position_size(...)  # 仅依赖胜率、盈亏比
    
    # 固定风险法
    risk_amount = equity * self._config.risk_pct_default
    size = risk_amount / stop_distance
    max_shares = equity * self._config.single_position_cap / entry_price
    return min(size, max_shares)
```

**缺失的风险指标输入**:
- 索提诺比率（下行风险调整收益）
- 卡玛比率（回撤调整收益）
- 夏普比率（总风险调整收益）
- 策略历史最大回撤
- 策略与组合其他策略的相关性

**顶级基金标准**: Renaissance 的仓位公式近似为 `position = signal_strength × confidence_factor × risk_adjustment`，其中 `risk_adjustment` 直接由 Sortino/Sharpe 等三角验证指标驱动。

### 3.3 差距三：无"风险预算"概念

当前架构中：
- `single_position_cap` 是**硬上限**（小资金40%，大资金8%）
- 这个上限**不随策略风险特征变化**

顶级基金标准（以 Two Sigma 为例）：
- 每个策略有**风险预算**（如跟踪误差预算 = 2%）
- 风险预算根据策略的**风险调整后收益**动态分配
- 高 Sortino/Sharpe 策略获得更多风险预算 → 更大仓位

**公式对比**:

```python
# 当前实现（硬上限）
max_position = equity × single_position_cap  # 0.40 或 0.08

# 顶级基金标准（风险预算驱动）
risk_budget = base_budget × risk_adjustment_factor(sortino, calmar, sharpe)
max_position = equity × risk_budget / (volatility × correlation_penalty)
```

### 3.4 差距四：三角验证指标未用于"防御性减仓"

当前实现：
- 三角验证只在**策略准入时**使用
- 一旦策略通过准入，后续运行时**不再监控**三角验证指标
- 即使策略的实时 Sortino 从 1.6 跌到 0.5，仓位也不会自动降低

顶级基金标准（以 AQR 为例）：
- **持续监控**策略的风险指标
- 当 Sortino < 阈值时，自动触发**防御性减仓**
- 减仓幅度与指标偏离程度成正比

## 四、三角验证内化为仓位调整因子的设计方案

### 4.1 核心设计：六维仓位调整因子 (6D-TVF)

将三角验证、订单流、希腊字母六大维度内化为**连续仓位调整因子** [0, 1]：

```python
class SixDimPositionAdjustmentFactor:
    """六维仓位调整因子 — 将风险指标、订单流、希腊字母内化为仓位乘数
    
    六维体系:
    L1 风险收益层 (三角验证): Sortino + Calmar + Sharpe
    L2 市场微观层 (订单流): OFI + CVD偏离 + 智能资金流向
    L3 期权风险层 (希腊字母): Delta暴露 + Gamma风险 + Theta衰减 + Vega波动
    """
    
    def compute_adjustment(
        self,
        # L1: 三角验证指标
        sortino: float,
        calmar: float,
        sharpe: float,
        # L2: 订单流指标
        ofi_score: float = 0.0,           # 订单流失衡指数 [-10, 10]
        cvd_divergence: float = 0.0,      # CVD与价格偏离度
        smart_money_flow: float = 0.0,    # 智能资金流向强度
        # L3: 希腊字母指标
        delta_exposure: float = 0.0,      # 组合Delta暴露 [-1, 1]
        gamma_risk: float = 0.0,          # Gamma风险 (高Gamma=高风险)
        theta_decay: float = 0.0,         # Theta日衰减金额
        vega_exposure: float = 0.0,       # Vega暴露 (IV敏感度)
        # 阈值配置
        sortino_threshold: float = 1.5,
        calmar_threshold: float = 0.8,
        sharpe_threshold: float = 1.2,
    ) -> float:
        """
        计算六维仓位调整因子 [0, 1]
        
        分层计算:
        - L1权重: 0.40 (风险收益是核心)
        - L2权重: 0.35 (市场微观结构提供实时信号)
        - L3权重: 0.25 (期权风险约束)
        """
        # === L1: 三角验证因子 ===
        sortino_factor = self._sigmoid_adjust(sortino, sortino_threshold, scale=0.5)
        calmar_factor = self._sigmoid_adjust(calmar, calmar_threshold, scale=0.3)
        sharpe_factor = self._sigmoid_adjust(sharpe, sharpe_threshold, scale=0.4)
        l1_tvf = (0.40 * sortino_factor + 
                  0.30 * calmar_factor + 
                  0.30 * sharpe_factor)
        
        # === L2: 订单流因子 ===
        # OFI: 正值=买方主导(利好多头), 负值=卖方主导
        ofi_norm = 1.0 / (1.0 + math.exp(-ofi_score / 3.0))  # Sigmoid归一化到[0,1]
        
        # CVD偏离: 正值=CVD上升但价格未涨(潜在上涨), 负值=背离
        cvd_factor = min(1.0, max(0.0, 0.5 + cvd_divergence * 0.5))
        
        # 智能资金流向: 正值=聪明钱流入, 负值=流出
        smf_norm = 1.0 / (1.0 + math.exp(-smart_money_flow / 2.0))
        
        l2_tvf = (0.40 * ofi_norm + 
                  0.30 * cvd_factor + 
                  0.30 * smf_norm)
        
        # === L3: 希腊字母因子 ===
        # Delta暴露: 绝对值越大方向性风险越高, 应降低仓位
        delta_factor = 1.0 - abs(delta_exposure)  # Delta=0时factor=1, Delta=1时factor=0
        
        # Gamma风险: 高Gamma=价格敏感度高=应降低仓位
        gamma_threshold = 0.05  # 经验阈值
        gamma_factor = 1.0 / (1.0 + math.exp((gamma_risk - gamma_threshold) / 0.02))
        
        # Theta衰减: 日衰减占权利金比例越高, 时间价值损耗越快
        theta_threshold = -0.02  # 日衰减2%
        theta_factor = 1.0 / (1.0 + math.exp((theta_decay - theta_threshold) / 0.01))
        
        # Vega暴露: IV敏感度越高, 波动率风险越大
        vega_threshold = 0.10  # 每1% IV变化影响10%权利金
        vega_factor = 1.0 / (1.0 + math.exp((vega_exposure - vega_threshold) / 0.05))
        
        l3_tvf = (0.30 * delta_factor + 
                  0.25 * gamma_factor + 
                  0.25 * theta_factor + 
                  0.20 * vega_factor)
        
        # === 六维综合 ===
        final_tvf = (0.40 * l1_tvf + 
                     0.35 * l2_tvf + 
                     0.25 * l3_tvf)
        
        return min(1.0, max(0.0, final_tvf))
    
    def _sigmoid_adjust(self, value: float, threshold: float, scale: float) -> float:
        """Sigmoid调整: 阈值处=0.5, 2倍阈值处≈0.88"""
        return 1.0 / (1.0 + math.exp(-(value - threshold) / scale))
```

### 4.2 与现有仓位计算的集成

```python
def compute_position_size_v2(
    self, equity: float, entry_price: float, stop_price: float,
    win_rate: float = 0.0, win_loss_ratio: float = 0.0,
    # L1: 三角验证
    sortino: float = 0.0, calmar: float = 0.0, sharpe: float = 0.0,
    # L2: 订单流 (从MicrostructureAnalyzer获取)
    ofi_score: float = 0.0,
    cvd_divergence: float = 0.0,
    smart_money_flow: float = 0.0,
    # L3: 希腊字母 (从GreeksCalculator获取)
    delta_exposure: float = 0.0,
    gamma_risk: float = 0.0,
    theta_decay: float = 0.0,
    vega_exposure: float = 0.0,
) -> float:
    """集成六维仓位调整因子的仓位计算"""
    
    # 1. 基础仓位（凯利或固定风险）
    base_size = self._compute_base_size(equity, entry_price, stop_price, 
                                        win_rate, win_loss_ratio)
    
    # 2. 六维仓位调整因子 [0, 1] ⭐ 新增
    tvf = SixDimPositionAdjustmentFactor().compute_adjustment(
        sortino=sortino, calmar=calmar, sharpe=sharpe,
        ofi_score=ofi_score,
        cvd_divergence=cvd_divergence,
        smart_money_flow=smart_money_flow,
        delta_exposure=delta_exposure,
        gamma_risk=gamma_risk,
        theta_decay=theta_decay,
        vega_exposure=vega_exposure,
    )
    
    # 3. 应用调整因子
    adjusted_size = base_size * tvf
    
    # 4. 单仓上限约束（保留现有逻辑）
    max_shares = equity * self._config.single_position_cap / entry_price
    
    return min(adjusted_size, max_shares)
```

### 4.2a 订单流指标来源与计算

订单流指标从 `MicrostructureAnalyzer` 获取：

| 指标 | 来源方法 | 含义 | 仓位影响 |
|------|---------|------|---------|
| **OFI** | `calc_ofi(lookback_seconds=60)` | 订单流失衡指数 | 买方主导→增仓, 卖方主导→减仓 |
| **CVD偏离** | `get_cvd_divergence()` | 累计成交量差与价格偏离 | 正偏离→潜在上涨→增仓 |
| **智能资金流向** | `calc_smart_money_flow()` | 大单/聪明钱流向 | 流入→增仓, 流出→减仓 |

**关键洞察**: 订单流提供**实时市场微观结构信号**，比历史风险指标（Sortino等）更及时地反映当前市场情绪。例如：
- 策略历史Sortino=2.0（优秀），但实时OFI=-8（强烈卖方主导）→ 应减仓
- 策略历史Sortino=1.5（及格），但实时聪明钱大量流入 → 可增仓

### 4.2b 希腊字母指标来源与计算

希腊字母指标从 `GreeksCalculator` 获取：

| 指标 | 来源方法 | 含义 | 仓位影响 |
|------|---------|------|---------|
| **Delta暴露** | `get_portfolio_greeks()['delta']` | 方向性风险 | Delta→1→满仓方向性风险→减仓 |
| **Gamma风险** | `get_portfolio_greeks()['gamma']` | 价格敏感度 | 高Gamma→大幅波动风险→减仓 |
| **Theta衰减** | `get_portfolio_greeks()['theta']` | 时间价值损耗 | 日衰减>2%→时间成本高→减仓 |
| **Vega暴露** | `get_portfolio_greeks()['vega']` | IV敏感度 | 高Vega→波动率风险→减仓 |

**关键洞察**: 希腊字母提供**期权特有风险约束**，这是股票策略不具备的维度。例如：
- 临近到期（T<5天）且Theta日衰减>5% → 即使信号强烈也应减仓
- IV处于历史高位（IV Percentile>90%）且Vega暴露高 → 应减仓防范IV回归

**关键改进**: 六维因子使仓位调整从"历史后视镜"升级为"实时全景雷达"，同时考虑风险收益、市场微观结构、期权特有风险。

### 4.4 防御性减仓触发机制

```python
def check_defensive_drawdown(
    self, 
    current_sortino: float,
    current_calmar: float,
    entry_sortino: float,
    entry_calmar: float,
) -> float:
    """
    防御性减仓检查
    当实时风险指标显著低于入场时，触发减仓
    """
    sortino_decay = current_sortino / entry_sortino if entry_sortino > 0 else 0
    calmar_decay = current_calmar / entry_calmar if entry_calmar > 0 else 0
    
    # 任一指标衰减超过50%，触发减仓
    if sortino_decay < 0.5 or calmar_decay < 0.5:
        reduction_factor = min(sortino_decay, calmar_decay)
        return reduction_factor  # 如 0.4 → 减仓至40%
    
    return 1.0  # 不减仓
```

## 五、评判总结

### 5.1 当前三角验证的合理之处

1. **门控设计正确**: 作为策略准入的"安全壳"，三角验证的阈值设定（Sortino≥1.5, Calmar≥0.8, Sharpe≥1.2）符合行业最佳实践
2. **权重分配合理**: Sortino(0.40) > Calmar(0.30) ≈ Sharpe(0.30)，体现了对下行风险的重视
3. **瀑布式短路**: 任何一关BLOCK即终止，避免了资源浪费
4. **综合评分**: Sigmoid归一化 + 加权平均的评分方法数学上严谨

### 5.2 与顶级基金标准的核心差距

| 差距 | 严重程度 | 说明 |
|------|---------|------|
| 三角验证是离散门控而非连续调整因子 | 🔴 **高** | 通过和不通过策略获得相同仓位上限 |
| 仓位计算完全忽略三角验证指标 | 🔴 **高** | compute_position_size 不接收 Sortino/Calmar/Sharpe |
| **订单流未参与仓位调整** | 🔴 **高** | OFI/CVD/智能资金流向仅用于信号生成，不影响仓位 |
| **希腊字母未参与仓位调整** | 🔴 **高** | Delta/Gamma/Theta/Vega仅用于风控监控，不影响仓位 |
| 无风险预算动态分配机制 | 🟡 **中** | single_position_cap 是静态硬上限 |
| 无防御性减仓机制 | 🟡 **中** | 运行时风险恶化不触发自动减仓 |
| 三角验证只在准入时使用 | 🟡 **中** | 运行时持续监控缺失 |

### 5.3 是否应当内化？

**结论: 应当内化，但需分阶段实施**

**阶段一（立即）**: 在 `compute_position_size` 中增加六维因子参数，作为可选的仓位调整乘数

**阶段二（短期）**: 在 `ModeConfig` 中增加 `tvf_enabled`、`tvf_threshold`、`order_flow_weight`、`greeks_weight` 配置，支持按资金规模启用/禁用各维度

**阶段三（中期）**: 
- 实现运行时三角验证指标监控 + 防御性减仓触发器
- 集成 `MicrostructureAnalyzer` 实时订单流数据到仓位计算
- 集成 `GreeksCalculator` 实时希腊字母数据到仓位计算

**阶段四（长期）**: 建立风险预算体系，将六维指标作为风险预算分配的核心输入

### 5.4 风险与注意事项

1. **过度拟合风险**: 将六维因子内化为仓位因子可能引入新的过拟合路径，需严格的样本外验证
2. **参数敏感性**: 6D-TVF 的 Sigmoid 参数（scale值）需要大量历史数据校准
3. **与现有风控的冲突**: 需确保 6D-TVF 与现有的 `single_position_cap`、`kelly_fraction` 等机制协调工作
4. **计算复杂度**: 
   - 实时计算 Sortino/Calmar 需要维护收益率序列
   - 实时获取 OFI/CVD 需要调用 `MicrostructureAnalyzer`
   - 实时获取希腊字母需要调用 `GreeksCalculator`
   - 建议采用**异步更新**策略：L1每5分钟更新，L2/L3每秒更新
5. **数据源可靠性**: 订单流数据在极端行情下可能失真（如闪崩时OFI剧烈波动），需设置**异常值过滤**
6. **希腊字母模型风险**: Black-Scholes假设在市场极端条件下失效，需准备**降级方案**（如使用二叉树或蒙特卡洛）

## 六、最终评判

**三角验证当前是"评判门控"，而非"仓位调整因子"。**

从桥水/Renaissance/Two Sigma 等顶级基金的高度来看，这是一个**架构级缺失**。顶级基金的核心仓位哲学是：

> **风险指标不是准入考试的及格线，而是仓位大小的连续调节器。**

当前实现中，一个 Sortino=1.5（刚及格）的策略和一个 Sortino=3.0（优秀）的策略，在通过三角验证后获得了**完全相同的仓位上限**。这在顶级基金看来是不可接受的——优秀的策略应当获得更多的风险预算（即更大的仓位）。

**建议**: 将六维因子（三角验证 + 订单流 + 希腊字母）内化为仓位调整因子（6D-TVF），使仓位大小与策略的风险收益特征、市场微观结构、期权特有风险动态匹配。这是从"及格/不及格"思维到"实时全景优化"思维的跃迁。
