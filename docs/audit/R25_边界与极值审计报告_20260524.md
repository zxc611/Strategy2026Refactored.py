## 维度1：数组/集合边界

### P1-1: risk_service.py — _equity_series[-1]在空deque时崩溃 (P1)
- **文件**: `risk_service.py`
- **行号**: 1973
- **代码**: `current_loss = self._daily_peak_equity - list(self._equity_series)[-1] if self._equity_series else 0`
- **边界条件**: `_equity_series` 是 `deque(maxlen=60)`，在 `_check_daily_drawdown()` 调用前传入，当 `_prev_5day_avg_profit > 0` 且 `_equity_series` 为空时，`list(self._equity_series)[-1]` → `IndexError`
- **当前行为**: 虽然有 `if self._equity_series` 保护，但 `deque` 对象在布尔上下文中始终为 `True`（非空容器），即使 `len()` 为 0 也是 Truthy。`deque()` 是 Truthy 但 `list(deque())` 为空列表，`[-1]` 会抛 `IndexError`
- **正确行为**: 应检查 `len(self._equity_series) > 0` 而非 `if self._equity_series`
- **后果**: 在权益序列首次更新前若触发 `_check_daily_drawdown`，导致风控线程崩溃，可能遗漏日回撤硬停止检查
- **同样的问题**: `risk_service.py:1898` `recent[-1]` — 虽然上层 `_check_circuit_breaker` 检查了 `len(self._equity_series) < 10` 后 `recent = list(self._equity_series)`，`len(recent) < 3` 后退出，路径安全

### P1-2: strategy_ecosystem.py — cached_evs[-1]索引在空列表时崩溃 (P1)
- **文件**: `strategy_ecosystem.py`
- **行号**: 1380
- **代码**: `"final_cached_ev": round(cached_evs[-1], 6)`
- **边界条件**: 当 `cached_evs` 列表为空时
- **当前行为**: 如果上游返回空EV列表，直接索引 `[-1]` 抛出 `IndexError`
- **正确行为**: 应添加 `if cached_evs:` 守卫，返回合适的默认值
- **后果**: EV缓存返回空时导致报告生成崩溃

### P1-3: box_spring_strategy.py — candidates[0]在空列表时崩溃 (P1)

### P1-4: _tick_dispatch_layer — on_tick中instrument_id可能为空字符串 (P1)

- **后果**: 空合约ID的tick被分发到随机分片，造成无意义的数据库写入和内存占用

## 维度2：数值极值边界

### P0-1: position_service.py — open_price=0时除零导致profit_pct为无穷大或NaN (P0)
- **文件**: `position_service.py`
- **行号**: 402-406
- **代码**: 
  ```python
  if record.volume > 0:
      profit_pct = (price - record.open_price) / record.open_price
  else:
      profit_pct = (record.open_price - price) / record.open_price
  ```
- **边界条件**: `record.open_price == 0` 时除零
- **当前行为**: 当 `open_price` 为 0 时，直接除零 → `ZeroDivisionError`
- **正确行为**: 应在行402中添加 `record.open_price > 0` 检查（`if record.volume != 0 and record.open_price > 0:`）
- **后果**: 开仓价格为0的异常持仓（如数据错误）会导致 `on_tick` 崩溃，所有持仓的止盈止损检查中断

### P0-2: box_spring_strategy.py — entry_premium=0时的除零链 (P0)
- **文件**: `box_spring_strategy.py`
- **行号**: 116-117 (SpringPosition.pnl_ratio), 128 (should_accept_loss), 701 (compute_equity_based_lots)
- **代码**:
  ```python
  return self.current_premium / self.entry_premium  # line 118
  loss_pct = 1.0 - (self.current_premium / self.entry_premium)  # line 128
  ```
- **边界条件**: 权利金价格=0时（流动性枯竭、涨停跌停期权）
- **当前行为**: `entry_premium=0` 时直接除零 → `ZeroDivisionError`
- **正确行为**: 在 `pnl_ratio` 和 `should_accept_loss` 属性中已有 `if self.entry_premium <= 0: return 0.0/True`，→ **已受保护，降到P1**
- **复查结果**: 属性方法有保护，但 `compute_equity_based_lots` 中 `cost_per_lot = premium_price * 10000.0` 当 premium_price<=0时返回1手 → **路径安全**
- **降级**: 已受保护但保护不完整。`pnl_ratio` 返回0.0（无盈亏）在权利金为0时误导——权利金0意味着无限收益
- **后果**: 权利金为0的期权（深度虚值）盈亏计算逻辑错误

### P1-5: quant_core.py — price<=0时close/prev_close除零链 (P1)
- **文件**: `quant_core.py`
- **行号**: 89 (`close <= 0 or high < low`), 144 (`momentum = (curr_close - prev_close) / prev_close if prev_close > 0`)
- **边界条件**: 价格=0或负价格
- **当前行为**: `MultiPeriodTrendScorer.update()` 行89有 `close <= 0` 守卫返回 empty_result；行144有 `if prev_close > 0` 保护
- **路径安全**，但 `high < low` 的检查在行89—如果 high==low（无波动），tr=0，后续atr=0时plus_di/minus_di除atr在行133有`if atr > 0`保护 → **路径安全**

### P1-6: quant_core.py — AdaptiveHMM无inf/nan保护的观测值 (P1)
- **文件**: `quant_core.py`
- **行号**: 370-371
- **代码**: `if not math.isfinite(observation): return self._empty_result()`
- **边界条件**: 观测值为 `Inf`、`-Inf`、`NaN`
- **当前行为**: 有 `math.isfinite()` 检查 → **已受保护**
- **后果**: 无

### P2-2: risk_service.py — _check_circuit_breaker中std_diff=1.0硬编码退化值 (P2)
- **文件**: `risk_service.py`
- **行号**: 1916
- **代码**: `std_diff = 1.0`（当`len(diffs) < 2`时）
- **边界条件**: 权益序列不足时，1.0的默认std_diff在one_min_ago_val较小时（如初始资金100000）产生极低阈值 `2.5 * 1.0 / 100000 = 0.000025`，任何微小回撤都触发断路器
- **当前行为**: 初始阶段权益微小波动可能触发断路器
- **正确行为**: 应使用相对阈值（如默认的 `DEFAULT_ANOMALY_THRESHOLD = 0.05`）作为保底

- **行号**: 552-562

