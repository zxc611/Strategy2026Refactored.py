## 三、P0-R9-15修复验证报告

### 3.1 问题定位

**文件：** `evaluation/cascade_judge.py:165-174`

**问题：** Sortino分母使用n_downside（下行样本数），标准定义应为N（总样本数）

**标准定义：**
```
Sortino Ratio = (R_p - MAR) / σ_d

其中：
R_p = 组合收益率
MAR = 最小可接受收益率（通常为0）
σ_d = 下行标准差 = sqrt(sum(min(0, R_i - MAR)^2) / N)
N = 总样本数
```

### 3.2 修复代码

**修复前：**
```python
downside = [min(0, ret - mar) ** 2 for ret in returns]
n_downside = sum(1 for d in downside if d > 0)
downside_dev = math.sqrt(sum(downside) / n_downside) if n_downside > 0 else 0.0
```

**修复后：**
```python
downside = [min(0, ret - mar) ** 2 for ret in returns]
# P0-R9-15修复: Sortino分母使用总样本数N（标准定义），而非n_downside
# 参考: https://en.wikipedia.org/wiki/Sortino_ratio
# 标准定义: Sortino = (mean_ret - MAR) / downside_deviation
# 其中: downside_deviation = sqrt(sum(min(0, ret - MAR)^2) / N)
n_total = len(returns)
downside_dev = math.sqrt(sum(downside) / n_total) if n_total > 0 else 0.0
```

#### 3.3.5 参数改变→结果改变验证（验收标准5）

**数学验证：**

- mar = 0.0

**修复前（错误）：**
```
downside = [0, 0.0004, 0, 0.0001, 0] = [0, 0.0004, 0, 0.0001, 0]
n_downside = 2 (非零元素个数)
downside_dev = sqrt((0.0004 + 0.0001) / 2) = sqrt(0.00025) = 0.0158
```

**修复后（正确）：**
```
downside = [0, 0.0004, 0, 0.0001, 0]
n_total = 5
downside_dev = sqrt((0.0004 + 0.0001) / 5) = sqrt(0.0001) = 0.01
```

**差异：** 0.0158 ≠ 0.01，修复后结果改变

**结论：** ✅ 非空转
