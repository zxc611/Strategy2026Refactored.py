## 二、P0 严重问题（代码克隆/语义重复/行为分叉）

### P0-24：FutureLeakException 与 FutureDataLeakException 语义相同但类名不同

| 维度 | 详情 |
|------|------|
| **重复文件** | `param_pool/l1_quantification/meta_audit_engine.py:433` vs `strategy/kline_data_service.py:24` vs `param_pool/L1参数量化/v7_meta_audit_v2.py:480` vs `参数池/L1参数量化/meta_audit_engine.py:419` |
| **重复类** | `FutureLeakException(Exception)`（3处） vs `FutureDataLeakException(Exception)`（1处） |
| **重复内容** | 四个文件中定义了语义完全相同的"未来数据泄漏"异常，但类名不同（`FutureLeakException` vs `FutureDataLeakException`） |
| **风险** | 捕获 `FutureLeakException` 时无法捕获 `FutureDataLeakException`，导致异常处理遗漏；中文目录 `参数池` 与 `param_pool` 并存加剧了重复 |
| **根因** | 参数池L1量化模块与策略数据服务模块独立开发，未共享异常定义；中文目录镜像残留 |

**优化方案：**
1. 将 `FutureLeakException` 统一迁移至 `infra/exceptions.py`，所有模块从该入口导入
2. 废弃 `FutureDataLeakException`，统一使用 `FutureLeakException`
3. 删除中文目录 `参数池/L1参数量化/meta_audit_engine.py` 中的重复定义

**预期收益：** 消除4处重复异常定义，统一异常捕获逻辑，消除异常遗漏风险。

## 三、P1 中等问题（功能重叠需统一抽象）

### P1-31：TickEvent 在 strategy 和 infra 模块重复定义

| 维度 | 详情 |
|------|------|
| **infra版** | `infra/event_bus_events.py:38-47`：`class TickEvent(BaseEvent)`，含 `instrument_id` 和 `tick_data` 字段 |
| **strategy版** | `strategy/tick_event_types.py:30-35`：`class TickEvent(MarketEvent)`，含 `tick` 字段 |
| **风险** | 两个 `TickEvent` 分别继承自 `BaseEvent` 和 `MarketEvent`，字段不同但语义相同（均表示Tick数据事件），可能导致事件总线消费者混淆 |

**预期收益：** 统一事件模型，消除消费者混淆风险。

### P2-12：回撤决策冲突——DrawdownManager vs DrawdownMonitorService

| 维度 | 详情 |
|------|------|
| **DrawdownManager** | `governance/mode_exit_rules.py`：治理层策略性回撤管理，VaR/ATR动态升级，渐进式响应（减仓→暂停→全停） |

## 八、v7.0 关键结论

1. **异常类是最大盲区**：`FutureLeakException` 与 `FutureDataLeakException` 语义相同但类名不同，导致异常捕获遗漏；中文目录 `参数池` 与 `param_pool` 并存加剧了重复。
2. **数据模型重复隐蔽**：`OrderResult` 在 `order_base.py` 和 `order_executor.py` 中完全复制，`RiskCheckResult` 同名不同义（Enum vs dataclass）。
3. **常量硬编码是维护噩梦**：`RISK_FREE_RATE` 在8+文件中硬编码，`_CHINA_TZ` 在10+处重复定义，修改时极易遗漏。
4. **接口契约缺乏统一约束**：`health_check` / `get_service_name` 在6+个服务类中隐式重复实现，但无统一 ABC/Protocol 约束。
5. **序列化工具使用不一致**：`json_dumps`/`json_loads` 已存在，但部分模块仍直接使用原生 `json`，可能导致 datetime/Enum 序列化错误。

#### P0-29：断路器状态管理完全重复——CircuitBreakerHalfOpen vs CircuitBreakerService内嵌逻辑

| 维度 | 详情 |
|------|------|
| **重复文件** | `infra/resilience_circuit.py:9-94` vs `risk/safety_meta_circuit.py:35-340` |

### 10.3 v8.0 关键结论

1. **断路器与限流器是架构层最大行为分叉**：`CircuitBreakerService` 内嵌了 `CircuitBreakerHalfOpen` 却又自建了一套状态机；`RateLimiter`(令牌桶)与`SimpleRateLimiter`(滑动窗口)在同一系统内使用不同算法，限流行为不一致。
2. **DI容器三轨并行**：`ServiceContainer`、`SingletonRegistry`、`ServiceFactory` 三套注册-获取机制并存，基础设施层缺乏统一的服务注册中心。
3. **调度器双轨并行**：`StrategyScheduler`(APScheduler)与`SchedulerService`(自研)同时存在，调度行为不统一。
4. **监控指标双系统**：`MetricsRegistry`(Prometheus风格)与`PathCounter`(路径层级字典)各自维护独立存储，无法互操作。
5. **配置热更新三入口**：三个同名但语义不同的 `check_config_hot_reload`，调用方难以区分正确入口。
6. **版本管理三轨并行**：`ParamVersionManager`、`config_version_tracker`、`ModelVersionManager` 三套版本管理机制完全独立实现。

### 12.3 v9.0 关键结论

1. **算法实现是最隐蔽的重复**：`_compute_option_mtm_price`（Black-Scholes）在 `backtest_state.py` 和 `backtest_loop_core.py` 中完全复制；`_norm_cdf` 在6+处重复定义；夏普比率在5+处独立实现。这些金融核心算法的重复会导致回测与生产定价结果不一致。
2. **模块拆分是重复的主要根因**：`backtest_loop_*` 系列、`delay_time_sharpe_3d*`、`preprocess_*` 等模块拆分后，原文件未清理重复函数，形成"拆分即复制"的恶性循环。
3. **中文目录是重复放大器**：`参数池` 与 `param_pool`、`L1参数量化` 与 `l1_quantification` 并存，同一功能在四个路径下存在副本。
4. **ID生成缺乏统一规范**：订单ID、信号ID、仓位ID、HFT信号ID各自为政，前缀长度、UUID截断长度、时间戳格式均不统一，给日志追踪和订单关联带来困难。
5. **原子文件写入未统一**：`os.replace(tmp, target)` 在5+处内联使用，而 `atomic_replace_file` 已存在但未推广。
6. **随机数种子管理混乱**：`random.seed` 和 `np.random.seed` 在生产和测试代码中分散设置，缺乏统一的种子管理器，影响回测可复现性。

### 14.3 v10.0 关键结论

1. **缺乏统一基础设施抽象是最大根因**：`infra` 层未提供状态机、序列化mixin、缓存、哈希计算、HTTP客户端等通用抽象，导致各业务模块重复实现。本轮12项P1问题中有6项直接源于此。

8. **strategy_ecosystem.py 承担了过多聚合职责**：多次遍历6个策略槽位进行不同目标的聚合计算（持仓数、资金分配、活跃状态），未提取公共遍历抽象。

**用户声称"经过修复"属实，但修复主要集中在低风险、易执行的删除和re-export操作。**

**核心发现：**
1. **"已修复"声明存在夸大**：约29%（20/69）的问题可判定为已修复，但大部分仅为删除文件或改为re-export，未触及深层架构问题。
2. **48%（33/69）的问题仍明确存在**，主要为需要设计决策和架构调整的深层问题（生产/回测行为分叉统一、事件系统统一、配置管理统一等）。

4. **备份目录确实已清理**，这是少数完全真实的修复项之一。
5. **中文目录 `参数池/` 是最大隐患**：仍有22+个独立文件未迁移，与 `param_pool/` 形成双轨制维护。
6. **时区常量 `_CHINA_TZ` 是系统性遗漏**：25+文件仍独立定义，修改时区规则时极易遗漏。
