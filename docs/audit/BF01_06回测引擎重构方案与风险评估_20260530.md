## 一、BF-01：止损仅检查收盘价 → Bar内极值穿透止损

### 1.1 问题根因

```
当前代码（L2678, L2700-2713）：
  price = bar.get("close", 0.0)          # 仅取收盘价
  if price <= pos.stop_loss_price:       # 用收盘价判断止损
      should_close = True

实盘场景：
  Bar内high=4100, low=4050, close=4080, stop_loss=4060
  → 实盘：low=4050 < stop_loss=4060，止损在4060触发
  → 回测：close=4080 > stop_loss=4060，止损不触发 ← 偏差！
```

### 1.2 重构方案

**方案A（推荐）：Bar内OHLC顺序穿透检查**

```python
def _check_stop_on_bar_intra(pos, bar, params):
    """BF-01修复：Bar内价格路径模拟，检查止损/止盈是否被穿越

    根据OHLC关系推断Bar内价格路径：
    - 如果close > open（阳线）：先触达open→high→low→close
    - 如果close < open（阴线）：先触达open→low→high→close
    - 如果close == open（十字星）：先触达high→low→close

    Returns:
        (should_close, close_reason, fill_price)
    """
    o = bar.get("open", 0.0)
    h = bar.get("high", 0.0)
    l = bar.get("low", 0.0)
    c = bar.get("close", 0.0)

    sl = pos.stop_loss_price
    sp = pos.stop_profit_price

    if pos.volume > 0:  # 多头
        if c >= o:  # 阳线：open→high→low→close
            if h >= sp: return True, "StopProfit", sp
            if l <= sl: return True, "StopLoss", sl
        else:  # 阴线：open→low→high→close
            if l <= sl: return True, "StopLoss", sl
            if h >= sp: return True, "StopProfit", sp
    elif pos.volume < 0:  # 空头
        if c >= o:  # 阳线：open→high→low→close
            if h >= sl: return True, "StopLoss", sl
            if l <= sp: return True, "StopProfit", sp
        else:  # 阴线：open→low→high→close
            if l <= sp: return True, "StopProfit", sp
            if h >= sl: return True, "StopLoss", sl

    return False, None, None
```

**方案B（可选增强）：Tick插值精确止损**

- 使用`_interpolate_ticks_in_bar()`（已实现，L6070-6160）将Bar拆为Tick序列
- 逐Tick检查止损触发点
- 更精确但性能开销大（每Bar约10-30个Tick点）
- 建议：仅在`enable_tick_backtest=True`时启用

## 二、BF-02：开仓价格=收盘价 → 根据order_type获取成交价

### 2.1 问题根因

```
当前代码（L2372）：

  pos = _BacktestPosition(open_price=price, ...)

实盘场景：
  市价单：应以next_bar.open成交（看到信号→下一Bar开盘执行）
  限价单：应在limit_price成交（排队等候）
  当前回测：全部用本Bar收盘价 ← 偏差！
```

### 2.2 重构方案

**将占位实现`_simulate_market_order_slippage`（L377-424）接入主流程**

```python
def _get_fill_price(bar, order_type, direction, params, next_bar=None):
    """BF-02修复：根据订单类型获取成交价

    Args:
        bar: 当前Bar数据
        order_type: "market" / "limit" / "stop"
        direction: 1(买入) / -1(卖出)
        params: 参数dict
        next_bar: 下一Bar数据（用于市价单延迟执行场景）

    Returns:
        fill_price: 成交价格
    """
    price_mode = params.get("market_order_price_mode", "weighted")
    slippage_bps = params.get("market_order_slippage_bps", 5.0)

    if order_type == "limit":
        return params.get("limit_price", bar.get("close"))

    if order_type == "market":
        if next_bar is not None and params.get("execution_on_next_bar", False):
            base_price = next_bar.get("open")  # 下一Bar开盘价
        else:
            # 当前Bar内加权成交价
            o = bar.get("open", 0.0)
            h = bar.get("high", 0.0)
            l = bar.get("low", 0.0)
            c = bar.get("close", 0.0)
            if price_mode == "weighted":
                base_price = (o + h + l + c) / 4.0
            elif price_mode == "open":
                base_price = o
            elif price_mode == "close":
                base_price = c
            elif price_mode == "random":
                base_price = o + (c - o) * _pyrandom.random()
            else:
                base_price = c

        slippage = base_price * slippage_bps / 10000.0
        return base_price + direction * slippage

    return bar.get("close")
```

## 三、BF-03：决策-执行同Bar → 延迟执行队列

### 3.1 问题根因

```
当前代码（L3082-3155）：
  for idx in range(len(bar_data)):
      bar = bar_data.iloc[idx]
      _check_positions(bt, bar, params)      # (1) 持仓检查
      _check_state_transition(...)            # (2) 状态转换
      should_open = _check_signal(...)        # (3) 信号检测
      if should_open:
          _try_open(bt, bar, params)          # (4) 立即开仓 ← 信号与执行同Bar！

实盘场景：
  Bar[i]末产生信号 → Bar[i+1]开盘价执行（至少1Bar延迟）
  当前回测：Bar[i]产生信号 → Bar[i]收盘价执行 ← 偏差！
```

### 3.2 重构方案

**引入延迟执行队列`_PendingOrderQueue`**

```python
@dataclass
class _PendingOrder:
    signal_bar_idx: int           # 信号产生Bar索引
    signal_bar_time: Any          # 信号产生时间
    strategy: str                 # 策略名
    symbol: str                   # 合约
    direction: int                # 1/-1
    lots: int                     # 手数
    order_type: str               # "market"/"limit"/"stop"
    limit_price: Optional[float]  # 限价单价格
    delay_bars: int               # 延迟Bar数（≥1）
    created_at_equity: float      # 创建时权益（用于风控检查）
    expire_bars: int              # 过期Bar数（超时未执行则取消）


class _PendingOrderQueue:
    def __init__(self):
        self._orders: List[_PendingOrder] = []

    def enqueue(self, order: _PendingOrder):
        self._orders.append(order)

    def check_and_execute(self, bt, bar, bar_idx, params):
        """检查到期的延迟订单并执行"""
        executed = []
        remaining = []
        for order in self._orders:
            elapsed = bar_idx - order.signal_bar_idx
            if elapsed >= order.delay_bars:
                # 执行条件检查：权益是否足够、信号是否仍有效
                if self._validate_order_still_valid(bt, order, bar, params):
                    fill_price = _get_fill_price(bar, order.order_type,
                                                  order.direction, params)
                    _execute_open(bt, bar, order, fill_price, params)
                    executed.append(order)
                else:
                    remaining.append(order)  # 信号失效但未过期
            elif elapsed >= order.expire_bars:
                pass  # 过期取消
            else:
                remaining.append(order)
        self._orders = remaining
        return executed
```

### 3.3 主循环修改

```python

```

## 四、BF-04：无maker/taker手续费区分 → 四维费率模型

### 4.1 问题根因

```
当前FEE_STRUCTURE（L183-191）：三维模型
  品种 × 方向(开/平今/隔夜) → 费率

  品种 × 方向(开/平) × 角色(maker/taker) → 费率
```

## 五、BF-05：无逐Bar Mark-to-Market盯市 → 浮动PnL计入equity

### 5.1 问题根因

```
当前代码（L3154-3155）：
  bt.equity_curve.append(bt.equity)    # bt.equity仅含已实现PnL

  问题：持仓期间的浮动盈亏不计入equity_curve
  → Sharpe/最大回撤/Calmar基于"已实现PnL"而非"总权益"
  → 10手持仓浮亏5%但不反映在equity中 ← 偏差！

实盘场景：
  逐Bar盯市：equity = cash + sum(position.mark_to_market_price * lots * multiplier)
  当前回测：equity = initial_cash + sum(closed_trade_pnl)
```

### 5.2 重构方案

**新增`_mark_to_market`函数**

```python
def _mark_to_market(bt, bar, params):
    """BF-05修复：逐Bar盯市 — 计算所有持仓的浮动PnL并更新盯市权益

    Returns:
        mtm_equity: 包含浮动PnL的总权益
    """
    float_pnl_total = 0.0

    for sym, pos in bt.positions.items():
        if pos.volume == 0:
            continue

        current_price = bar.get("close", 0.0)

        if pos.volume > 0:  # 多头
            float_pnl = (current_price - pos.open_price) * pos.lots * pos.multiplier
        else:  # 空头
            float_pnl = (pos.open_price - current_price) * pos.lots * pos.multiplier

        # 扣减持仓保证金机会成本（可选）
        if params.get("include_margin_cost", False):
            margin = pos.open_price * pos.lots * pos.multiplier * params.get("margin_ratio", 0.10)
            hold_bars = getattr(pos, 'hold_bars', 0)
            funding_cost = margin * params.get("funding_rate_annual", 0.03) * hold_bars / (240 * 4 * 60)
            float_pnl -= funding_cost

        float_pnl_total += float_pnl

    mtm_equity = bt.equity + float_pnl_total
    return mtm_equity
```

**主循环修改**

## 六、BF-06：完美流动性假设 → 成交量约束+内嵌流动性检查

### 6.1 问题根因

```
当前代码（_try_open L2381附近）：
```

### 6.2 重构方案

**方案1：成交量参与率约束（推荐）**

```python
def _check_volume_constraint(lots, bar, symbol, params):
    """BF-06修复：成交量约束检查

    Args:
        lots: 拟下单手数
        bar: 当前Bar数据
        symbol: 合约代码
        params: 参数dict

    Returns:
        (adjusted_lots, volume_warning)
    """
    max_participation_rate = params.get("max_participation_rate", 0.10)  # 最大参与率10%
    bar_volume = bar.get("volume", 0)

    if bar_volume <= 0:
        return 0, "ZERO_VOLUME"

    max_lots_by_volume = int(bar_volume * max_participation_rate)
    if max_lots_by_volume <= 0:
        max_lots_by_volume = 1  # 至少允许1手

    if lots > max_lots_by_volume:
        original_lots = lots
        lots = max_lots_by_volume
        return lots, f"VOLUME_CONSTRAINED: {original_lots}→{lots} (participation={lots/bar_volume:.1%})"

    return lots, None
```

**方案2：将validate_liquidity_stress内嵌为回测风控**

```python
def _check_liquidity_in_realtime(bt, bar, symbol, lots, params):
    """BF-06修复：回测内实时流动性检查（替代后置验证）

    检查项：
    1. bid_ask_spread是否异常扩大（>正常值×3）
    2. 参与率是否超限（>max_participation_rate）
    3. 流动性层级是否过低（option_far的lots需缩减）
    """
    spread = bar.get("bid_ask_spread", 0)
    normal_spread = params.get("normal_spread_bps", 5.0) * bar.get("close", 1) / 10000
    if spread > normal_spread * 3:
        return False, "SPREAD_WIDENED"

    tier = _infer_liquidity_tier(symbol, bar)
    if tier == "option_far":
        lots = max(1, lots // 2)  # 深度虚值期权手数减半

    return True, None
```
