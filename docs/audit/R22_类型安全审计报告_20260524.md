### P1-1. RingBuffer 公共方法缺少参数/返回类型标注
- **文件**: `shared_utils.py`, **行号**: 207-215
- **代码**:
```python
def append(self, item):          # item类型缺失
def extend(self, items):         # items类型缺失

def __getitem__(self, index):    # index和返回类型缺失
```

## 维度二：类型标注矛盾

### P1-3. _stats中last_signal_time初始None后赋datetime.now()
- **文件**: `signal_service.py`, **行号**: 79-84
- **代码**: `self._stats = {'last_signal_time': None, ...}` → 后赋值 `datetime.now()`
- **后果**: 同一键在不同生命周期持有None/datetime不同类型，违反类型稳定性。

### P2-10. _BacktestState.bar/params与None默认值矛盾
- **文件**: `参数池/task_scheduler.py`, **行号**: 1150-1151
- **代码**: `bar: pd.Series = None`, `params: Dict[str, float] = None`
- **后果**: 严格类型检查器报错，函数体内做多余None防御。

### P2-11. apply_decision_score_filter返回类型不实
- **文件**: `signal_service.py`, **行号**: 693
- **代码**: `-> Optional[Dict[str, Any]]:` 但始终 `return signal`（非None且原地改）
- **后果**: 调用方做多余None检查，误以为返回独立副本。

### P2-12. StrategyCoreService.params缓加载类型漂移
- **文件**: `strategy_core_service.py`
- **代码**: 声明`Optional[Dict]`，加载后始终非None，所有访问点需None守卫
- **后果**: 热路径出现不一致访问模式(getattr vs .get)

## 维度三：隐式类型强制转换

### P1-4. ParamsService.get_int中float()->int()级联强转
- **文件**: `params_service.py`, **行号**: 1022/1041-1043
- **代码**: `float_val = float(val); int_val = int(float_val)` (截断非四舍五入)
- **后果**: 3.99→3而非4；nan→ValueError

### P1-5. AdaptiveSignalThreshold中bool->int隐式转换
- **文件**: `signal_service.py`, **行号**: 913
- **代码**: `pass_rate = sum(self._recent_passes) / len(...)` 其中_recent_passes为List[bool]
- **后果**: 依赖bool是int子类；混入非bool值统计失真

### P1-6. ParamsService._track_recent_pass中bool->int隐式转换
- **文件**: `params_service.py`, **行号**: 1027
- **代码**: 同P1-5模式
- **后果**: 同P1-5

### P2-13. to_float32中float(value)隐式转换
- **文件**: `shared_utils.py`, **行号**: 53
- **代码**: `float(value)` 当value为"abc"/None时抛异常
- **后果**: 热路径异常成本高

### P2-14. _calc_initial_stop使用魔术数字0.005

## 维度四：Optional处理缺陷

### P0-1. _execute_pursuit_exit中pos在锁外访问(线程安全)
- **文件**: `strategy_tick_handler.py`, **行号**: 896-903
- **代码**:
```python
with pe._lock:
    pos = pe._positions.get(instrument_id)  # Optional
    if pos: ...
if pos and ...:  # 锁外检查pos
    pe._positions.pop(instrument_id, None)
```
- **后果**: 多线程环境中可能竞态。代码意图不清晰。

### P1-7. ParamsService.get返回Any，调用方不检None
- **文件**: `params_service.py`, **行号**: 1022
- **代码**: `val = self._params.get(key, default)` default可为None
- **后果**: `get_int`返回0时无法区分"参数为0"还是"缺失用默认值0"

### P1-8. _default_cooldown_seconds恢复时未检类型
- **文件**: `signal_service.py`, **行号**: 535
- **代码**: 配置可能残留为str类型
- **后果**: 恢复后下游float运算失败

### P1-9. ModeEngine.filter_signal_by_mode config=None无条件放行
- **文件**: `mode_engine.py`, **行号**: 802-803
- **代码**: `if self._config is None: return True, ''`
- **后果**: 初始化完成前所有信号不受约束

### P2-15. TVFParamLoader._params类变量与实例变量混淆
- **文件**: `tvf_param_loader.py`, **行号**: 55-56
- **代码**: `_params: Dict[str, Any] = {}` (类变量)
- **后果**: 当前单例模式无问题，多实例会数据污染

## 维度七：鸭子类型风险

### P0-2. _process_tick通过字符串迭代搜索属性名获取引擎引用
- **文件**: `strategy_tick_handler.py`, **行号**: 364-384
- **代码**:
```python
for attr_name in ('_box_spring', '_spring', '_box_spring_strategy'):
    candidate = getattr(self, attr_name, None)
    if candidate and hasattr(candidate, '_pullback_mgr'):
        self._pullback_mgr_ref = candidate._pullback_mgr
        break
else:
    self._pullback_mgr_ref = None
```
- **后果**: 任一属性重构移除_pullback_mgr则静默跳过，回调节点拉回信号功能退化
