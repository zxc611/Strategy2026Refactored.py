## 二、M1 生产模块（Production）功能结构

### 2.1 架构概览

生产模块采用 **Facade + 三层分层 + Service组合 + 惰性导入** 的架构模式，核心入口为 `ProductionQuantSystem.py` 和 `StrategyCoreService`。共计 **12 类核心子系统**，214个文件（210个子系统文件+4个根目录入口）。所有子系统 `__init__.py` 采用 `__getattr__` 惰性导入，`__all__` 白名单≤7。Mixin已消灭（G2b），6个Service通过ServiceFactory注入，1个向后兼容Shim（_InstrumentHelperMixin）保留。根目录仅保留4个入口文件，所有实现内聚于子系统目录，无Facade壳残留。config_params循环依赖已消除（自引用import移除+模块级import改延迟代理）。
