## 二、验收标准7：排序指标与扫描目标一致

### 2.1 核查结果：**❌ 不符合**

### 2.2 代码定位

**文件：** `参数池/task_scheduler.py`

**扫描参数网格定义（第428-437行）：**
```python
PARAM_GRID_ROUND1 = {
    "close_take_profit_ratio": OPTION_TAKE_PROFIT_GRID,
    "close_stop_loss_ratio": OPTION_STOP_LOSS_GRID,
    "max_risk_ratio": [0.10, 0.15, 0.20, 0.25, 0.30, 0.40, 0.50, 0.80],
    "lots_min": [1, 2, 3, 5],
    "signal_cooldown_sec": [0.0, 30.0, 60.0, 90.0, 120.0, 180.0],
    "non_other_ratio_threshold": [0.25, 0.30, 0.35, 0.40, 0.45, 0.50, 0.55, 0.60, 0.65, 0.70, 0.75, 0.80, 0.85, 0.90],
}
```

**排序指标映射定义（第954-958行）：**
```python
PARAM_SORT_METRIC_OVERRIDE = {
    'max_risk_ratio': 'risk_adjusted_score',  # 风控参数按风控指标排序
    'close_stop_loss_ratio': 'stop_loss_hit_rate',  # 止损参数按止损指标排序
    'close_take_profit_ratio': 'profit_loss_ratio',  # 止盈参数按盈亏比排序
}
```

### 2.3 问题分析

| 扫描参数 | 是否有排序映射 | 排序指标 | 是否一致 |
|---------|:------------:|---------|:-------:|
| close_take_profit_ratio | ✅ | profit_loss_ratio | ✅ |
| close_stop_loss_ratio | ✅ | stop_loss_hit_rate | ✅ |
| max_risk_ratio | ✅ | risk_adjusted_score | ✅ |
| **lots_min** | ❌ | **缺失** | ❌ |
| **signal_cooldown_sec** | ❌ | **缺失** | ❌ |
| **non_other_ratio_threshold** | ❌ | **缺失** | ❌ |

**实证证据：**
- PARAM_GRID_ROUND1扫描6个参数
- PARAM_SORT_METRIC_OVERRIDE仅定义3个参数的排序映射
- **3个参数缺失排序指标映射**，违反验收标准7

### 2.4 修复方案

需要在PARAM_SORT_METRIC_OVERRIDE中添加：

```python
PARAM_SORT_METRIC_OVERRIDE = {
    'max_risk_ratio': 'risk_adjusted_score',
    'close_stop_loss_ratio': 'stop_loss_hit_rate',
    'close_take_profit_ratio': 'profit_loss_ratio',
    # 新增以下3项：
    'lots_min': 'total_return',  # 仓位参数按总收益排序
    'signal_cooldown_sec': 'signal_quality',  # 冷却参数按信号质量排序
    'non_other_ratio_threshold': 'state_accuracy',  # 状态阈值按状态准确率排序
}
```
