## 四、架构问题缓解

### 原问题：全局单例污染导致测试失败

**现象：** 单独运行测试PASS，批量运行FAIL（19项）

**根因：** 
- `SignalService._default_cooldown_seconds` 等实例属性被前序测试修改
- `EventBus._global_event_bus` 单例跨测试共享

**缓解方案：**

```python
# ali2026v3_trading/tests/conftest.py
def _reset_all_singletons():
    modules_to_reset = [k for k in sys.modules if k.startswith('ali2026v3_trading')]
    singleton_attrs = ['_global_instance', '_instance', '_global_signal_service', ...]
    for mod_name in modules_to_reset:
        mod = sys.modules.get(mod_name)
        for attr in singleton_attrs:
            if hasattr(mod, attr):
                setattr(mod, attr, None)

@pytest.fixture(autouse=True)
def _isolate_global_state():
    _reset_all_singletons()
    yield
    _reset_all_singletons()
```

**效果：** 全局单例在每个测试前后重置，测试间状态隔离，批量运行100%通过。
