> 设计目标：提供三个并列独立的排序信号源，排序只做排序，风控由模块层负责。

---

## 一、方案定位

### 1.1 核心原则

1. **排序与风控解耦**：排序信号源只负责"排好序"，不负责熔断、仓位、预算。模块层面已有完善的风控方法，排序层不重复造轮子。
2. **三个信号源并列独立**：不是 Layer 1 → Layer 2 → Layer 3 的串行过滤，而是三个可独立运行、独立调用的排序引擎。
3. **量化三种信号**：分别量化单品种内排序、联动簇排序、全域排序三种视角，由上层策略决定用哪个或如何组合。

### 1.2 与旧方案的本质区别

| 维度 | 旧方案（串行三层） | 新方案（并列信号源） |
|------|------------------|-------------------|
| 架构关系 | Layer 1 输出 → Layer 2 输入 → Layer 3 输入 | 信号源A/B/C 各自独立，无依赖 |
| 职责边界 | 排序+风控+熔断+仓位混在一起 | 排序只做排序，风控归模块层 |
| 使用方式 | 必须三层全跑 | 可单独用A、或B、或C，或组合 |
| 扩展性 | 改一层可能影响下游 | 改一个信号源不影响其他两个 |
| 回测对比 | 只能对比"三层vs单层" | 可任意对比 A vs B vs C vs A+B |

## 三、信号源A：单品种月份排序（Intra-product Sorter）

### 3.1 职责边界

- **做**：读取单一品种下挂期权五态数据，按月份加权排序，输出最优月份
- **不做**：跨品种比较、GMM聚类、历史分位数、熔断、仓位建议

### 3.2 输入

```python
{
    'product_id': 'IF',
    'month_data_list': [
        {
            'month': '2507',           # 月份代码
            'opt_type': 'CALL',        # CALL/PUT
            'counts': {'CR': 120, 'CF': 80, 'WR': 30, 'WF': 20, 'Other': 10},
            'last_update': {'CR': ts, 'CF': ts, 'WR': ts, 'WF': ts, 'Other': ts},
            'greeks': {'delta': 0.45, 'gamma': 0.03, 'theta': -0.5, 'vega': 0.2},
            'option_price': 45.0,
            'month_type': 'front_month',  # front_month / next_month / quarter_month
        },
        # ... 最多5个月份
    ]
}
```

### 3.3 核心计算流程

```python
# 1. 非对称状态时效衰减
decayed = decay_engine.decay_counts(counts, last_update, now)

# 2. 月份加权（当月35%、下月25%、季月1 20%、季月2 12%、季月3 8%）
weights = (0.35, 0.25, 0.20, 0.12, 0.08)

# 3. 方向净得分 = (CR + CF - WR - WF) / Total，范围 [-1, 1]
net_score = (cr_eff + cf_eff - wr_eff - wf_eff) / total_eff

# 4. 加权得分 = net_score × month_weight
weighted_score = net_score × w

# 5. 排序输出
sorted_by_weighted_score DESC
```

### 3.4 可选：Greeks硬过滤（参数化）

信号源A 支持可选的硬过滤，但**过滤是参数化的，不是强制的**。

```python
# 配置项
ENABLE_HARD_FILTER = False  # 默认关闭，由上层策略决定

DELTA_MIN = 0.15
DELTA_MAX = 0.85
GAMMA_MAX_FRONT_MONTH = 0.08
THETA_MAX_FRONT_MONTH_PCT = 0.015
```

> **关键决策**：硬过滤是否开启，由上层策略根据市场环境决定，不是排序层写死的。

### 3.5 可选：纯净模式（pure_mode）

```python
sorter = IntraProductSorter(pure_mode=True)  # 零过滤、零修正
```

纯净模式下，`corrected = dict(decayed)`，直接用原始衰减值排序，不做任何 Greeks 修正。

### 3.6 输出

```python
{
    'signal_source': 'A',           # 标识信号源
    'product_id': 'IF',
    'best_month': '2507',
    'best_score': 0.245,
    'correct_up_pct': 0.72,
    'tier': 1,                      # 1/2/3/4，基于 correct_up_pct
    'month_count': 5,
    'candidates': [
        {'month': '2507', 'score': 0.245, 'correct_up_pct': 0.72},
        {'month': '2508', 'score': 0.180, 'correct_up_pct': 0.65},
        # ...
    ],
    'timestamp': 1719300000.0,
}
```

## 四、信号源B：联动品种簇排序（Inter-product Cluster Sorter）

### 4.1 职责边界

- **做**：在品种簇内部比较各品种的信号源A输出，识别共振，输出簇内排序
- **不做**：强制过滤品种（共振弱不等于不能交易，排序只排不杀）、仓位分配

### 4.2 输入

```python
{
    'cluster_id': 'index_futures',  # 品种簇标识
    'products': ['IF', 'IH', 'IC', 'IM'],
    'product_results': [
        # 各品种的信号源A输出
        {'product_id': 'IF', 'best_score': 0.245, 'correct_up_pct': 0.72, ...},
        {'product_id': 'IH', 'best_score': 0.180, 'correct_up_pct': 0.65, ...},
        # ...
    ]
}
```

### 4.3 核心计算

```python
# 1. 簇平均正确率
group_accuracy = sum(p['correct_up_pct'] for p in products) / len(products)

# 2. 各品种共振度 = 品种正确率 / 簇平均正确率
for p in products:
    p['resonance'] = p['correct_up_pct'] / group_accuracy

# 3. 共振加权得分（可选，参数化）
if ENABLE_RESONANCE_WEIGHTING:
    p['score_b'] = 0.70 * p['best_score'] + 0.30 * (p['resonance'] - 1.0)
else:
    p['score_b'] = p['best_score']  # 默认：不额外加权，只排不杀

# 4. 排序输出
sorted_by_score_b DESC
```

### 4.4 可选：共振否决

```python
ENABLE_RESONANCE_VETO = False  # 默认关闭
RESONANCE_VETO_THRESHOLD = 0.80

# 如果启用，低于阈值的品种标记为 vetoed，但不删除，仍保留在排序中
if p['resonance'] < RESONANCE_VETO_THRESHOLD:
    p['vetoed'] = True
```

> **原则**：排序只标记，不删除。是否交易由上层风控决定。

### 4.5 输出

```python
{
    'signal_source': 'B',
    'cluster_id': 'index_futures',
    'group_accuracy': 0.68,
    'sorted_products': [
        {'product_id': 'IF', 'score_b': 0.245, 'resonance': 1.06, 'vetoed': False},
        {'product_id': 'IC', 'score_b': 0.180, 'resonance': 0.97, 'vetoed': False},
        # ...
    ],
    'timestamp': 1719300000.0,
}
```

## 五、信号源C：全域品种排序（Global Cross-product Sorter）

### 5.1 职责边界

- **做**：对所有品种的信号源A输出做全局比较，输出跨品种排名
- **不做**：仓位分配、风险预算、熔断判断

### 5.2 核心计算

```python
# 1. 收集所有品种的信号源A输出
all_results = [sorter_a.rank(p) for p in all_products]

# 2. 全局排序（直接按 best_score 降序）
all_results.sort(key=lambda x: -x['best_score'])

# 3. 可选：历史分位数参考（仅作为参考标记，不做硬性门槛）
for i, r in enumerate(all_results):
    r['global_rank'] = i + 1
    r['global_percentile'] = (len(all_results) - i) / len(all_results)
```

### 5.3 输出

```python
{
    'signal_source': 'C',
    'total_products': 50,
    'sorted_products': [
        {'product_id': 'IF', 'global_rank': 1, 'best_score': 0.32, 'global_percentile': 0.98},
        {'product_id': 'AU', 'global_rank': 2, 'best_score': 0.28, 'global_percentile': 0.96},
        # ...
    ],
    'timestamp': 1719300000.0,
}
```

## 六、信号源组合与上层使用

### 6.1 独立使用

```python
# 只用信号源A：找单品种内最优月份
sorter_a = IntraProductSorter(pure_mode=True)
result_a = sorter_a.rank('IF', month_data)

# 只用信号源B：找指数簇内最优品种
sorter_b = InterProductClusterSorter(cluster_id='index_futures')
result_b = sorter_b.rank(product_results)

# 只用信号源C：找全市场最优品种
sorter_c = GlobalSorter()
result_c = sorter_c.rank(all_product_results)
```

### 6.2 组合使用（由上层策略决定）

```python
# 示例：取信号源B（簇内）和信号源C（全局）的交集
cluster_top3 = [p['product_id'] for p in result_b['sorted_products'][:3]]
global_top10 = [p['product_id'] for p in result_c['sorted_products'][:10]]

final_selection = set(cluster_top3) & set(global_top10)
```

### 6.3 与模块层风控的边界

| 职责 | 排序信号源 | 模块层风控 |
|------|-----------|-----------|
| 排序 | ✅ 三个信号源 | ❌ 不参与 |
| 熔断 | ❌ 不做 | ✅ 模块层已有 |
| 仓位上限 | ❌ 不做 | ✅ 模块层已有 |
| 单日回撤控制 | ❌ 不做 | ✅ 模块层已有 |
| 黑名单/白名单 | ❌ 不做 | ✅ 模块层已有 |
| 止损/止盈 | ❌ 不做 | ✅ 模块层已有 |

## 八、旁路观测与数据决策

### 8.1 旁路观测通道

在信号源A中，可选启用旁路观测，记录"有过滤"与"无过滤"两种路径的排序差异。

```python
BYPASS_MODE_ENABLED = False  # 仅研究环境启用
```

旁路观测独立运行，不影响主排序输出。

### 8.2 三种信号源的独立回测

| 回测对象 | 验证目标 | 基准对比 |
|---------|---------|---------|
| 信号源A（纯净版） | 五态加权本身是否有预测力 | 随机选择 / 等权 |
| 信号源A（过滤版） | 硬过滤是否提升预测力 | 信号源A（纯净版） |
| 信号源B | 簇共振是否提供增量信息 | 信号源A |
| 信号源C | 全局排序是否有增量信息 | 信号源A |
| 组合（A+B） | 组合是否优于单一 | 信号源A / 信号源B |

### 8.3 决策流程

```
Phase 0：前置验证
    │
    ├── 信号源A（纯净版）回测通过？
    │       ├── NO → 五态信号本身无效，方案终止
    │       └── YES → 继续
    │
    ├── 信号源A（过滤版） vs 纯净版
    │       ├── 过滤版显著更优 → 启用硬过滤
    │       ├── 纯净版显著更优 → 禁用硬过滤
    │       └── 差异不显著 → 默认禁用（简单优先）
    │
    ├── 信号源B vs 信号源A
    │       ├── B显著更优 → 启用簇共振加权
    │       └── 否 → 默认禁用
    │
    └── 信号源C vs 信号源A
            ├── C显著更优 → 启用全局排序
            └── 否 → 默认禁用
```

> **极简启动**：只做 Phase 0（信号源A纯净版），验证五态信号本身是否有效。有效再继续，无效则终止，不浪费资源。

## 十、风险声明

1. **五态信号有效性风险**：若信号源A（纯净版）回测不通过，整个方案失去基础。
2. **过度工程化风险**：旧方案的教训——不要在排序层做风控，不要在排序层做三层递进。
3. **参数膨胀风险**：三个信号源各有参数，需分层验证，避免同时调参导致过拟合。
4. **旁路观测开销**：研究环境可承受，生产环境必须关闭。
