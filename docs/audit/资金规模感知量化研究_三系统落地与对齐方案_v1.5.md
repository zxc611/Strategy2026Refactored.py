## 一、核心哲学转变：从夏普到可承担风险下的盈亏比

### 1.1 投资哲学层面的校准

本次优化的核心不是"换一个指标"，而是**投资哲学层面的根本转变**：

| 维度 | 夏普比率思维（机构/大资金） | 可承担风险下的盈亏比思维（小资金） |
|:---|:---|:---|
| **追求目标** | 每单位总波动的超额收益最大化 | 每单位**可承担亏损**的潜在收益最大化 |
| **风险定义** | 净值波动的标准差（双向波动都算风险） | 单笔交易预设的最大亏损额（只把不利波动当风险） |
| **收益定义** | 平稳、持续的超额收益 | 脉冲式、非线性的高盈亏比回报 |
| **核心问题** | "我的收益稳不稳？" | "我亏1块钱，最终能赚回几块钱？" |
| **典型策略** | 多品种分散、低杠杆、高频交易 | 弹簧策略、高共振突破、期权跨式 |
| **数学表达** | `(Rp - Rf) / σp` | `Σ(盈利额) / Σ(亏损额)` 或 `平均盈利% / 平均亏损%` |

### 1.2 为什么弹簧策略用夏普衡量会失真

弹簧策略天然是盈亏比导向的：**多次小亏，换取一次大赚**（胜率~20%，盈亏比~5:1）。

用夏普比率衡量弹簧策略的问题：
- 夏普把**向上波动也视为风险**，弹簧策略的大赚一笔会被惩罚
- 弹簧策略收益分布严重右偏，夏普假设正态分布不成立
- 结果：夏普低→评判不通过→错失优质策略

**结论：不是策略有问题，是尺子不匹配。**

#### 3.1.1 新增 `_ClosedTrade` 数据类

```python
@dataclass
class _ClosedTrade:
    pnl: float              # 净盈亏（含滑点佣金）
    pnl_pct: float          # 盈亏百分比
    close_reason: str       # 平仓原因（StopProfit/StopLoss/HardTimeStop/EOD/...）
    hold_minutes: float     # 持仓时长（分钟）
    open_reason: str = ""   # 开仓原因（CORRECT_RESONANCE/BOX_SPRING/...）
```

**设计决策**：记录`pnl_pct`而非仅`pnl`，因为盈亏比是百分比维度的比值，绝对金额会受资金规模影响。

新增字段 `closed_trades: List[_ClosedTrade]`，初始化为空列表。

```python
    pnl=net_pnl, pnl_pct=pnl_pct, close_reason=close_reason,
    hold_minutes=hold_min, open_reason=getattr(pos, 'open_reason', ''),
))
```

#### 3.1.4 新增 `_compute_profit_loss_ratio_metrics()` 函数

从交易级数据计算以下12项盈亏比统计指标：

| 指标 | 说明 | 计算方式 |
|:---|:---|:---|
| `win_loss_ratio` | 盈亏比 | `avg_win_pct / avg_loss_pct` |
| `profit_factor` | 盈利因子 | `Σ(win_pnl) / Σ(abs(loss_pnl))` |
| `avg_win_pct` | 平均盈利% | 盈利交易的平均pnl_pct |
| `avg_loss_pct` | 平均亏损% | 亏损交易的平均abs(pnl_pct) |
| `win_rate` | 胜率 | `win_trades / total_trades` |
| `total_trades` | 总交易数 | len(closed_trades) |
| `win_trades` | 盈利交易数 | count(pnl > 0) |
| `loss_trades` | 亏损交易数 | count(pnl < 0) |
| `max_consecutive_losses` | 最大连续亏损次数 | 遍历交易序列计算 |
| `max_consecutive_wins` | 最大连续盈利次数 | 遍历交易序列计算 |
| `recovery_efficiency` | 回撤恢复效率 | `dd_depth / (avg_recovery_bars * initial_equity)` |
| `calmar` | Calmar比率 | `年化收益 / 最大回撤` |

#### 3.1.5 更新7个回测函数返回值

以下函数的返回字典均追加 `**plr_metrics`：

1. `run_backtest` — 主策略回测
2. `run_backtest_box_extreme` — 箱体极值回测
3. `run_backtest_box_spring` — 箱体弹簧回测
4. `run_backtest_hft` — 高频回测
5. `run_backtest_arbitrage` — 套利回测
6. `run_backtest_market_making` — 做市回测
7. `run_backtest_hft_with_disturbance` — HFT干扰测试回测
8. `run_backtest_hft_tick_fidelity` — HFT tick保真度回测

#### 3.2.4 新增 `_judge_profitability_by_scale` — 资金规模感知评分

核心逻辑：

```python
# 小资金模式（profit_loss_ratio）：
score = 0.40 * wl_score + 0.25 * pf_score + 0.20 * re_score + 0.15 * cl_score

# 中资金模式（balanced）：
score = 0.20 * wl + 0.15 * pf + 0.20 * sharpe + 0.20 * calmar + 0.15 * re + 0.10 * cl

# 大资金模式（sharpe_dominant）：
score = 0.30 * sharpe + 0.30 * calmar + 0.10 * wl + 0.10 * pf + 0.10 * re + 0.10 * cl
```

各子项评分公式：
- `wl_score = min(1.0, win_loss_ratio / full_score_at)` — 线性映射
- `pf_score = min(1.0, profit_factor / full_score_at)` — 线性映射
- `re_score = min(1.0, recovery_efficiency / full_score_at)` — 线性映射
- `cl_score = max(0.0, 1.0 - (consecutive_losses - full) / (zero - full))` — 线性衰减
- `sharpe_score / calmar_score` — 使用边际递减函数（与v1.4一致）

## 六、三种资金规模的投资哲学映射

### 6.1 小资金（< 50万）：盈亏比主导

**核心问题**："亏1块能赚回几块？"

- 可以承受多次小亏，只要一次大赚能覆盖所有亏损
- 弹簧策略（胜率20%，盈亏比5:1）是理想选择
- 通过门槛降低到0.65，因为高盈亏比策略天然波动大
- 恢复时限放宽到48h/72h，因为小资金可以等待

**评分公式**：
```
score = 0.40 × 盈亏比 + 0.25 × 盈利因子 + 0.20 × 恢复效率 + 0.15 × 连亏容忍
```

### 6.2 中资金（50万~500万）：平衡模式

**核心问题**："在追求盈亏比的同时，收益是否足够稳定？"

- 盈亏比和Sharpe/Calmar各占一半权重
- 适合共振策略（胜率40%，盈亏比2:1）
- 通过门槛0.70，恢复时限36h/48h

**评分公式**：
```
score = 0.20 × 盈亏比 + 0.15 × 盈利因子 + 0.20 × Sharpe + 0.20 × Calmar 
      + 0.15 × 恢复效率 + 0.10 × 连亏容忍
```

### 6.3 大资金（> 500万）：Sharpe主导

**核心问题**："收益是否足够稳定？波动是否可控？"

- Sharpe + Calmar 占60%权重，机构标准
- 适合高频策略、做市策略（胜率55%，盈亏比1.2:1）
- 通过门槛0.75，恢复时限24h（机构要求快速恢复）

**评分公式**：
```
score = 0.30 × Sharpe + 0.30 × Calmar + 0.10 × 盈亏比 + 0.10 × 盈利因子 
      + 0.10 × 恢复效率 + 0.10 × 连亏容忍
```

## 七、关键设计决策

| 决策点 | 选择 | 理由 |
|:---|:---|:---|
| 盈亏比统计在哪个层级计算 | 回测函数内（`_compute_profit_loss_ratio_metrics`） | 需要访问`closed_trades`和`equity_curve`，仅在回测函数内有完整数据 |
| 评判模式切换方式 | `capital_scale`参数 | 比修改全局配置更安全，支持同一进程内多规模并行评判 |
| 向后兼容策略 | 不传`capital_scale`时走原有逻辑 | 避免破坏现有调用链路 |
| `_ClosedTrade`记录`pnl_pct` | 是 | 盈亏比是百分比维度，绝对金额受资金规模影响 |
| 小资金连亏满分线=3 | 弹簧策略典型连亏3-5次 | 3次以内属于正常波动 |
| 小资金通过门槛=0.65 | 弹簧策略Sharpe通常<1.0 | 原有0.75门槛会把优质弹簧策略误杀 |
