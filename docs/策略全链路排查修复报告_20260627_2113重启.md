#### 根因13详细分析

**问题**：`R23-FR-04-FIX`的K线过期淘汰逻辑（877行）：
```python
if _tick_epoch > 0 and (time.time() - _tick_epoch) > _kline_max_age:  # 60s
    return  # 丢弃tick
```

模拟交易环境中tick时间戳是历史交易时间（如`1782527884` ≈ 2026-06-26交易时段），而`time.time()`当前约`1782543600`（2026-06-27 21:xx）。差值约15716秒(4.4小时) >> 60秒阈值，导致**所有20000+/30s的tick全部被丢弃**。

日志级别为debug，生产环境不可见，导致此问题长期隐藏。

**修复**：
1. 仅淘汰"未来时间戳"（`_tick_epoch > time.time() + 300`）
2. 仅淘汰"比同合约上一个tick更旧超过阈值"的tick（`_tick_epoch < _prev_tick_epoch - _kline_max_age`）
3. 日志级别从debug提升为warning
4. 新增`_kline_age_last_tick_epoch`字典记录每个合约的上一个tick时间戳
