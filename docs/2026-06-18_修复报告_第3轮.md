### 1.2 根因分析

#### 问题1: get_cache_stats方法不存在

**代码路径**:
- `width_cache.py:28` - `WidthStrengthCache` 门面类
- `width_cache_diagnosis_mixin.py:11` - `WidthCacheDiagnosisMixin` 诊断服务
- `width_cache_query_mixin.py:127` - `get_cache_stats` 定义位置

**问题机制**:
```python
# width_cache.py
class WidthStrengthCache(WidthCacheStateService):
    def __init__(self, ...):
        self._diagnosis_service = WidthCacheDiagnosisMixin()
        # ❌ 缺少: self._diagnosis_service._facade = self

# width_cache_diagnosis_mixin.py
class WidthCacheDiagnosisMixin:
    # ❌ 缺少: __init__ 和 __getattr__ 方法
    def print_status_diagnosis(self, ...):
        if hasattr(self, 'get_cache_stats'):  # ❌ self是DiagnosisMixin实例，不是门面
            ...
```

**修复方案**:
```python
# width_cache_diagnosis_mixin.py
class WidthCacheDiagnosisMixin:
    def __init__(self):
        self._facade = None

    def __getattr__(self, name):
        if self._facade is not None:
            for klass in type(self._facade).__mro__:
                if name in klass.__dict__:
                    attr = klass.__dict__[name]
                    if hasattr(attr, '__get__'):
                        return attr.__get__(self._facade, type(self._facade))
                    return attr
        raise AttributeError(...)

# width_cache.py
self._diagnosis_service = WidthCacheDiagnosisMixin()
self._diagnosis_service._facade = self  # ✅ 设置反向引用
```

#### 问题2: 历史K线加载跳过

**代码路径**:
- `lifecycle_callbacks.py:115` - 设置 `p._subscribed_instruments`
- `historical_data_manager.py:81` - 从 `state_store` 获取 `_subscribed_instruments`
- `lifecycle_bind.py:242` - 调用 `_start_historical_kline_load()`

**问题机制**:
```python
# lifecycle_callbacks.py:115
p._subscribed_instruments = p._init_instruments_result['subscribed_instruments']
# ❌ 未同步到 state_store

# historical_data_manager.py:81
subscribed = list(self._state_store.get('_subscribed_instruments') or [])
# ❌ 返回空列表，因为 state_store 中没有该键
```

**修复方案**:
```python
# lifecycle_callbacks.py:115
p._subscribed_instruments = p._init_instruments_result['subscribed_instruments']
_ss = getattr(p, '_state_store', None)
if _ss is not None:
    try:
        _ss.set('_subscribed_instruments', p._subscribed_instruments)
    except (ValueError, KeyError, TypeError, AttributeError):
        pass
```

#### 问题3: _status_counts为空

**这是正常现象**，不是bug：

- `register_option()` 只注册期权基本信息（internal_id, underlying_future_id, month, opt_type）
- `_status_counts` 的计算需要tick数据触发 `_recalc_all_state_for_underlying()`
- 策略刚启动时，尚未收到tick数据，所以 `_status_counts` 为空是正常的
