## 六、架构设计说明

### 6.1 模块定位

```
L0 基础设施:  threading, logging, json, yaml
L2 业务层:   shadow_strategy_engine.py  ← 本次新增
L3 策略层:   strategy_core_service.py  ← 集成修改
```

**依赖方向：** L3 → L2 → L0（单向，无循环）

### 6.2 核心类结构

```
ShadowStrategyEngine
├── 影子A(反向逻辑)
│   ├── process_shadow_a_signal()  → long↔short, buy↔sell
│   └── 参数: incorrect_reversal (独立锁定)
├── 影子B(随机动作)
│   ├── process_shadow_b_signal()  → random.choice(['long','short'])
│   └── 参数: correct_trending (独立锁定)
├── Alpha比率监控
│   ├── compute_alpha_metrics()    → Sharpe/MaxDD/Alpha/EV
│   ├── 连续衰减检测              → >20%连续2窗→降级
│   └── 绝对期望值底线            → EV<0→暂停实盘
├── 独立日志
│   ├── _flush_trade_log()        → JSONL(按类型+日期)
│   ├── generate_daily_summary()  → summary_*.json
│   └── generate_weekly_summary() → weekly_*.json
└── 健康检查
    └── get_health_status()       → OK/CRITICAL
```

### 6.3 关键设计决策

1. **参数独立锁定时机**：在`__init__`中自动锁定，而非等Walk-forward完成。理由：引擎必须在任何时候可用，参数可在Walk-forward后通过`relock_params()`更新。

2. **MIN_TRADES_FOR_METRICS=5**：绝对期望值底线触发至少需要5笔交易，避免早期少数交易导致误触发。

3. **主策略交易is_open=False**：`record_master_trade()`记录时直接标记为关闭（PnL已知），与影子策略的开仓→平仓两阶段不同。

4. **惰性初始化**：`strategy_core_service`中影子引擎在首次使用时才加载（`get_shadow_strategy_engine()`单例），避免启动时循环依赖。

5. **异常隔离原则**：影子引擎任何异常不影响主策略交易（_feed_shadow_engine用try/except+debug日志）。
