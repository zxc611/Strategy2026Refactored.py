## 3. 根因分析

### 3.1 根因1（核心）：Timer调用对象错误

**问题代码**：[strategy_2026.py:666](file:///C:/Users/xu/AppData/Roaming/InfiniTrader_QhZijintianfengPythonX64/pyStrategy/demo/ali2026v3_trading/strategy/strategy_2026.py#L666)

```python
_auto_start_timer = threading.Timer(0.1, self.on_start)
```

`self` 是 `Strategy2026` 实例，`self.on_start` 是 `Strategy2026.on_start` 的绑定方法。

**调用路径对比**：

| 版本 | Timer调用对象 | 调用路径 | t_type_bootstrap.on_start()是否被调用 |
|------|--------------|---------|--------------------------------------|
| 2026-07-13 16:20（工作） | `t_type_bootstrap.on_start` | Timer → t_type_bootstrap.on_start() → real.on_start() + self.trading=True | ✓ 被调用 |
| 当前V5（失效） | `Strategy2026.on_start` | Timer → Strategy2026.on_start()（绕过t_type_bootstrap） | ✗ 未被调用 |

**影响**：
- `t_type_bootstrap.on_start()` 从未被Timer调用
- `t_type_bootstrap.on_start()` 中的 `self.trading = True` 未执行
- 虽然 `_sync_trading_to_outer_ref(True)` 补偿了 `trading=True`，但**C++可能通过检测 `t_type_bootstrap.on_start()` 是否被调用来判断策略启动状态**

### 3.2 根因2（次要）：_sync_trading_to_outer_ref提前同步trading=True

**问题代码**：[strategy_2026.py:256-269](file:///C:/Users/xu/AppData/Roaming/InfiniTrader_QhZijintianfengPythonX64/pyStrategy/demo/ali2026v3_trading/strategy/strategy_2026.py#L256)

Timer调用 `Strategy2026.on_start()` 时，`_sync_trading_to_outer_ref(True)` 立即将 `t_type_bootstrap.trading` 设为True。

**影响**：
- C++可能检测到 `t_type_bootstrap.trading=True`，认为策略已在运行
- 当用户点击"执行"按钮时，C++认为策略已在运行，跳过 `on_start()` 调用
- 但14:32 C++在trading=True时仍调用on_start，说明这不是唯一原因

### 3.3 根因3（环境因素）：C++状态机间歇性卡住

**证据**：
- 12:04-12:21：V5代码相同，C++正常调用on_start
- 13:16-14:06：V5代码相同，C++不调用on_start
- 14:32：V5代码相同，C++恢复调用on_start

**可能原因**：
- C++内部状态机因Timer绕过 `t_type_bootstrap.on_start()` 而进入不确定状态
- C++可能在某些时序条件下卡住，等待 `t_type_bootstrap.on_start()` 被正式调用
- 14:32恢复的原因不明（可能是C++内部超时重置，或用户操作触发的某种状态转换）

### 3.4 排除的假设

| 假设 | 证据 | 结论 |
|------|------|------|
| on_start方法被覆盖 | Grep全项目仅2处定义 | 排除 |
| BaseStrategy有元类/描述符 | base.py L30 普通类 | 排除 |
| __getattr__拦截on_start | L334-340只在属性不存在时触发 | 排除 |
| on_init返回值问题 | 12:04和13:16都返回None | 排除 |
| on_init耗时超时 | 12:04耗时125s，13:16耗时117s，都正常 | 排除 |
| .pyc缓存不同步 | 13:49 .pyc与源码同步 | 排除 |
| trading=True直接阻止C++调用 | 14:32 trading=True时C++仍调用 | 排除（不是直接原因） |

## 4. 修复方案

### 4.1 核心修复：Timer调用t_type_bootstrap.on_start()

**文件**：`strategy_2026.py`
**位置**：[L665-671](file:///C:/Users/xu/AppData/Roaming/InfiniTrader_QhZijintianfengPythonX64/pyStrategy/demo/ali2026v3_trading/strategy/strategy_2026.py#L665)

**修改前**：
```python
try:
    _auto_start_timer = threading.Timer(0.1, self.on_start)
    _auto_start_timer.daemon = True
    _auto_start_timer.start()
    logging.info("[Strategy2026.on_init] auto_start已调度（0.1s后执行），策略将自动进入运行状态")
except Exception as _auto_start_err:
    logging.error("[Strategy2026.on_init] auto_start调度失败(非致命，用户可手动点击执行): %s", _auto_start_err)
```

**修改后**：
```python
try:
    # FIX-20260714-V6: Timer必须调用t_type_bootstrap.on_start()，而非Strategy2026.on_start()
    # 根因: V5代码Timer调用self.on_start(Strategy2026.on_start)，绕过t_type_bootstrap.on_start()
    #       导致C++不认为on_start被正式调用，状态机卡住，不调用on_start/on_stop/on_destroy
    # 修复: 通过_outer_ref获取t_type_bootstrap实例，调用其on_start()
    #       t_type_bootstrap.on_start()会: 1.调用real.on_start() 2.设置self.trading=True 3.日志
    #       这与2026-07-13 16:20工作版本完全一致
    _outer = getattr(self, '_outer_ref', None) or self
    _auto_start_timer = threading.Timer(0.1, _outer.on_start)
    _auto_start_timer.daemon = True
    _auto_start_timer.start()
    _target = "t_type_bootstrap" if _outer is not self else "Strategy2026(fallback)"
    logging.info("[Strategy2026.on_init] auto_start已调度（0.1s后执行，目标=%s），策略将自动进入运行状态", _target)
except Exception as _auto_start_err:
    logging.error("[Strategy2026.on_init] auto_start调度失败(非致命，用户可手动点击执行): %s", _auto_start_err)
```

### 4.2 修复原理

修改后的调用路径：
```
Timer(0.1, _outer.on_start)
  → t_type_bootstrap.on_start()           ← 关键！正式调用入口
    ├─ real = self._ensure_real_strategy()
    ├─ result = real.on_start()           ← Strategy2026.on_start()
    │   ├─ IDEMPOTENT检查（_start_executed=False → 完整启动）
    │   ├─ super().on_start() → trading=True
    │   └─ _sync_trading_to_outer_ref(True)
    ├─ self.trading = True                ← t_type_bootstrap.trading = True
    └─ logging.info("[t_type_bootstrap] on_start完成, trading=True")
```

与2026-07-13 16:20工作版本完全一致。

### 4.3 不需要修改的部分

- `t_type_bootstrap.py`：当前V4代码（移除auto_start）保持不变，auto_start由strategy_2026.py的Timer负责
- `IDEMPOTENT SKIP`逻辑：保持不变，用户点击执行时C++调用on_start → IDEMPOTENT SKIP
- `_sync_trading_to_outer_ref`：保持不变，仍然需要同步trading状态
