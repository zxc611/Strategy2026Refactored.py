## 4. 手动验证：修复彻底

### H-001 隐藏BUG：`or`链将0当falsy

**修复前代码**：
```python
last_price = tick.get('last_price') or tick.get('LastPrice')
```

**问题**：Python的`or`运算符将0视为falsy值。
- `tick = {'last_price': 0, 'LastPrice': 3500}` → `last_price = 3500`（**错误fallback！**）
- `tick = {'last_price': 0}` → `last_price = None`（被L174捕获return False → DATA_LOSS）

**修复后代码**：
```python
last_price = tick.get('last_price')
if last_price is None:
    last_price = tick.get('LastPrice')
```

**验证**：
- `tick = {'last_price': 0}` → `last_price = 0` → L174 `0 is None` = False → 不fallback → **正确**
- `tick = {'last_price': None, 'LastPrice': 3500}` → fallback到3500 → **正确**
- `tick = {'last_price': 0, 'LastPrice': 3500}` → `last_price = 0` → 不fallback → **正确**（0是结算tick真实价格）

**结论**：根因彻底消除。原始`or`链是P0-1的**真正根因**——修复前last_price=0永远到不了L181。

### P0-1 价格=0降级验证

**修复后路径**：
- `tick = {'last_price': 0, ...}` → L173取值0 → L174 `0 is None` False → L179 `not isinstance(0,(int,float))` False, `0 < 0` False → **不进入warning分支** → L190 `return True`
- **结论**：结算tick价格=0被正确放行，零DATA_LOSS。
