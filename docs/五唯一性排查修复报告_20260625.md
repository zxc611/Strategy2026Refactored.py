# 生产策略代码五唯一性排查修复报告

> **排查日期**: 2026-06-25
> **排查范围**: `ali2026v3_trading/` 全部生产策略代码（排除 tests/、docs/、__pycache__/、tests/precompute_fixtures/）
> **排查原则**: 保留一处其它调用（Keep-One-Others-Call）
> **排查轮次**: 两轮（第一轮独立排查 + 第二轮交叉验证）

---

## 一、排查方法论

### 1.1 五唯一性定义

| 维度 | 含义 | 检查标准 |
|------|------|----------|
| **ID唯一** | 每个模块的 `MODULE_ID` 全局唯一 | 同一 ID 不得出现在多个文件 |
| **数据入口\出口唯一** | 模块导入路径唯一，数据读写入口唯一 | 同一模块不得用两种路径导入；同一数据源不得多处独立定义 |
| **方法唯一** | 一个地方同一件事只做一次 | 同一算法/逻辑不得在多处独立实现 |
| **传递链路（键）唯一** | 常量、枚举、配置键全局唯一 | 同一常量不得在多处赋不同值；同一枚举不得多处定义 |
| **正则唯一** | 同类正则表达式全局唯一 | 同一用途的正则不得有不兼容的多版本 |

### 1.2 两轮排查流程

- **第一轮**: 启动 5 个并行搜索子代理，分别针对五个维度独立扫描全部生产代码，发现 42 项违规。
- **第二轮**: 启动 2 个交叉验证子代理，对第一轮结果进行复核（确认/调整/补充），确认 14 项、调整 3 项严重度、发现 2 项比报告更严重、0 项误报，并额外发现 43 项违规。

**合计发现约 85 项违规**（含重复项）。经第二轮去重合并后实际归并为 **约 75 项独立违规**，全部已修复（详见第五节统计摘要）。
