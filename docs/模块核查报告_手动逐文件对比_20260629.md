## 零、统计方法与可信度声明

### 0.1 行数统计口径

| 口径 | 定义 | 本报告采用 |
|------|------|------------|
| **total行** | `len(f.readlines())`，含空行/注释/docstring | **是** |
| code行 | 仅含可执行代码行 | 否 |
| SLOC | 去除空行和注释的有效代码行 | 否 |

**选择理由**: total行是唯一无歧义的统计口径，任何工具（`wc -l`/`Get-Content`/`len(readlines())`）结果一致。code行/SLOC需定义何为"注释"、何为"代码"，不同工具结果不同。

### 0.2 历史报告行数偏差根因

| 报告 | M1行数 | 实际行数 | 偏差 | 根因 |
|------|--------|----------|------|------|
| 0629旧版 | 88,111 | 121,614 | -27.5% | 统计的是git HEAD版本（已提交旧代码），非磁盘当前版本 |
| 0616版 | 121,294 | 121,614 | -0.3% | 部分文件取了git行数，部分取了磁盘行数；governance/lifecycle文件名错误 |

### 5.2 短期：自动化行数统计

在`scripts/`下创建`count_lines.py`，CI中集成：

```python
"""统计M1各子系统文件数和行数，输出Markdown表格。"""
import os, glob

SUBSYSTEMS = [
    ('strategy', 'm1-01'), ('config', 'm1-02'), ('data', 'm1-03'),
    ('risk', 'm1-04'), ('risk_engine', 'm1-05'), ('order', 'm1-06'),
    ('position', 'm1-07'), ('signal', 'm1-08'), ('governance', 'm1-09'),
    ('lifecycle', 'm1-10'), ('infra', 'm1-11'),
]

def count_py(directory):
    total_files, total_lines = 0, 0
    for f in sorted(glob.glob(os.path.join(directory, '**', '*.py'), recursive=True)):
        with open(f, 'r', encoding='utf-8') as fh:
            total_lines += len(fh.readlines())
        total_files += 1
    return total_files, total_lines

if __name__ == '__main__':
    print(f"| 子系统 | 文件数 | 行数 |")
    print(f"|--------|--------|------|")
    grand_files, grand_lines = 0, 0
    for name, _ in SUBSYSTEMS:
        files, lines = count_py(name)
        print(f"| {name}/ | {files} | {lines:,} |")
        grand_files += files
        grand_lines += lines
    print(f"| **M1总计** | **{grand_files}** | **{grand_lines:,}** |")
```
