## 二、深度验证发现的隐患及修复

### 隐患：check_pending_orders锁外撤单时订单状态可能已变

- **发现阶段**: 深度验证P2-14
- **严重度**: 🟡 中等
- **根因**: 锁外执行`cancel_order`时，订单状态可能已被`on_trade_update`修改为FILLED。使用快照数据追单可能导致参数不正确
- **修复**: 锁外撤单前重新获取锁检查最新状态，仅对仍为SUBMITTED/PENDING的订单执行撤单和追单

**修复前**:
```python
for order_id, order in timeout_orders:
    self.cancel_order(order_id)  # 可能撤已成交订单
    retry_count = order.get('retry_count', 0)  # 使用过时快照
    if retry_count < self._max_chase_retries:
        self._chase_reorder(order, retry_count + 1)
```

**修复后**:
```python
for order_id, order_snapshot in timeout_orders:
    with self._lock:
        current_order = self._orders_by_id.get(order_id)
        if not current_order or current_order['status'] not in ('SUBMITTED', 'PENDING'):
            continue  # 状态已变，跳过
    self.cancel_order(order_id)
    retry_count = current_order.get('retry_count', 0) if current_order else order_snapshot.get('retry_count', 0)
    if retry_count < self._max_chase_retries:
        self._chase_reorder(current_order or order_snapshot, retry_count + 1)
```

## 五、关键修改详细说明

### 5.1 BUG-E5 排序桶双向信号支持

**修改点1** — `_update_sort_bucket` 入口条件：
```python
# 修改前：只允许correct_rise
if current_status != 'correct_rise':
    self._remove_from_sort_bucket(...)
    return

# 修改后：同时允许correct_fall
if current_status not in ('correct_rise', 'correct_fall'):
    self._remove_from_sort_bucket(...)
    return
```

**修改点2** — `select_otm_targets_by_volume` 期货方向选择：
- 期货上涨(`_future_rising=True`) → 选CALL → 统计correct_rise
- 期货下跌(`_future_rising=False`) → 选PUT → 统计correct_rise（PUT的正确上涨=期货下跌+PUT上涨）

### 5.2 BUG-E10 cooldown_seconds参数生效

新增 `_cooldown_durations` 字典，`set_cooldown` 时存储自定义冷却时长，`_is_in_cooldown` 优先使用自定义时长。

### 5.3 BUG-H21 追单加价递增

```python
# 修改前：每次只加1个tick
new_price = original_order['price'] + tick_size * 1

# 修改后：按重试次数递增（1tick, 2ticks, 3ticks）
chase_ticks = min(retry_count, 3)
price_offset = tick_size * chase_ticks
```

### 5.4 BUG-H29 锁外撤单

将超时订单的撤单和追单操作移到锁外执行，避免平台I/O阻塞锁持有期间影响其他线程。
