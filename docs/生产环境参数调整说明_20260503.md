# 生产环境参数调整说明

**文档版本**: 1.0  
**生成日期**: 2026-05-03  
**适用系统**: ali2026v3_trading 策略模块

---

## 一、概述

本文档说明策略从调试环境部署到生产环境时需要调整的关键参数。这些参数直接影响系统对全量数据的接收能力和运行性能。

### 1.1 当前状态

| 检查项 | 当前状态 | 生产要求 |
|--------|----------|----------|
| 内存模式 | `debug`（仅保留最新1条tick） | `production`（全量接收） |
| 队列容量 | ✅ 1000万（16×62.5万） | 已满足 |
| Writer线程 | ✅ 6个并发写入 | 已满足 |
| 全量上限 | 500万tick | 建议5000万 |

### 1.2 风险等级

| 等级 | 参数 | 不调整的后果 |
|------|------|-------------|
| 🔴 P0 | `INTRADAY_MEMORY_MODE` | 全量tick数据被丢弃，仅保留每品种最新1条 |
| 🟠 P1 | `INTRADAY_MAX_TOTAL_TICKS` | 极端行情下超过500万tick后开始丢弃 |
| 🟡 P2 | `INTRADAY_MAX_TICKS_PER_SYMBOL` | 高频品种超过1000条后开始丢弃 |

---

## 二、必须调整的参数

### 2.1 INTRADAY_MEMORY_MODE（P0 必须调整）

**当前默认值**: `debug`  
**生产环境值**: `production`

**影响说明**:
- `debug` 模式：`_full_capture=False`，RealTimeCache 仅保留每品种最新1条tick
- `production` 模式：`_full_capture=True`，RealTimeCache 全量接收并存储tick序列

**代码位置**: `data_service.py:29-32`
```python
_INTRADAY_MODE = os.getenv('INTRADAY_MEMORY_MODE', 'debug').lower()
_INTRADAY_FULL_CAPTURE = _INTRADAY_MODE in ('production', 'prod', 'full')
```

### 2.2 INTRADAY_MAX_TOTAL_TICKS（P1 建议调整）

**当前默认值**: `5,000,000`  
**生产环境建议值**: `50,000,000`

**影响说明**:
- RealTimeCache 全量tick存储上限
- 超过上限后新tick会被丢弃并记录警告日志

**容量估算**:
```
极端行情全天tick量：
- 50个品种 × 高频品种100万tick/天 = 5000万tick
- 建议设置：5000万（约10GB内存）
```

### 2.3 INTRADAY_MAX_TICKS_PER_SYMBOL（P2 可选调整）

**当前默认值**: `1,000`  
**生产环境建议值**: `10,000`

**影响说明**:
- 每个品种保留的tick序列最大长度
- 高频品种（如IF/IC/IM）可能需要更大值

## 三、调整方法

### 方法一：环境变量（推荐）

**Windows 系统**:
```cmd
:: 临时设置（当前终端有效）
set INTRADAY_MEMORY_MODE=production
set INTRADAY_MAX_TOTAL_TICKS=50000000
set INTRADAY_MAX_TICKS_PER_SYMBOL=10000

:: 永久设置（系统环境变量）
setx INTRADAY_MEMORY_MODE "production"
setx INTRADAY_MAX_TOTAL_TICKS "50000000"
setx INTRADAY_MAX_TICKS_PER_SYMBOL "10000"
```

**Linux 系统**:
```bash
# 临时设置（当前终端有效）
export INTRADAY_MEMORY_MODE=production
export INTRADAY_MAX_TOTAL_TICKS=50000000
export INTRADAY_MAX_TICKS_PER_SYMBOL=10000

# 永久设置（添加到 ~/.bashrc 或 /etc/environment）
echo 'export INTRADAY_MEMORY_MODE=production' >> ~/.bashrc
echo 'export INTRADAY_MAX_TOTAL_TICKS=50000000' >> ~/.bashrc
echo 'export INTRADAY_MAX_TICKS_PER_SYMBOL=10000' >> ~/.bashrc
source ~/.bashrc
```

## 四、参数优先级

参数加载遵循以下优先级（从高到低）：

```
环境变量 > config.json 配置文件 > params_default.json 默认值 > 代码硬编码默认值
```

**示例**:
- 环境变量 `INTRADAY_MEMORY_MODE=production` 会覆盖配置文件中的 `debug` 默认值
- 配置文件中的值会覆盖代码中的硬编码默认值

## 五、验证方法

### 5.1 启动日志验证

策略启动时会输出以下日志，确认参数已生效：

```
[IntradayMemoryState] Initialized mode=FULL_CAPTURE max_per_symbol=10000 max_total=50000000 env=INTRADAY_MEMORY_MODE=production
```

**验证要点**:
- `mode=FULL_CAPTURE` 表示生产模式已启用
- `max_per_symbol` 和 `max_total` 应为配置的值

### 5.2 运行时验证

通过诊断接口检查当前状态：

```python
from ali2026v3_trading.data_service import get_realtime_cache

cache = get_realtime_cache()
stats = cache.get_stats()
print(f"模式: {stats.get('mode')}")
print(f"当前tick数: {stats.get('total_ticks')}")
print(f"品种数: {stats.get('symbol_count')}")
```

### 5.3 环境变量检查

**Windows**:
```cmd
echo %INTRADAY_MEMORY_MODE%
echo %INTRADAY_MAX_TOTAL_TICKS%
echo %INTRADAY_MAX_TICKS_PER_SYMBOL%
```

**Linux**:
```bash
echo $INTRADAY_MEMORY_MODE
echo $INTRADAY_MAX_TOTAL_TICKS
echo $INTRADAY_MAX_TICKS_PER_SYMBOL
```

## 六、内存需求估算

### 6.1 全量数据内存需求

| 配置项 | 计算公式 | 内存需求 |
|--------|----------|----------|
| RealTimeCache tick存储 | 5000万 × 200字节/tick | ~10GB |
| Shard Queue 缓冲 | 1000万 × 200字节/tick | ~2GB |
| DuckDB 内存 | 默认75%物理内存 | ~48GB（64GB机器） |
| **总计** | - | **~60GB** |

### 6.2 推荐硬件配置

| 配置项 | 最低要求 | 推荐配置 |
|--------|----------|----------|
| 物理内存 | 32GB | 64GB |
| 磁盘空间 | 100GB | 500GB SSD |
| CPU核心 | 4核 | 8核+ |
