### 问题2: DuckDB临时表不存在
**错误**: `Catalog Error: Table with name temp_ticks_10360 does not exist!`

**根因**: `_TimedDuckDBConnection`的`register`和`execute`方法通过独立的executor任务执行，导致临时视图在不同线程上下文中不可见。

**代码位置**: 
- `ds_db_connection.py:109-116`: register方法使用executor
- `ds_data_writer.py:298-310`: register和execute分离调用

**机制**:
1. `register`通过`_execute_with_timeout`在executor线程A中执行
2. `execute`通过另一个`_execute_with_timeout`在executor线程B中执行
3. DuckDB临时视图是线程本地的，线程B看不到线程A注册的视图

## 修复方案

### 修复1: 更新health_monitor.py测试合约
将所有2605改为2607：
- `diagnose_subscribe_api_return`: al2605C25200 → al2607C25200 等
- `diagnose_option_metadata_diff`: al2605C25200 → al2607C25200 等

### 修复2: 原子化Arrow插入操作
**核心思路**: 将`register`+`insert`+`unregister`合并为单个原子操作，确保在同一线程执行。

**修改位置**: `ds_data_writer.py:293-340`

**实现**:
1. 定义`_atomic_insert_with_arrow`函数，封装register+insert+unregister
2. 检测是否为`_TimedDuckDBConnection`实例
3. 如果是，使用`conn._execute_with_timeout`执行整个原子操作
4. 如果不是，直接调用原子操作函数

**关键代码**:
```python
def _atomic_insert_with_arrow(_conn, _view_name, _arrow_table, _columns_str, _in_transaction):
    _conn.register(_view_name, _arrow_table)
    _conn.execute(f"INSERT INTO ticks_raw ... FROM {_view_name}")
    _conn.unregister(_view_name)

if hasattr(conn, "_execute_with_timeout") and hasattr(conn, "_conn"):
    conn._execute_with_timeout(_atomic_insert_with_arrow, conn._conn, ...)
else:
    _atomic_insert_with_arrow(conn, ...)
```
