    │       └─► 线程A查询失败（2354次）

#### 根因详情

**根因1：ticks_raw表字段缺失（主因）**

- **问题：** `ds_db_connection.py`使用`CREATE TABLE IF NOT EXISTS`创建表
- **影响：** 表已存在时不会重建，旧表缺少`bid_volume/ask_volume`字段
- **表现：** 所有tick写入失败，错误信息：`Table "ticks_raw" does not have a column with name "bid_volume"`
- **修复：** 改为`DROP TABLE IF EXISTS + CREATE TABLE`强制重建

**根因2：temp_ticks表不存在（次因）**

- **问题：** 多线程共享内存数据库连接时，temp_ticks视图被互相覆盖
- **影响：** 线程A注册视图后，线程B清理视图，导致线程A查询失败
- **表现：** 错误信息：`Table with name temp_ticks does not exist!`
- **修复：** 使用线程唯一视图名`temp_ticks_{threading.get_ident()}`

**根因3：合约订阅但无数据落盘**

- **问题：** 平台推送tick到策略，但写入失败导致数据未落盘
- **影响：** 合约监控显示"NO_TICK"（实际是tick收到但写入失败）
- **表现：** 20个合约全部显示无tick
- **修复：** 修复根因1和根因2后，tick写入成功，数据正常落盘

**根因链路：**
```
tick处理 → get_risk_service() 
         → SingletonRegistry.register_singleton("risk_service.default", result)
         → ServiceContainer().register(name, instance)
         → 每次创建新ServiceContainer实例
         → register()不检查重复
         → 日志输出registered
```

**修复方案：**

1. **ServiceContainer单例模式：**
```python
# service_container.py
class ServiceContainer:
    _instance = None
    _instance_lock = threading.RLock()
    
    def __new__(cls):
        with cls._instance_lock:
            if cls._instance is None:
                cls._instance = super().__new__(cls)
                cls._instance._initialized = False
            return cls._instance
    
    def __init__(self):
        if getattr(self, '_initialized', False):
            return
        self._initialized = True
        self._services: Dict[str, Any] = {}
        # ...
```

2. **register方法幂等检查：**
```python
def register(self, name: str, service: Any) -> None:
    """注册服务（带层级校验）"""
    if name in self._services:
        return  # 已注册则直接返回
    self._services[name] = service
    # ...
```

**修复效果：**
- ServiceContainer全局唯一实例
- register方法幂等检查
- risk_service.default仅注册1次
