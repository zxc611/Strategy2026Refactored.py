### 2.3 误报分析

3项误报的详细分析：

**M-1 `_processed_trade_ids` 无界增长**
```python
# strategy_core_service.py L1724-1725
if len(self._processed_trade_ids) > 10000:
    self._processed_trade_ids.clear()
```
评判报告遗漏了此清理逻辑。实际已有10000上限+clear()机制，不存在无界增长。clear()虽激进（全量清除而非LRU），但trade_id是短生命周期标识，重复处理的风险极低。

**M-2 greeks_calculator `_thread_local` 导入未使用**
当前 greeks_calculator.py 中不存在 `_thread_local` 或 `threading.local` 的任何导入或引用。此问题可能在早期版本中存在，已被修复。

**M-3 `_anchor_vwap_order` LRU清理未触发**
```python
# order_flow_analyzer.py L294-297
while len(self._anchor_vwap_order) > self._max_anchor_orders:
    oldest_key = next(iter(self._anchor_vwap_order))
    del self._anchor_vwap_order[oldest_key]
```
LRU清理在 `get_anchor_vwap` 方法中已实现，每次访问时检查并清理超限条目。评判报告称"未在任何调用路径上触发清理"不准确。

### 3.2 P2-3：order_flow_analyzer.py 注释与代码对齐

**问题**：`_linear_trend_slope` 方法注释声称"使用Welford算法增量计算"，但实际代码使用批量O(n)均值法。Welford算法是一种在线/增量计算方差和协方差的数值稳定算法，而当前实现是标准的批量线性回归。

**修复**：将注释更正为实际算法描述。

```python
# 修改前
    @staticmethod
    def _linear_trend_slope(points: List[Tuple[float, float]]) -> float:
        """
        计算一组点 (x, y) 的线性回归斜率
        优化：使用Welford算法增量计算，减少数值误差
        """

# 修改后
    @staticmethod
    def _linear_trend_slope(points: List[Tuple[float, float]]) -> float:
        """
        计算一组点 (x, y) 的线性回归斜率
        使用归一化时间戳 + 批量均值法，O(n)复杂度
        """
```

**验证**：`python -m py_compile order_flow_analyzer.py` ✅ 通过

### 3.3 P2-4：strategy_core_service.py 锁职责文档化

**问题**：4把 `RLock` 无注释说明各自保护的数据范围和职责，维护者无法判断加锁顺序和死锁风险。

**修复**：为每把锁添加行内职责注释。

```python
# 修改前
        # 线程锁
        self._lock = threading.RLock()
        self._scheduler_lock = threading.RLock()
        self._state_lock = threading.RLock()
        self._trading_lock = threading.RLock()

# 修改后
        # 线程锁（职责说明）
        self._lock = threading.RLock()              # 主锁：保护 _stats, _subscribed_instruments 等共享状态
        self._scheduler_lock = threading.RLock()    # 调度锁：保护 _scheduler_manager 的 job 操作
        self._state_lock = threading.RLock()        # 状态锁：保护 StrategyState 状态转换（start/stop/pause）
        self._trading_lock = threading.RLock()      # 交易锁：保护 on_trade 成交处理的互斥（防重复成交）
```

**锁层次分析**（隐含加锁顺序，避免死锁）：
1. `_state_lock` — 最外层，状态转换入口
2. `_lock` — 中间层，共享状态读写
3. `_scheduler_lock` — 内层，调度器操作
4. `_trading_lock` — 独立层，成交处理

**验证**：`python -m py_compile strategy_core_service.py` ✅ 通过

### 3.5 M-5：data_service.py DB_FILE/PARQUET_PATH 配置外部化

**问题**：`DB_FILE` 和 `PARQUET_PATH` 硬编码为 `D:\ali2026_data\` 路径，仅通过 `os.getenv()` 回退，未集成到 ConfigService 统一配置中心。

**修复**：实现三级优先级配置解析链（环境变量 > ConfigService > 默认值）。

```python
# 修改前
DB_FILE = os.getenv('DUCKDB_FILE', 'D:\\ali2026_data\\ticks.duckdb')
PARQUET_PATH = os.getenv('TICK_DATA_PATH', 'D:\\ali2026_data\\ticks.parquet')

# 修改后 - 三级优先级解析
def _resolve_duckdb_file():
    env_val = os.getenv('DUCKDB_FILE')
    if env_val:
        return env_val
    cfg = _get_data_paths_config()
    if cfg and cfg.duckdb_file:
        return cfg.duckdb_file
    return 'D:\\ali2026_data\\ticks.duckdb'

DB_FILE = _resolve_duckdb_file()
PARQUET_PATH = _resolve_parquet_path()
```

**ConfigService 集成**：新增 `DataPathsConfig` 数据类，包含 `duckdb_file`、`parquet_path`、`flush_windows_str`，支持环境变量覆盖和 config.json 配置。

**验证**：`python -m py_compile data_service.py` ✅ 通过，`python -m py_compile config_service.py` ✅ 通过

### 3.6 M-6：RealTimeCache._flush_windows 配置外部化

**问题**：`DEFAULT_FLUSH_WINDOWS = [(12, 0, 12, 50), (15, 30, 20, 0)]` 完全硬编码，无法通过环境变量或配置文件调整。

**修复**：删除 `DEFAULT_FLUSH_WINDOWS` 类变量，改用 `_resolve_flush_windows()` 动态解析，支持 `INTRADAY_FLUSH_WINDOWS` 环境变量。

```python
# 修改前
class RealTimeCache:
    DEFAULT_FLUSH_WINDOWS = [(12, 0, 12, 50), (15, 30, 20, 0)]
    def __init__(self, ..., flush_windows=None):
        self._flush_windows = flush_windows or self.DEFAULT_FLUSH_WINDOWS

# 修改后
class RealTimeCache:
    def __init__(self, ..., flush_windows=None):
        self._flush_windows = flush_windows or _resolve_flush_windows()
```

**环境变量格式**：`INTRADAY_FLUSH_WINDOWS='12:0-12:50,15:30-20:0'`（格式：`时:分-时:分`，逗号分隔多窗口）

**容错**：无效格式自动跳过并回退到默认值 `[(12,0,12,50), (15,30,20,0)]`。

**验证**：`python -m py_compile data_service.py` ✅ 通过，`grep DEFAULT_FLUSH_WINDOWS data_service.py` → 0 matches
