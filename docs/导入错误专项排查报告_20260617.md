## 四、修复标准

### 4.1 通用原则
1. **最小改动**：只修复阻断导入的具体字符/路径，不重构逻辑、不调整格式、不删除空行。
2. **保留语义**：修复后的日志文本应与上下文语义一致；若原文残缺无法还原，以"语义通顺 + 不引入新错误"为准。
3. **不扩大范围**：不修复非导入阻断问题（如命名规范、性能、类型注解）。
4. **UTF-8 无 BOM**：所有 `.py` 文件保存为 UTF-8 无 BOM。

### 4.2 语法错误修复规范
| 错误模式 | 修复规则 |
|----------|----------|
| `unterminated string literal` | 在语义断点处补齐缺失的闭合引号（`"` 或 `'`），保持引号风格与同行一致 |
| `unterminated f-string literal` | 同上，确保 `f"..."` 闭合 |
| `unterminated triple-quoted string` | 补齐 `"""` 或 `'''` |
| `unmatched )` | 删除多余的右括号，使括号配平 |
| `invalid syntax (forgot a comma?)` | 在字符串字面量与后续参数间补 `,` |
| `invalid character 'X'` | 将全角字符替换为半角，或将其纳入字符串字面量内 |
| `leading zeros in decimal integer` | 通常是注释/字符串未闭合导致后续中文被当代码；先修复字符串闭合 |

### 4.3 导入路径修复规范
| 错误模式 | 修复规则 |
|----------|----------|
| `ModuleNotFoundError: No module named 'xxx.l1_quantification.yyy'` | 改为指向迁移后真实位置 `ali2026v3_trading.tests.precompute_fixtures.yyy` |
| `__getattr__` 未代理的子模块 | 优先修改调用方指向真实路径；不在 `__getattr__` 中新增代理（避免掩盖迁移事实） |

### 4.4 验收断言标准
每修复 5 个问题，执行以下断言集合，全部通过方可继续下一批：
```python
# 1. 已修复文件可独立编译
import py_compile; py_compile.compile(<file>, doraise=True)
# 2. 已修复文件可被 import（不抛 SyntaxError / ModuleNotFoundError）
import importlib; importlib.import_module(<dotted_path>)
# 3. 包根入口可导入（最终目标）
import ali2026v3_trading
```
