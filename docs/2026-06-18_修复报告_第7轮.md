## 问题诊断

### 错误现象
- 17:30重启后无日志输出
- strategy.log最后写入：17:20:36
- 进程重启：17:20:53
- 重启后完全无新日志

### 根因分析
RotatingFileHandler在进程重启时可能出现文件句柄问题，导致无法写入新日志。

**日志文件状态**:
```
strategy.log:   38.68MB (未达到100MB轮转阈值)
strategy.log.1: 100MB
strategy.log.2: 100MB
strategy.log.3: 100MB
```

## 修复方案

### 手动触发日志轮转
重命名旧日志文件，强制创建新文件：
```
strategy.log → strategy.log.old
```

重启后，RotatingFileHandler将创建新的strategy.log文件。
