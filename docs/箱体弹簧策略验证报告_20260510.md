## 一、核心验证：最小虚值当月期权监测能力

### 1.1 双通道扫描架构

弹簧策略采用**双通道扫描**架构，确保不遗漏最小虚值当月期权：

```
scan_springs()
├── 通道1: _scan_from_sort_buckets()  — 从WidthStrengthCache排序桶扫描
│   └── 优先扫描最近月(nearest_month)，CALL+PUT双向，top_n=5
│   └── 回退扫描次近月(all_months[1:2])
│
└── 通道2: _scan_from_option_info()   — 从_option_info全量扫描（保底）
    └── 遍历所有期权元数据，筛选虚值+当月+行权价最近
    └── 按distance排序，取每方向前3名
```

**关键设计决策**：
- 通道1利用排序桶的O(1)查询效率，但只包含五态分类中的期权
- 通道2从`_option_info`全量扫描，**确保所有虚值期权都能被检测到**
- 通道1无结果时自动降级到通道2，保证零遗漏

### 1.2 通道2：最小虚值当月期权筛选逻辑

```python
# _scan_from_option_info() 核心筛选链：

1. underlying_future_id == future_id    → 同标的
2. CALL: future_price < strike (虚值)    → 最小虚值Call
   PUT:  future_price > strike (虚值)    → 最小虚值Put
3. distance = |future_price - strike| / future_price ≤ 3%  → 行权价附近
4. min_days_to_expiry ≤ DTE ≤ max_days_to_expiry (2-5天)  → 当月近月
5. candidates.sort(key=lambda x: x['distance'])  → 按虚值程度排序
6. 取每方向前3名 → 最小虚值的当月期权
```

**验证结论：✅ 通道2确保能监测到最小虚值的当月期权品种。**

### 1.3 通道1 vs 通道2 对比

| 维度 | 通道1 (sort_buckets) | 通道2 (option_info) |
|------|---------------------|---------------------|
| 数据源 | WidthStrengthCache排序桶 | _option_info全量字典 |
| 扫描范围 | 仅五态分类中的期权 | **所有已注册期权** |
| 月份优先 | 最近月优先 | DTE 2-5天过滤 |
| 虚值筛选 | 依赖桶内排序 | **显式虚值判断** |
| 效率 | O(1)查询 | O(n)遍历 |
| 遗漏风险 | 可能遗漏非五态期权 | **零遗漏** |

## 四、Gamma暴露估算验证

### 4.1 BS模型Gamma计算

```python
d1 = (ln(S/K) + 0.5*σ²*T) / (σ*√T)
gamma = e^(-0.5*d1²) / (S * σ * √T * √(2π))
gamma_exposure = gamma * S² / premium
```

### 4.2 平值/轻度虚值附近的特性

- 当 S ≈ K（平值附近），d1 ≈ 0，gamma达到最大值
- gamma_exposure = gamma * S² / premium 衡量"每元权利金的Gamma暴露"
- 弹簧策略要求 distance ≤ 2%，确保在Gamma最敏感区域

**验证结论：✅ Gamma暴露估算在平值/轻度虚值附近准确，与策略设计一致。**


## 八、期望值模型验证

```
假设参数：
  胜率 = 20% (5次中1次脉冲成功)
  平均盈利 = 2.0x - 1.0x = 1.0x (止盈2倍)
  平均亏损 = 0.95x (止损95%归零)

期望值 = 20% × 1.0 - 80% × 0.95
       = 0.20 - 0.76
       = -0.56

修正：实际盈利时pnl_ratio=2.0，亏损时pnl_ratio=0.05
  平均盈利 = 2.0 - 1.0 = 1.0 (100%收益)
  平均亏损 = 1.0 - 0.05 = 0.95 (95%亏损)

期望值 = 20% × 1.0 - 80% × 0.95 = -0.56 (负期望!)

需要调整：胜率需达到 ~49% 或止盈比需达到 ~4.8x 才能正期望
```

**建议**：当前参数下期望值为负，建议将`stop_profit_ratio`从2.0提升至5.0（匹配文档"1赔5"的描述），或将`iv_low_percentile`从15%降至5%（更严格的弹簧识别提升胜率）。
