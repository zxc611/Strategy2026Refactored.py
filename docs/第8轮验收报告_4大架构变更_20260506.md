## 1. 任务1：合并3条期权注册路径为1个方法1处位置

### 1.1 问题
存在3条独立的期权注册路径，竞态条件、重复注册、不一致的数据来源导致CU/ZN/AU期权5态判定失败。

### 1.2 修复

| 路径 | 修改前 | 修改后 |
|------|--------|--------|
| **路径A** (`t_type_service._preload_from_db`) | 后台线程，从DB的option_instruments+latest_prices查询 | **删除** |
| **路径B** (`strategy_lifecycle_mixin._preload_options_into_t_type`) | analytics warmup后台线程，从DB的ticks_raw窗口函数查询 | **删除** |
| **路径C** (`subscription_manager._auto_register_options_to_ttype`) | on_start后订阅完成触发，从params_service metadata查 | **删除** |
| **唯一入口** (`_init_t_type_service_and_preload`) | 仅调路径B | **直接从配置文件metadata注册**，同步阻塞，失败raise RuntimeError |

### 1.3 删除代码统计

| 文件 | 修改前行数 | 修改后行数 | 删减 |
|------|-----------|-----------|------|
| t_type_service.py | 640 | 339 | -301 |
| strategy_lifecycle_mixin.py | 1570 | 1511 | -59 |
| subscription_manager.py | 1569 | 1458 | -111 |

### 1.4 验证

## 2. 任务2：合约配置文件作为订阅唯一来源 + 初始化失败必须中断

### 2.1 问题
策略加载配置文件失败后重试带病运行，不中断初始化，导致期权5态异常。

### 2.2 修复

| 变更项 | 修改前 | 修改后 |
|--------|--------|--------|
| `_start_analytics_warmup_async` | 后台daemon线程，异常仅logging | **同步阻塞执行**，RuntimeError传播到on_init |
| `_init_t_type_service_and_preload` | try/except吞异常，设t_type_service=None | 失败**raise RuntimeError**，on_init捕获后transition_to(ERROR) |
| 期货注册0个 | warning日志 | **raise RuntimeError**中断初始化 |
| 期权注册0个 | warning日志 | **raise RuntimeError**中断初始化 |
| t_type_service未创建 | 设None继续 | **raise RuntimeError**中断初始化 |

### 2.3 验证
- [x] `_start_analytics_warmup_async`改为同步调用`_init_analytics_services`
- [x] RuntimeError在期货0注册时抛出
- [x] RuntimeError在期权0注册时抛出（仅当options_total>0时）
- [x] on_init的except块会捕获RuntimeError并transition_to(ERROR)，返回False
- [x] 语法检查通过
