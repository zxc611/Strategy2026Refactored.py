### 2.2 止盈止损价格计算流程

```
_add_position()
  → _get_tp_sl_ratios_by_reason(open_reason)  [position_service.py:489-504]
    → 优先: StateParamManager.get_params(open_reason) 读取动态参数
    → 回退: _HARDCODED_REASON_DEFAULTS[open_reason]     读取硬编码默认值
  → _apply_crm_stop_loss_adjustment(sl_ratio, open_reason)  [position_service.py:524-536]
    → CycleResonanceModule.get_risk_surface(strategy) 获取止损倍数
  → 计算止盈止损价:
    多头: sp_price = open_price × tp_ratio
          sl_price = open_price × (1 - sl_ratio)
    空头: sp_price = open_price / tp_ratio
          sl_price = open_price × (1 + sl_ratio)
```

### 2.3 时间止损PLR弹性调整

```
_check_time_stop(record)
  → _map_reason_to_strategy(open_reason) → strategy
  → crm.get_risk_surface(strategy) → max_hold_seconds
  → PLR弹性:
    plr_ratio ≥ 1.5 (EXCELLENT): adjusted_hold = max_hold × 1.5
    plr_ratio ≥ 1.2 (GOOD):      adjusted_hold = max_hold × 1.2
    plr_ratio < 0.8 (POOR):      adjusted_hold = max_hold × 0.6
  → elapsed ≥ adjusted_hold → 触发平仓 "TimeStop@{min}min(plr_adj)"
```

### 7.1 问题发现与分析

#### 7.1.1 日志分析结果（22:05~23:05）

#### 7.1.2 四个核心BUG根因

**BUG-1: 平仓价格fallback导致胖手指误判**
- **现象**: 平仓价格fallback到开仓价 → 胖手指防护用市价作参考价 → 偏离24%~47% → 被拦截 → 重试耗尽
- **根因**: `OrderContext`缺少`ref_price`字段，胖手指防护无法获取正确的参考价格
- **影响**: TimeStop每5秒重新触发，形成死循环

**BUG-2: 重试耗尽后错误重置_closing标志**
- **现象**: 重试耗尽后重置`_closing=False` → TimeStop重新触发 → 死循环
- **根因**: 缺少"CANNOT_CLOSE"终态，无法阻止重复尝试
- **影响**: 日志刷屏，资源浪费

**BUG-3: 订单字典清理导致成交回调失败**
- **现象**: `_orders_by_id`容量清理 → 已成交CLOSE订单被删除 → `on_trade`查找失败
- **根因**: 平台返回`direction=0, offset=0`不正确，fallback查找失败
- **影响**: 持仓状态不一致，平仓订单丢失

**BUG-4: R37-TRADING-RECOVERY绕过风控**
- **现象**: R37对OPEN订单也恢复trading → 绕过风控保护
- **根因**: 未区分OPEN/CLOSE订单类型
- **影响**: 风控失效，潜在风险

#### 7.1.3 "半拉子工程"问题

**问题1: 间接平仓路径绕过标准流程**
- 4个间接平仓路径（熔断减仓/紧急平仓/弹簧平仓/HFT追单）绕过`_trigger_close_position`
- 未传`ref_price`，导致胖手指防护失效

**问题2: open_signal_snapshot是死字段**
- 写入不读取，未参与决策
- 设计初衷：根据开仓信号成就状态决定平仓策略

**问题3: close_method是死字段**
- 仅日志输出，不参与决策
- 设计初衷：根据平仓方式决定重试策略

**问题4: 两阶段策略覆盖不全**
- `_TWO_STAGE_STRATEGY_OVERRIDES`缺`box`/`high_freq`
- 导致部分策略无法享受两阶段止损保护

**问题5: 时间止损无差异化**
- `_check_time_stop`的`max_hold_minutes`不按`strategy_group`差异化
- box策略和普通策略使用相同时间阈值

#### 7.1.4 隔夜仓时间问题

- **当前实现**: `elapsed = (now - record.open_time).total_seconds() / 60` — 自然时间差
- **问题**: 周五15:00开仓 → 周一9:00检查，自然时间66小时，但实际交易时间只有几小时
- **影响**: 隔夜仓被误判为超时，提前平仓
