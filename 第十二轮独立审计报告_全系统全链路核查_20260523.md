1. **数值精度是系统性隐患**：67项问题覆盖除零、溢出、NaN传播、浮点误差、未来数据泄漏，测试系统的回测结果可信度存疑
2. **Pandas操作存在全链路风险**：21项问题中2项P0涉及数据行错位和类型崩溃，直接影响生产系统数据质量
3. **环境兼容性是部署炸弹**：23项问题中4项P0可在部署时直接导致系统无法启动
4. **交易所规则适配全面缺失**：46项问题中14项P0，涨跌停、交割日、保证金、持仓限额、tick_size对齐等核心规则均未实现，实战中可能导致无效委托、资金损失、超限交易

### 8.2 建议

1. **立即修复**：27项P0问题（尤其交易所规则适配的涨跌停/交割日/PnL乘数）
2. **优先修复**：十二轮共识P0问题（10项）
3. **规划修复**：累计~752项问题的分级修复路线图

```python
def _calc_pnl(self, direction: str, open_price: float, close_price: float, 
              volume: int, instrument_id: str = '', is_close_today: bool = False) -> float:
    contract_size = self.DEFAULT_CONTRACT_MULTIPLIER
    commission_rate = self.DEFAULT_COMMISSION_RATE
    if instrument_id:
        try:
            from ali2026v3_trading.params_service import get_params_service
            params_svc = get_params_service()
            meta = params_svc.get_instrument_meta_by_id(instrument_id)
            if meta:
                product = meta.get('product', '')
                product_cache = params_svc.get_product_cache(product)
                if product_cache:
                    contract_size = safe_float(product_cache.get('contract_size', 1.0))
                    commission_rate = safe_float(product_cache.get('commission_rate', self.DEFAULT_COMMISSION_RATE))
        except Exception:
            pass
    if direction in ('long', '买', 'BUY'):
        raw_pnl = (close_price - open_price) * volume * contract_size
    else:
        raw_pnl = (open_price - close_price) * volume * contract_size
    # 手续费扣减
    open_commission = abs(open_price * abs(volume) * contract_size * commission_rate)
    close_commission = abs(close_price * abs(volume) * contract_size * commission_rate)
    # 平今差异化
    if is_close_today:
        try:
            from ali2026v3_trading.config_params import get_config
            cfg = get_config()
            close_today_mult = cfg.get('close_today_commission_multiplier', 1.0)
            close_commission *= close_today_mult
        except Exception:
            pass
    total_commission = open_commission + close_commission
    return raw_pnl - total_commission
```

### 19.3 C部分：本轮修复隐患排查

1. **Sharpe ddof=1修改影响**：修改后Sharpe值会系统性略降（样本标准差>总体标准差），但这是正确的统计方法。需确认下游评判系统是否依赖旧的偏高Sharpe值。经验证，strategy_judgment_engine.py的Sharpe一致性检查使用独立计算，不受影响。

2. **_option_multiplier参数化影响**：默认值保持10000向后兼容，但新增params入口允许动态配置。无破坏性变更。

3. **exclude_rollover_signals copy修复影响**：添加copy()后内存使用略增，但避免了数据污染。回测管线中bar_data通常<1M行，copy开销可接受。

4. **calendar_days.fillna(-1)影响**：NaN→-1后trading_days=0，下游days_to_expiry=0会触发交割日风控检查(EX-05)，属于安全降级。

5. **_contract_changed首行True影响**：首行强制标记为合约边界，strike_price/expire_date等字段首行被重置为NaN后由ffill填充。若首行数据本身正确，ffill会立即恢复，无实质影响。
